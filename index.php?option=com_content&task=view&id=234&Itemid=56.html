中小企業資訊系統建置系列（五）電子報發送系統 PHPList

Created at Thursday, 19 January 2006 08:00      Last Updated on Wednesday, 12 October 2011 11:04 Written by 林崇仁
行銷一直是企業拓展市場，提供消費者新產品或服務的重要的方法，早期的行銷包括、廣告、報紙、雜誌、或是傳單等等。而不論是哪一種方式的行銷，都需要花費 企業許多額外的成本，而這成本又不是一般的中小型企業所能負擔的。拜於網路的普及，中小企業也能夠以低成本，透過網路來做無區域限制的網路行銷。不過要如 何做、是否有好的軟體可以使用？這又是另一門的學問。本期筆者要分享的是一套功能強大的開放源碼電子報發送系統— PHPList。 開放源碼軟體 PHPList是一套 webbase 的系統，它解決了一般發送電 子報的幾個主要的問題。

1. 建立及維護電子報發送對象
就像 PHPList 的名字，系統本身是以 List 的方式來建立發送的對象，主要可以依據不同的發送對象來建立 List。假設你有一批的客戶群是屬於製造業，那你便可以建立一個以製造業為主的 List。可是問題來了，一般電子報的名單，數量可能會高達萬人甚至上百萬，如何將龐大的資料建置至系統是一個大問題？在這裡，PHPList 除了由系統 key in 資料外，還提供與不同的資料庫作連結，以及 CSV 的匯入的介面來處理龐大的資料。另外，也可以藉由 PHPList 所提供的網路介面，來讓大眾訂閱電子報。

另一個使用上的問題是，如果您要對一個以上的 List 去發送電子報時，而有可能有某些的客戶是隸屬於好幾個不同的 List，如何避免將相同的電子報不斷寄送給相同的電子郵件信箱？PHPList 系統本身就做好防範了，相同的電子報不會重複發送到相同的電子郵件信箱。

2. 大量電子報的發送
當名單的資料庫建置完成後，另一個接踵而至的問題來囉。如果您的電子報系統在短時間內發送出幾百萬的電子郵 件時，你所遇到的問題將會是，你的 ISP (Internet Service Provider) 馬上會打電話警告你，發垃圾郵件是不被允許的，甚至會將你的網路停用。PHPList 提供一個批次發送的機制，您可以設定每個時段發送固定量的郵件，如此一來，ISP 業者就不會盯上你，而且也不會造成公司每次在發送電子報時，就造成公司網路的壅塞。

3. 知道電子報的發送結果
另一個重要的問題是，您如何知道電子報發送的結果？PHPList 本身有很好的統計介面，你可以知道在這次的發送中，有多少的電子報是成功的發送，而有多少的電子報是被退回 (Bounce)。值得一提的是，你可以設定當某個電子郵件信箱被 Bounce 多少次，該電子郵件信箱便為無效的，你可以藉此機制來維持名單的有效性。

總結來說，PHPList 是一個相當好用而且成熟的系統。不過由於現在垃圾郵件滿天飛，許多的公司也都架設防垃圾郵件的軟體，所以另外一個重要的課題就是如何不讓你的電子報被 spam。這裡筆者的幾個建議是：

  1. 申請比較知名的 ISP 廠商
  2. 向 ISP 業者申請 IP 反解服務
  3. 避免以純圖檔寄送
  4. 避免附加特殊的壓縮檔案
  5. 電子報裡不附加網路釣魚的連結
當然依據不同的 Spam 系統設定，對垃圾郵件的定義都不同，以上的只是幾個大方向讓大家參考。不過最重要也要提醒大家的是，藉由系統散播的訊息或電子報是有意義的資訊，不要造成電子郵件使用者的困擾。
