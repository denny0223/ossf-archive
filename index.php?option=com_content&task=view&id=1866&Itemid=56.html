Linux 散佈套件已相容美國國防部 IPv6 標準

建立日期 2008-12-09 08:00      作者是 謝良奇/編譯

Linux 基金會 (Linux Foundation) 宣佈所有主要 Linux 散佈套件目前皆已支援 IPv6。該非營利組織表示，這些散佈套件商已符合美國國防部 (Department of Defense) 訂定的採購標準，並已可供部署於 IPv6 網路之中。

2005 年美國政府要求所有單位採購的電腦與網路設備，都必須符合下一代的網際網路協定需求。美國國防部為求加速進階到 IPv6 技術，將 2008 設定為完成此一程序的目標年度。主要軟體廠商為避免被排除於美國國防部採購案之外，必須實現規格需求，並驗證其產品符合國防資訊系統局 (Defense Information Systems Agency) 的 IPv6 特別相容運行認證 (IPv6 Special Interoperability Certification)。美國國防部並於今年 6 月強制實施此一認證。

儘管 Linux 從 2005 年起已經有了相對穩固的 IPv6 支援，然而為完全達成美國國防部標準，許多工作仍待完成。Linux 基金會 IPv6 工作小組分析了美國國防部認證要求，並找出 Linux IPv6 堆疊為達完全相容性所必須完成的調整。

Linux 基金會 IPv6 工作小組 (IPv6 Workgroup) 日前宣佈所有主要的 Linux 散佈套件，如今已符合美國國防部的認證政策。此一相容性工作，由 IBM、紅帽 (Red Hat)、Novell、Nokia 與若干主要 Linux 廠商共同推動。

IPv6 為網際網路協定第 6 版 (Internet Protocol Version 6)，一舉將 IP 位址擴充至 128 位元，由於大幅增加可用的位址數量，有助簡化網路路由。鑑於 IPv4 可用位址逐漸減少乃至於未來完全耗盡，採用 IPv6 勢在必行，然而相關轉換工作仍進展緩慢。

Linux 基金會成立的 IPv6 工作小組由 IBM 的 Linux 網路中心的首席網路架構師 Venkata Jagana 帶領，其任務便是要讓基於 Linux 的裝置，能夠符合下一代網際網路標準。根據該組織表示，大多數常見的商業級 Linux 散佈套件，如今已符合美國國防部標準。

IBM 可說是網際網路工程工作小組 (Internet Engineering Task Force，IETF) 早期制定 IPv6 的參與者，並於 1997 年在其 AIX UNIX 作業系統中實作此一協定。數年後 IBM 便開始投入 Linux 的 IPv6 開發。

IBM 的 Kathy Bennett 在聲明中指出，在 2000 年，IBM 已經體認到 Linux 必須符合 IPv6 標準與相容運行性，並與 Linux 社群和散佈套件商合作做出開發貢獻。該成果與 Linux 基金會 IPv6 工作小組的努力，今日領先產業達成美國國防部 IPv6 認證，對於 Linux 產業有莫大幫助。

紅帽今年 8 月便已率先宣佈，其商業級 Linux 散佈套件在通過美國國防部 IPv6 Master Test Plan 之後，已經正式收到認證。Linux 基金會則持續努力，讓所有主要散佈套件如今都足以通過此一測試。此舉也彰顯 Linux 基金會的價值，該組織可作為論壇，讓所有使用與部署 Linux 的公司齊聚一堂，協同解決共有的問題。

Linux 基金會執行董事 Jim Zemlin 聲明指出，此一 IPv6 強制需求是相當重大的工作，其困難度讓任何一家公司都無法獨自完成，這正是 Linux 基金會得以推動的協同工作，並且達成了 Linux 作業系統上實際的技術進展。
