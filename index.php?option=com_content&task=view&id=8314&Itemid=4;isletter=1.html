美國白宮釋出 IT 專案支出追蹤工具 IT Dashboard

建立日期 2011-04-14 00:00      最近更新在 2011-04-25 11:58 作者是 謝良奇／編譯
美國白宮 (White House) 日前釋出了 IT Dashboard 和 TechStat 工具集的原始程式碼。美國聯邦政府與協助政府為公共利益分享技術的 Code for America，加上 OpenPlans 的支援，透過 Code for America 的程式碼分享專案 Civic Common，推動了這項釋出計畫。 

2009 年推出的美國聯邦 IT Dashboard，是最受到外界認可的政府透明化計畫之一。此計畫得以讓公眾追蹤重大 IT 專案上的聯邦 IT 支出、狀態、投資報酬。政府運用這套工具評估專案效果以便更有效管理 700 億美元的年度 IT 預算。最近一份美國管理和預算辦公室 (Office of Management and Budget) 的報告指稱，在一項聯邦政府採用並於部署後 2 年內節省超過 30 億美元的程序中，IT Dashboard 扮演了主要角色。

聯邦資訊長 Vivek Kundra 在白宮部落格上指出，他們推出 IT Dashboard 與 TechStat Accountability Sessions，為的是改善聯邦政府的 IT 透明度與信責度。IT Dashboard 能協助他們更了解 IT 專案，提供 TechStat 彙報所需的效能資料。Kundra 表示，向公眾釋出 IT Dashboard 與 TechStat 工具集原始碼有兩個原因，首先，為了將該平台推向更高水準，他們希望利用集體才智和美國民眾的巧思，以加強其功能性，改進程式碼並解決現有挑戰。其次，全國與全球各地的資訊長，例如荷蘭的 Maarten Hillenaar、西維吉尼亞州的 Kyle Schafer、芝加哥市的 Jason DeHaan，都有興趣在各自的組織裡運用這些平台。 

Kundra 補充說，IT Dashboard 協助聯邦政府更有效管理其 IT 投資，如今其程式碼透過開放源碼的形式公開釋出，將能協助其他組織達成同樣目標。軟體開發者將能為 Dashboard 找出錯誤、開發優化功能、做出建議改善，並發掘他們未曾想像過的應用。TechStat 工具集為組織提供建立自有 TechStats，提供了完整的指引，以改進專案團隊與高階主管間的資訊暢通，提升 IT 計畫狀態衡量的準確度，並提高維持專案進行所需干預的品質與適時性。

作為美國首位聯邦資訊長的 Vivek Kundra，曾提出並擔任執行聯邦 IT 徹底重整的計畫。他應用 IT Dashboard 以追蹤所任職的管理和預算辦公室 IT 支出。鑑於他過去的經歷，Kundra 於 2009 上任時，外界認為這對開放源碼會是利多消息，他很有可能在美國政府中建構並採納開放源碼應用軟體。 

透過這次釋出，由政府機構開發的應用軟體，可以讓不同城市、州、國家的潛在政府使用者或開發者，加以實作或進一步客製化。美國聯邦通信委員會 (Federal Communications Commission) 資料長 Greg Elin 表示，他們不須建構、不須購買、不須採購，還有什麼比這更好的？

美國州際首席資訊官國家協會 (National Association of State CIOs，NASCIO) 讚譽這項釋出決定，並且在聲明中指出，有 38 個州與準州已經表達評估使用該工具的興趣。西維吉尼亞州資訊長同時也是 NASCIO 總裁 Kyle Schafer 表示，各州資訊長的現今優先考量的因素以及面臨的主要挑戰，就是監控 IT 投資的績效。鑑於現今各州預算問題，在這些關鍵問題上得以取用開放源碼模組，可說是價值非凡。

紅帽 (Red Hat) 美國公共部門技術策略長 Gunnar Hellekson 指出，比起該軟體本身，他對該軟體最初是如何釋出更感到興趣。他表示，多年來該政府釋出原始碼，但一直缺乏共同政策，或關於應該釋出方式的看法，今日這項消息為其他機構開啟了一條重要且公開的道路。釋出的不是一些修補程式，而是完整的應用軟體，其他機構可以使用他們的程序，來釋出自己的專案。

IT Dashboard 開放源碼的線上展示已經上線，程式碼可透過 SourceForge 下載，並以 GPL 授權釋出。
