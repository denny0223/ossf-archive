SpikeSource 建構 OSS 專案參與式測試支援架構

Created at Tuesday, 02 August 2005 08:00      Last Updated on Thursday, 08 March 2007 08:11 Written by 謝良奇
開放源碼專案重大版本釋出後的更新、如何評估臭蟲對於相依元件的 影響、如何尋找出最適修正等，都是企業在採納開放源碼會面臨的問 題。不同版本元件在不同硬體上針對不同功能需求的測試，引發組合 爆炸的挑戰。SpikeSource 技術長 Murugan Pal 認為，為應付這些 問題，開放源碼測試的參與架構會是可能的解答。 Pal 並且認為，儘管傳統與現存的開放源碼測試方法論是有用的，卻 沒有完全發揮參與架構的好處，而 SpikeSource 的目標，即在提高 企業對於開放源碼軟體的接納度，其做法包括測試、認證與支援服務。 其核心競爭力是該公司創新、自動化且經最佳化的先進測試技術。這 也是 SpikeSource 的營運模式，協助企業評估、維護，並且在其 IT 架構中納入開放源碼。

基於建構參與式測試系統的前提下，SpikeSource 納入多項做法，如 採用現存開放源碼測試工具並整合為自動化測試的共用框架，同時開 發新工具並貢獻社群；供專案提交者（committers）、客戶分享與彙 整測試結果的界面；發展讓核心專業不同的用戶從各角度進行軟體測 試的方法論。

日前 SpikeSource 並發表旗艦產品 SpikeSource Core Stack 的增 加版本，提供企業預先組態完成的訂製（made-to-order）安裝。開 發者透過此一產品，針對個別用途或客製需要，選取一系列應用軟體、 程式庫、Web 伺服器與其他工具。不久前成為 SpikeSource 的產品 副總裁 Joaquin Ruiz 表示，該產品不僅具訂製能力，更可加以動態 設定與安裝。除了其訂製能力，Forrester Research 資深分析師 Michael Goulde 認為，測試與維護才是 SpikeSource 最具潛力之處。 除了 Joaquin Ruiz，該公司日前也聘請 Anders Tjernlund 及 Sun 共同創辦人 Bill Joy，分別擔任支援服務副總裁與該公司董事會成員。

SpikeSource 執行長，同時也是前 Marimba 執行長 Kim Polese 日前 即於 Software 2005研討會中，發表有關開放源碼長期效應的演說， 她認為開放源碼是促成世界經濟演變的新動力，將改寫軟體產業的運 作模式。Polese 同時也指出開放源碼在引入企業產品環境時遭遇的問 題，在於測試與驗證開放源碼組態。而這正是 SpikeSource 企業模式 著力之處。
