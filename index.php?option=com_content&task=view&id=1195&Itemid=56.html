AMD 將支援 ATI 繪圖晶片的開放源碼 Linux 驅動程式開發

Created at Tuesday, 11 September 2007 16:00      Last Updated on Friday, 02 May 2008 22:06 Written by 謝良奇
AMD 已經決定與開發源碼社群合作，成為 Intel 之後第二家支援開放源碼驅動程式開發的主要繪圖晶片廠，要讓 Linux 用戶也能利用 ATI 繪圖晶片的高階 3D 功能。對於 ATI 兩大新系列繪圖晶片的開放源碼繪圖驅動程式開發，AMD 將提供開發者必要的資訊。

AMD 5 月在紅帽大會（Red Hat Summit）上表示過會加強 ATI 的驅動程式支援，並且將研擬出可滿足開放源碼社群的解決方案。日前在核心開發者大會（Kernel Developers Summit）上，AMD 進一步釋出更多讓開放源碼社群振奮的消息。

ATI 將會開發全功能的 Linux 2D 與 3D 驅動程式，雖然這些驅動程式仍然不會公開原始程式碼，不過該公司將釋出文件，讓第三方開發者自行發展並支援驅動程式。

ATI 過去已經支援 Linux 的私有繪圖驅動程式，但 Linux 用戶經常抱怨 ATI 的驅動程式難以安裝、常出狀況，效能表現也不盡理想，更缺少常見的功能支援，往往要等晶片釋出數月後才能見到相對應的驅動程式出現。

AMD 繪圖產品群軟體副總裁 Ben Bar-Haim 表示，他們近來發現越來越多桌面用戶在使用他們的產品。他們發覺到應該讓社群發展好的開放源碼軟體。AMD 正出資讓 Novell 的 Suse Linux 開發者撰寫驅動程式，提供硬體細節與技術支援。

ATI 過去一向對開放源碼的 Linux 驅動程式不感興趣，這次 ATI 與 AMD 會支援開放源碼驅動程式的確是令人訝異的轉變。Intel 已經在 2006 年投入開放源碼 Linux 驅動程式開發，Nvidia 則仍固守私有軟體陣營，該公司代表並未立即就 ATI 的動作發表評論。

AMD 表示已經與不少開放源碼團體合作，要加強 ATI Radeon 繪圖晶片針對 Linux 消費群用戶的相容性，紅帽系統工程師也是 Mozilla 基金會（Mozilla Foundation）委員會成員之一的 Christopher Blizzard 表示 AMD 會跟 Novell 及紅帽一同開發驅動程式。他並且認為，AMD 肯釋出文件是對開放源碼社群的一大保證。他說，這些人做了正確的事，在支援開放源碼上甚至比 Intel 更進一步。

驅動程式預計於 9 月初釋出，到時候也會包含 ATI 的開發軟體套件 Catalyst，將會支援 AMD 的 ATI Radeon X1000 與 HD 2000 繪圖晶片。AMD 希望先釋出有 2D 功能的開放源碼驅動程式，並逐漸擴展至 3D 圖形加速等功能。

AMD 表示今年底 Catalyst 的 Linux 軟體套件就會加入 AIGLX 的支援。AMD 私有繪圖驅動程式的開發會持續進行，下一個版本會大幅提升執行效能，並擴大對 HD 2000 晶片的支援。10 月預計的驅動程式更新會加入 3D 桌面繪圖技術 AIGLX 的支援，並在第四季推出專業工作站的版本。
