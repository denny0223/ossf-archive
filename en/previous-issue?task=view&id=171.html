具有自由分享精神的軟體專案開發工具
涂敏怡 2007/01/29

當各位在享受著自由軟體的好處時，是否曾想過自由軟體是如何進行開發的？身處在世界各地的軟體開發人 員，彼此又是如何合作互動，一起開發出具創意、高品質的自由軟體？自由軟體的開發承繼著電腦軟體剛開始發展時的自由、分享精神，發展至今已經有眾多的協同 開發工具與系統，為了讓各位能了解這些開發工具的發展過程，本期先從歷史角度來探討自由軟體的精神與自由軟體協同開發的關係、軟體協同開發的歷史、各種自 由軟體協同開發網站的背景和比較、以及由國人開發出來的 OpenFoundry.org 系統與其他鑄造場（如 SourceForge.net）的差異，從下一期開始我們將以長期專欄的方式進一步介紹如何利用 OpenFoundry.org 進行專案開發，敬請期待。

OpenFoundry 電子報主編 涂敏怡(tumi) 敬上

Openfoundry.org 與其他鑄造場的差異
jihuang

設立一個鑄造場園地，開放給自由軟體社群朋友進駐且經營軟體專案，這樣的想法早有許多個人或組織的先知 先賢嘗試過，並不是 OpenFoundry.org 首創。因此，OpenFoundry.org 和其他同樣性質鑄造場的差異為何，是許多內行人詢問執行團隊的第一個問題。

開放流程的開發方法論—Extreme Programming
謝良奇 2003/11/30
簡稱為 "XP" 的 Extreme Programming，和微軟的 Windows XP 可沒有關係。Extreme Programming 是一套輕量級的新開發程序，相對於過去歷史悠久的軟體開發方法，Extreme Programming 更加重視顧客在軟體發展程序中所扮演的角色。Extreme Programming 比起過去的軟體開發方法論，顯得更為簡單而有力，同時，要求軟體發展流程更加地開放、透明。

開放源碼專案管理軟體
謝良奇 2003/12/03
軟體開發可以說是極具複雜度的管理工作，許多的開發方法論、軟體工具相應誕生以解決軟體專案管理上遭遇的問題。適當的專案管理工具，有助於開發團隊彼此溝通、協調，執行所謂的協同開發程序。透過專案管理工具，軟體開發人員、專案經理，甚至是客戶，皆能隨時追蹤專案的開發進度以及目前的狀態。目前在開放源碼領域中，已有多套功能強大的專案管理工具問世。
