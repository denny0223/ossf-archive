微軟產品納入開放源碼程式碼 加強 Linux 系統管理能力

Created at Tuesday, 29 April 2008 08:00      Written by 謝良奇/編譯
看到微軟在年度微軟管理大會 (Microsoft Management Summit) 宣佈，旗下系統管理軟體整合了來自開放源碼的程式碼，令人不禁疑問，因應 Linux 與開放源碼的策略，微軟是否已經有了新的轉向。

微軟的管理軟體一直以來落後在 IBM、HP、BMC 與 CA 四大 IT 系統管理廠商之後，為了一改此頹勢，開放源碼蘊含的動能，看來被微軟視為足以刺激其 System Center 管理產品的進一步發展。 微軟宣佈將部份開放源碼 OpenPegasus 專案的程式碼，整合到 System Center Operations Manager 中，藉此擴展其管理軟體對 Linux 與 Unix 環境的管理能力。此一跨作業系統管理能力，為 System Center 用戶首要要求功能，顯示微軟體認到如今 IT 環境並非單一 Windows 環境。

對於微軟而言，既然無法擊敗 Linux，提供 IT 人員可供管理這類伺服器的微軟軟體產品，看起來會是較為有效的策略。即使此一市場，微軟已經算是較晚的進入者，面對 HP Openview、Tivoli、BMC 等廠商提供的相似產品，易於使用以及價格仍然可以為微軟帶來競爭優勢。

此外，開放同樣是一個有效且有用的方式。微軟新的資料中心管理軟體支援像 Web Services for Management 與 OpenPegasus 這樣的開放標準。這些動作有助微軟儘早成為舉足輕重的企業管理廠商。

Clear Channel 解決方案架構師 Curt Smith 正是 System Center 的用戶，Smith 表示不論進入何種企業，都不再是單一平台。Clear Channel 使用 Linux 管理廣播網路平台，也有一群使用 Apple OS X 加上 VMware 虛擬化產品的創意工作者。Smith 說，他需要從單一端點管理所有平台的功能，而不是只會搞得一團亂的一大堆工具。

微軟此次宣佈的新產品為 System Center Operations Manager 2007 Cross Platform Extensions，為 HP-UX、Red Hat Enterprise Linux、Novell SUSE Linux Enterprise Server 與 Sun Solaris 立即提供服務管理支援功能。

微軟伺服器與工具商務資深副總裁 Bob Muglia 表示，他們在 Windows 管理上有相當強勁的優勢，但用戶要的不只如此。對於微軟而言，從單純的 Windows 為焦點的管理公司踏出成為企業為焦點的管理公司，的確是相當重要的一步。

過去不乏管理 Linux 環境的第三方軟體，由於需要支付額外成本，加上並非來自微軟，所以無法與 System Center 達成深度整合，每次產品升級，整合性軟體必須連帶更新。如今藉由微軟提供的整合軟體，軟體公司可為 Unix、Linux 軟體提供更為深度的管理功能。

例如 Novell、Quest、Xandros 都宣佈各自的附加軟體，將使用 Cross Platform Extensions 提供微軟軟體所不及的 SUSE Linux Enterprise Server 深度管理功能。Novell 並且宣佈與微軟共同合作開發先進 Linux 管理解決方案，以簡化複雜 IT 環境管理。

在微軟與 Novell 雙方現有技術合作協議下，兩家公司將在產業標準與開放源碼元件上合作，進行管理功能的標準化。Novell 系統與資源管理資深副總裁兼總經理 Joe Wagnet 表示，Novell 過去在其系統與資源管理藍圖上已經提到，將在其 ZENworks 解決方案中，支援開放源碼與開放標準使用，以簡化與標準化系統管理。

Cross Platform Extensions 將免費包含在 System Center Operations Manager 2007 下一個  更新修補 (service pack) 中，微軟也預期將跨平台支援拓展到其他 System Center 產品，如 System Center Configuration Manager。

Muglia 指出很明顯他們在 Linux、Unix 上的專業不如 Windows 平台。這是微軟運用開放源碼專案管理 Linux 平台，而非自行開發的原因。

微軟不僅取用 OpenPegasus 的程式碼，還將與 IBM、HP 等公司一樣，加入 OpenPegasus 籌劃委員會 (OpenPegasus Steering Committee)，並以 OSI 認可的微軟公眾授權 (Microsoft Public License)，將程式碼回饋給專案。在此授權下，微軟回饋的程式碼在不更動智財權宣告的前提下，將可供外界自由修改與使用。
