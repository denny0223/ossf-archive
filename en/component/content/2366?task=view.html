Fedora 13 新增開放 3D 驅動程式和改善 Python 環境

Created at Thursday, 27 May 2010 08:00      Written by 謝良奇/編譯
知名 Fedora Linux 散佈套件的開發者日前宣佈，正式釋出代號 Goddard 的 Fedora 第 13 版，其中加入許多重要平台改進和若干新的桌面應用程式。不久前釋出的 Ubuntu 10.04 夾帶其音樂商店和雲端同步服務，成為許多人心中 Linux 散佈套件的優先選擇。然而，Fedora 專案領導人 Paul Frields 不認為如此，他說，Fedora 的目標，是成為志於貢獻開放源碼的使用者的第一選擇。他們想要的是貢獻的文化，而不僅止於消費。 Fedora 13 之中反映出對於懂技術的貢獻型用戶的注重，其中納入了開放源碼的 3D 影像驅動程式、絕佳的 Python 工具、優秀的色彩管理工具，以及其他不是那麼顯眼的改進。

為了在 Linux 桌面上獲得硬體加速的 3D 圖形，使用者一般必須仰賴顯示卡廠商提供的私有驅動程式。Linux 社群在建構替代性的開放源碼驅動程式上面臨了不小的困難，不過這些投入已經開始有所收穫。Fedora 13 針對 Nvidia 顯示卡提供了開放源碼的 3D 加速。除了吸引自由軟體支持者外，這也意味著重新包裝和散佈 Fedora 不再需要擔心私有驅動程式的問題。

這些驅動程式是更長遠計畫的其中一部分。Frields 表示，Fedora 早已有了支援 3D 的開放源碼 Radeon、Intel 驅動程式，這 3 套開放的驅動程式正在為更完整的 3D 堆疊做好準備。特別是隨著  GNOME 3 的即將到來，由於 GNOME 3 在新使用者界面上依賴影像合成 (compositing)，3D 對於使用者將更加重要。

另一項重要的改變是虛擬化技術。紅帽 (Red Hat) 已經選定 KVM 才是 Linux 虛擬化技術的未來。最近的 RHEL 6 beta 版揚棄 Xen 而改採 KVM，如今 Fedora 也不例外。KVM 的新功能包括為 guest 虛擬機器保留 PCI 位址，並提供共享的實體網路界面給虛擬機器，讓 guest 在和 host 相同的網路上獨立存在。儘管這些新功能無關乎桌面用戶，但是對於運行多重虛擬機器的伺服器來說，卻讓 Fedora 成為更引人注目的選擇。

Fedora 13 可平行安裝的 Python 3 環境，可以讓 Python 的開發者在 Python 2.6 之外，安裝 Python 3 堆疊，值得一提的是，此一方案不只是隔離執行時期環境與標準程式庫而已，也包括了許多常用的第三方模組，大大簡化了移植現有 Python 應用程式到新版程式語言的程序。

Fedora 13 搭配了最新穩定版本的 GNOME 2.30，此一新版桌面環境增加了 Nautilus 檔案管理程式的分割面板 (split-pane) 模式，Empathy 即時通訊軟體提供 Facebook 對話支援，並改善了 iPod 相容性。對於 KDE 的愛用者，Fedora 13 也納入了 KDE 4.4，提供一系列 PulseAudio 改進以及 Dolphin 檔案管理的新時間序列模式等。

當然，Fedora 13 並不只有技術上的改變。事實上，Fedora 13 可能是 Fedora 最具用戶親和性的釋出版本。Fedora 13 納入了社群網路應用軟體 Pino，可以連接至 Twitter 和 Identi.ca。Pino 會在選單列顯示狀態，提供有如 Ubuntu 搭載的 gwibber 的效果。

有賴 Fedora 貢獻者 Richard Hughes，Fedora 13 這次在 GNOME 桌面上增加了全新的色彩管理系統。新的配置管理器 (profile manager) 只要透過雙擊滑鼠鍵，即可輕鬆安裝廠商提供的 ICC 或 ICM 配置檔案。儘管對於一般用戶色彩管理並不重要，但對繪圖專業人士與影像愛好者來說，這可是一項藝術，而且也是　Apple 的 OS X 之所以普及的優勢之一。

Fedora 13 納入了新的相片管理工具 Shotwell，礙於空間限制，功能強大的 Fspot 並未納入 LiveCD。Shotwell 提供了輕量級的相片管理應用軟體。有更多需求的使用者，也能透過 Fedora 的軟體倉儲安裝 Fspot。

Fedora 13 納入支援 HTML5 video 的 Firefox 3.6，只要該影片是用 Ogg Theora 影片 codec 編碼即可。Fedora 也率先搭載了新的 Theora 1.1，這個開放影片 codec 的改進版本提供可與私有解決方案 H.264 匹敵的影片品質。
