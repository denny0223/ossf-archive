MySQL 社群成立拯救 MySQL 請願網站

Created at Tuesday, 12 January 2010 08:00      Last Updated on Friday, 29 January 2010 18:17 Written by 謝良奇/編譯
MySQL 創辦人 Michael “Monty” Widenius 針對 Oracle 欲購併 Sun 一事，積極發動與論活動加以反對後，不僅獲得社群廣大支持，社群更隨之成立新的 MySQL 支持活動 HelpMySQL.org 網站。這個成立於去年 12 月 28 日的網站，以拯救 MySQL (Save MySQL) 為標題，很快就吸引了眾多開發者、程式設計者與使用者的關注。

當 Oracle 對 Sun 的購併案，在缺少任何特殊條件強迫 Oracle 維持 MySQL 發展下，通過美國政府審核之際，錯愕的 MySQL 開發者社群轉而將最後希望放在歐盟委員會 (European Committee，EC) 的審議程序。其間 Oracle 試圖加速 EC 審議，召集該公司部份用戶發函要求 EC 以無條件方式通過此筆交易。

此舉很快引來 MySQL 創辦人與社群的強烈反彈，數以百計開發者、小型企業或非營利機構等，也去信 EC 要求針對交易中 MySQL 的部份增加特別條款。如今 MySQL 社群更進一步成立新網站以挽救該開放源碼資料庫的存續。

HelpMySQL.org 成立後不久，該網站的網路流量排名急速上升，根據 Alexa 的資料，日前已衝上約第 88,000 名。該網站的成立目的，是透過法律與契約方式，確保 MySQL 以開放源碼專案方式延續。

網站上提供一份請願書，使用者可以從 3 種選項中，任選單一選擇或多種組合以提案給 EC。第 1 個選項是，MySQL 必須出讓給合適的第三方，持續以 GPL 方式進行開發。第二選項則是，針對透過所有程式語言的用戶端程式庫使用 MySQL 的應用軟體、plugins 與 libmysqld，Oracle 必須承諾給予連接例外 (linking exception)，而 MySQL 本身必須維持 GPL 授權。第 3 則是，Oracle 必須將過去以及未來屆至 2012 年 12 月的所有 MySQL 版本，以 Apache Software License 2.0 或類似的許可式授權 (permissive license) 釋出，讓應用軟體與衍生 (分支) 版本開發者擁有程式碼彈性。

該網站提供 20 種語言，至今已經獲得超過 27,850 位使用者的簽署。從線上即時統計數據看來，多數意見希望讓 MySQL 維持現有 GPL 專案狀態，也就是第 1 個選項。

儘管 Oracle 之前透過新聞稿發表了一系列有關 MySQL 未來狀態與發展的承諾。許多開發者，尤其是 Monty，並不滿意 Oracle 的承諾方式。其中主要是這些承諾只在新聞稿中，對於購併案並無法律約束力。他認為 Oracle 無疑會將 MySQL Enterprise 的開發與 MySQL Community 拆分開來，同時無視來自社群的程式碼開發，將所有資金挹注在 Enterprise 版本。

不過幾位 MySQL 前高層人士倒是持有不同的看法。例如 MySQL 共同創辦人 David Axmark 雖然表示 Oracle 沒有理由支持該開放源碼資料庫，但也不認為 Oracle 會讓 MySQL 停擺。同時 Axmark 提出一個 Oracle 與 MySQL 可能並存的觀點。他認為，將 MySQL 與 Oracle 現有資料庫應用軟體的市場加以區隔，可以讓兩者並存。Axmark 說，假如他是 Oracle，他會將 MySQL 擺在 Oracle 毫無所獲的 web 市場，如此一來不會傷害到 Oracle 所依賴的企業市場獲利。

另一位高層則是 MySQL 前執行長 Marten Mickos。Marten 與 Monty 日前都曾經發表給歐盟的公開信，只不過兩人的意見相左，Marten 支持 Oracle 取得 MySQL，而 Monty 則是力阻 Oracle 掌控 MySQL。

在不同意見之間，不少人或許疑惑，MySQL 的 GPL 授權不是最好的保護傘，可以確保其他人成立 MySQL 分支嗎？正如 Marten 的公開信寫道，當用戶注意到 MySQL 開發趨緩或訂閱價格高漲之際，用戶可以轉而尋求分支版本以及提供 MySQL 服務的商業公司。

一份由 Monty 的新公司 Monty Program Ab 提供給 EC，由 MySQL 與歐盟事務專家 Florian Mueller 所執筆的意見書 (position paper) 寫道，當擁有 MySQL 的新公司停止 MySQL 的發展，任何一家商業公司都非常難以接替開發。最重要的一點在於，MySQL 相關營收大多依靠雙重授權與額外付費用戶。而在目前的基礎上，分支廠商會發現他們無從自這些營收來源中獲利。對於不願接觸或使用 GPL 程式碼的獨立軟體廠商或企業，分支廠商無法提供非 GPL 版本的 MySQL。

Muller 與 Monty 提出的問題，足夠歐盟和開放源碼社群重新思考，”任何人皆可分支程式碼” 是否足以成為 Oracle 照管 MySQL 程式碼、公司與社群的理由。
