紅帽宣佈不會開發消費市場 Linux 桌面產品

Created at Tuesday, 22 April 2008 08:00      Written by 謝良奇/編譯
期待看到 Linux 散佈套件商紅帽 (Red Hat) 在消費市場釋出桌面 Linux 作業系統的用戶，會因為這則消息感到失望。紅帽桌面團隊在部落格上宣佈，在可預見的未來，他們沒有計畫為消費市場開發傳統桌面產品。

以紅帽桌面系統最新消息 (What's Going On With Red Hat Desktop Systems? An Update) 為題，這篇文章寫到，紅帽作為公開的營利企業，在開發的產品與技術上必須以獲利為前提，桌面有著比伺服器更為艱難的處境。桌面市場因為受到單一廠商獨佔之故，許多人到今天仍認為 Linux 桌面並不足以作為替代方案。此一說詞多少令外界聯想到微軟。 不過文章中也表示，有越來越多的技術取向用戶及企業了解到 Linux 桌面已經足以作為實用替代方案。儘管如此，在 Linux 桌面上的生意維持不易，一直以來多的是失敗的例子，不是終止就是變成慈善事業般經營。好消息是，過去一兩年的技術發展的確有助加速 Linux 桌面的普及。

紅帽表示，將持續將重心放在與其 Linux 伺服器作業系統相容的 Red Hat Enterprise Linux 桌面版本、開放源碼且受社群支援的免費 Fedora Linux 作業系統，以及去年對外宣佈的 紅帽全球桌面 (Red Hat Global Desktop，RHGD) 計畫。

RHGD 計畫因商業原因受到延遲，仍處於開發階段，但紅帽希望能在數月內完成 RHGD。RHGD 是專為巴西、俄國、印度與中國等新興市場，所設計的產品版本，將交由 Intel 通路夥伴以小型零售商方式加以部署。

對某些產業分析師而言，紅帽的決定並不意外。Illuminata 分析師 Gordon Haff 認為消費者桌面這個字眼是個關鍵。紅帽本來就有 Red Hat Enterprise Linux Desktop，所以對於企業來說，紅帽已經有桌面產品。紅帽也有 Fedora，對於不想付費購買作業系統的用戶，Fedora 可以免費供其取得使用，必要時還可以安裝額外的影音播放器。

他還認為消費者版本需要包含像這類播放器，也是紅帽之所以不想碰這個市場的理由。紅帽一向走純開放源馬路線，消費者桌面版本必須納入部份私有編碼器 (codecs) 等軟體，與紅帽的原則背道而馳。

部份分析師進一步支持紅帽的決定，認為紅帽並非消費市場企業，由於 Windows 通常隨硬體一同出貨，紅帽在吸引用戶付費取得 Linux 桌面系統上將遭遇難題。ZDNet.com 部落客 Dana Blankenhorn 認為，紅帽一直以來都在組織市場、企業市場、伺服器市場，在桌面上的動作基本上並不受重視。

對於 Blankenhorn 而言，比起微軟的獨佔，Ubuntu 這個 Linux 消費桌面市場的主要對手，才是紅帽下此決定的主因。 Blankenhorn 表示 Linux 並非 Windows，並非一間公司而是代表一整個產業生態系。每個參與者都可以選擇自己最擅長的部份。任何一個稍微對開放源碼界有點了解的人，都知道 Ubuntu 在桌面的投注程度，而且也佔有該 Linux 市場區隔的領導地位。

此外，Haff 表示，談到錢與營收問題，究竟有多少用戶會付費在家中電腦安裝 Linux 桌面？他認為很少。

另一家 Linux 散佈套件商 Novell 也有相似的策略，專注在企業 Linux 桌面。Novell 總裁兼執行長 Ronald Hovsepian 日前才表示過，Suse Linux 在消費桌面市場上，仍需要相當長的時間來開發，未來 3 到 5 年的桌面市場主要仍是企業為主。

紅帽企業行銷副總裁 Michael Chen 表示，紅帽把焦點置於企業市場的基礎架構軟體上，對此，紅帽提供了 Red Hat Enterprise Linux Desktop。至於消費者桌面，需要不同的支援體系與應用軟體。紅帽在 2008 與 2009 的桌面目標，將確保桌面產品補足其伺服器與中介軟體產品的不足。
