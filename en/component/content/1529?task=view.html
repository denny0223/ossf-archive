昇陽推出 MySQL 5.1 改善執行效能融合多項功能

Created at Tuesday, 15 April 2008 08:00      Written by 謝良奇/編譯
昇陽 (Sun) 公開了全球最受歡迎開放源碼資料庫的最新版本 MySQL 5.1。昇陽在第六屆 MySQL 大會 (MySQL Conference & Expo) 上釋出 MySQL 5.1 接近定案 (near final) 版本，改善了執行效能並簡化大型資料庫應用軟體的管理工作。

在聚集近 2000 位與會者，全球最大開放源碼資料庫社群場合上，昇陽宣佈 MySQL 5.1 最終版本將於本季公開問世，但並未給定明確的推出日期。 二月時昇陽購併 MySQL 所取得的此一開放源碼資料庫，在新版中強調更先進的分區 (partitioning) 與副本 (replication) 功能，新增了事件排程器與提供企業用戶運用的資料庫升級諮詢軟體。

昇陽資料庫事業群資深副總裁 Marten Mickos 表示，一開始因為低成本選擇 MySQL 的用戶，將會因為 MySQL 的高效能、延展性與廣泛的平台支援，持續使用 MySQL。MySQL 5.1 在內部測試中，展現高於前版本多達 15% 的效能提升，使其成為網路企業應用軟體的最適解決方案。

Spinn3r.com 創辦人兼任執行長 Kevin Burton 指出，他們將 MySQL 5.1 運用在部落格搜索資料庫上，每天索引約 1.8 千萬個 URL。5.1 版的新功能將 MySQL 著名的易用性、延展性與效能進一步延伸，持續滿足了今日資料庫應用軟體的需求。

新版本將以更快速度回應資料查詢需求。這多虧更強大的分區功能。資料表 (Table) 與索引 (Index) 分區支援了 5 種形式的水平資料分區，包括範圍 (range)、雜湊 (hash)、關鍵值 (key)、串列 (list) 與複合 (composite)。經由分區技術，僅搜尋相關分區的資料而非整個資料表與索引，大大縮短查詢回應時間。

MySQL 5.1 擁有資料列為基礎 (Row-based) 與混合式 (Hybrid) 的副本功能。以資料列為基礎的副本功能會複製主伺服器與從伺服器間的變動資料，而非實際的 SQL 敘述。混合式副本則依據個別 SQL 作業的不同，選擇使用以敘述為基礎或以資料列為基礎的副本功能，使得應用軟體享有最佳效率及安全性的副本執行策略。

新版 MySQL 加入事件排程器，允許開發者與資料庫管理者將經常重複上演且基於 SQL 的工作加以自動排程在資料庫伺服器上執行。新功能針對資料庫特定工作提供了比 Linux 等系統 CRON 或 Windows 工作排程器更好的支援。

對於 MySQL 企業版訂戶而言，新的升級諮詢軟體 Upgrade Advisor 提供了自動化規則，用來幫資料庫管理者監管部署 MySQL 5.1 可能發生的特定錯誤並提供建議。一旦偵測到可能發生問題，軟體會提供建議給管理者，以便修正升級中的問題。

一如既往，MySQL 5.1 會以三種不同版本出現。免費提供的社群伺服器採用 GPL 授權，商業版本伺服器針對企業用戶提供了訂閱支援，此外還有可供 OEM 廠商整併到硬體的嵌入式版本。

MySQL 5.1 預定於本季中釋出，針對多種軟硬體平台提供支援，包括 Red Hat Enterprise Linux、SuSE Enterprise Linux Server、微軟 Windows、Solaris 10、Mac OS X、FreeBSD、HP-UX、IBM AIX、IBM i5/OS 與其他 Linux 散佈套件。

Amazon 與 Google 皆在服務中運用 MySQL，昇陽執行長 Jonathan Schwartz 表示該公司將把此資料庫推向大型用戶。他並且承諾將為此產品增加延展性及後端應用軟體的支援，以便與 IBM、甲骨文 (Oracle) 及微軟競爭。
