Linux 與 iPhone 上將可見到 VMware 管理軟體

Created at Tuesday, 16 September 2008 08:00      Written by 謝良奇/編譯

VMware 技術長 Stephen Herrod 在 VMworld 大會 (VMworld conference) 宣佈，該公司下一版的 VirtualCenter 管理軟體，將出現在 Linux 與 iPhone 上，立刻贏得在場聽眾熱烈掌聲。

VMworld 大會日前在拉斯維加斯舉辦，會議主題演說中，Herrod 也說明了該公司對 VMware 核心虛擬機器技術的改進，企業將可藉此在虛擬伺服器上，執行需求更為嚴峻的應用軟體。

VirtualCenter Management Server 是 VirtualCenter 的控制節點，目前僅可在微軟的 Windows Server 作業系統上執行。預計明年推出的新版本，重新命名為 VCenter，根據 Herrod 的說法，將在 Linux 上運作，並提供虛擬應用裝置 (virtual appliance)，是一款預先設定所有必要軟體的虛擬設備。

而在 Windows 個人電腦上執行的 VirtualCenter 用戶端，該公司也正在移植到 Linux、Mac OS 與 Apple 的 iPhone 等設備上。儘管 Herrod 只展示了該軟體 iPhone 介面的照片，此一訊息仍舊贏得在場人士的掌聲。

綜觀以上消息，必須倚賴微軟 Windows 運行 VMware 伺服器管理軟體的情況，往後將有所改變。

大會上 VMware 不斷強調其應用軟體效能與可用性議題。Herrod 表示，VMware 的焦點在於不論執行的是何種應用軟體，都能確保滿足應用軟體對效能的要求。VMware 明年將提升其虛擬機器所能支應的計算能力，從今日的 2 顆 CPU 與 4M bytes RAM，到 4 顆 CPU 與 64G bytes RAM。I/O 處理能力 (throughput) 也由 300K bps 提高至每秒 9G bytes。在一個虛擬資源中心叢集 (virtual resource pool cluster) 中，IT 人員最多可加入 64 個伺服器節點。

Herrod 也談到了 VMware 預計在明年推出虛擬資料中心作業系統。這是供資料中心集合所有資源的一系列技術，包括儲存技術、網路，以及讓虛擬機器與所附帶政策更輕易移動的技術。

演說中，Herrod 介紹了 VirtualCenter 的附加元件 VMware Fault Tolerance。VMware Fault Tolerance 運用了 VMware 所謂的 vLockstep 技術，能不斷地在不同的實體伺服器上，為虛擬機器產生更新副本。當主要系統發生故障，VirtualCenter 將自動切換到副本上。根據 Herrod 表示，使用者根本不會發現此一切換。Herrod 並以此技術執行一個吃角子老虎應用軟體做為展示。

由於微軟數週前宣佈免費釋出其 hypervisor 產品，對照 VMware 在 VMworld 大會上的種種積極作為，不禁令人聯想，微軟是否成為虛擬化技術領導廠商 VMware 的競爭威脅。畢竟該軟體巨人免費提供 Hyper-V hypervisor 產品給 Windows 2003 或 2008 伺服器版本購買用戶，終端使用者很難不注意到這樣的訊息。

對於資料中心 Digital Sense 技術長 Michael Tran 而言，微軟進入此一市場有許多好處。他認為雖然微軟的產品或許在許多方面不及 VMware，但擁有小型企業需要的低成本與強大功能。

此外，儘管 VMware 過去公開否認其私有的 vmkernel 技術違反了 GPL 授權。然而該公司卻在最近的季報中指出，該公司在產品中廣為使用開放源碼軟體。VMware 並且宣稱考慮將該公司的 ESX hypervisor 開放源碼。顯然是 VMWare 在產品中載入了 Linux 的模組，以提供虛擬化的功能。

究竟 VMware 是否真的違反 GPL，開放源碼社群的意見相左。部份人士認為 VMware 在其 ESX 虛擬化技術中，發展了 Linux 的衍生作品，卻未曾將其貢獻出來。然而也有人援引源自 Linus Torvalds 的可載入模組例外 (loadable module exception)，認為 VMware 違反 GPL 與否，處於灰色的模糊地帶。

無論如何，如今的重點是 VMware 必須承擔此一風險，更值得納入考慮的是， VMware 或許沒有違反 GPL，但可能已經觸犯社群常情。面對來自開放源碼虛擬化產品漸增的競爭，VMWare 會不會在沒有法律壓力下，仍然做出開放源碼的決定，有待後續觀察。
