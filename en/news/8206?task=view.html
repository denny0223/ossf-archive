提升平台即服務實力 紅帽購併雲端軟體商 Makara

Created at Tuesday, 07 December 2010 08:00      Last Updated on Monday, 13 December 2010 19:37 Written by 謝良奇／編譯
為提升在平台即服務 (platform-as-a-service，PaaS) 上的實力，企業開放源碼軟體商紅帽 (Red Hat) 購併了雲端應用軟體部署與管理平台公司 Makara。交易細節並未公開。

紅帽計畫在其 Cloud Foundations 系列產品中運用 Makara 的技術。紅帽產品與技術總裁 Paul Cormier 表示，Cloud Foundations 的目的是為了讓客戶和開發者能輕鬆進入雲端。透過購併 Makara，他們希望進一步簡化應用軟體部署和管理。

具體而言，Makara 的技術將成為紅帽 PaaS 系列產品中的一部份，在結合紅帽 JBoss Enterprise Middleware 之後，Makara 平台能夠提供額外的監控、回復和延展工具。

451 Group 研究主任 Rachel Chalmers 表示，對於希望在雲端提供優異企業解決方案的軟體廠商，PaaS 逐漸成為他們的另一個市場。她認為，藉由取得 Makara 開發的雲端技術，紅帽如今足以透過創造解決方案給尋求在核心平台中納入部署、管理、自動延展能力的企業，進而跨入此一市場。

Paul 指出，紅帽在雲端運算成為顯學之前，早已提供了可延展及具彈性的雲端基礎架構。藉著購併 Makara，該公司將能在雲端提供可移植性和互通性。

藉由這項購併，紅帽已擁有了加強其 PaaS 願景所需的元件。Makara提供了基於 Amazon EC2 的依需服務 (on-demand service)。紅帽副總裁兼雲端事業部總經理 Scott Crenshaw 認為，更重要的是，紅帽如今為開發新應用程式並將現有應用程式移往雲端的客戶，消除了複雜度並提供平等的立足點。

Makara 執行長兼共同創辦人 Isaac Roth 表示，透過 Makara 平台，組織可以提供、部署、管理、監控並延展 Java 和 PHP 的應用軟體，至任何公開、私有或混合雲端。該管理軟體可轉移現有應用軟體，彙整來自所有虛擬機器實體的日誌到單一永續日誌上，並在需要時自動延展應用軟體和控制所使用的資源。

Roth 表示，儘管 Makara 主攻 JBoss、PHP 應用軟體，該平台也能支援標準 Java EE、Spring、Tomcast 、LAMP。

Crenshaw 指出，紅帽將把 Makara 的技術納入該公司的 PaaS 產品中。取代個別的 Makara 產品，Cloud Application Platform 將整合 JBoss Enterprise Middleware 基礎架構，即將推出的 Cloud Admin Portal、Cloud User Portal、PAAS Automation Engine、Application Configuration Engine 也將納入 Makara 的技術。他表示，PaaS 是紅帽今年稍早公開的 Cloud Foundations 策略的基礎元件。

Crenshaw 表示，紅帽的主要目標是提供可避免供應商綁定 (vendor lock-in) 的 PaaS。因為 Makara 可允許組織在任何虛擬化基礎架構上實作該平台，不論該架構是像 Amazon EC2 的公開雲端，或是基於 Red Hat Enterprise Virtualization、Citrix Xen、VMware 的私有雲。
