Mozilla 釋出 MPL 2.0 版

Created at Monday, 16 January 2012 14:47      Last Updated on Monday, 16 January 2012 16:01 Written by 謝良奇／編譯

Mozilla 日前釋出了 Mozilla Public License (MPL) 的第二版。此授權協議距離上一次更新已超過 10 年時間，修改重點針對其他自由軟體與開放源碼授權提供更佳的相容性，可望為 Mozilla 專案擴展更廣泛的程式碼重復利用的選擇。

新版 MPL 歷經 21 個月的公開諮詢。其基礎是 1998 年制定的 MPL 1.0 版，該版本僅於 1999 年翻修過一次。MPL 2.0 和 MPL 1.0 相同，都是檔案層次的公共版權 (copyleft) 授權，此一授權方案允許用戶在建立的專案中，混合使用採取 MPL 授權以及其他開放或私有授權的程式碼。

Mozilla 描述 MPL 時認為，該授權填補了自由與開放源碼軟體授權頻譜中的重要位置，介於不要求分享改動的 Apache 授權，以及要求改動必須加以分享的 GNU 系列授權之間。來自自由軟體基金會 (Free Software Foundation) 的 GNU General Public License 是廣為採用的自由軟體授權。

Mozilla 基金會 (Mozilla Foundation) 主席 Mitchell Baker 在部落格中談到，MPL 2.0 在精神上與前一版本相似，但更為精簡、優越，更相容於其他開放源碼與自由軟體授權。

Mozilla 表示，新版授權主要的改變在於 MPL 條款的簡化。例如條款中不再需要指定原始程式碼的可取得時限，而是要求可執行檔釋出時，都必須提供程式碼。

此外，授權標頭 (license headers) 更為簡短，通知要求 (notification requirements) 也簡化許多，Mozilla 表示授權內容也經過大幅縮減，應該更易於理解。

Firefox 瀏覽器、電子郵件用戶軟體 Thunderbird 上採用 MPL 的 Mozilla 專案，承諾將移往 MPL 2.0 版。儘管 Mozilla 將修改其程式碼基礎以使用新的授權，但並無確切時間表。

MPL 2.0 與之前的 MPL 1.1 一樣，已經成為自由軟體基金會與開放源碼組織 (Open Source Initiative) 所認可的自由軟體與開放源碼授權。

另外，此次更新版本的 MPL 也和其他授權一樣，為貢獻者提供了專利保護。Mozilla 基金會表示，該授權允許社群中的任意成員，保護受到侵權控訴的貢獻者。
