Oracle 和 IBM 宣佈於 OpenJDK 專案上協同合作

Created at Thursday, 14 October 2010 00:00      Last Updated on Tuesday, 26 October 2010 21:26 Written by 謝良奇／編譯
Oracle 和 IBM 日前宣佈兩家公司共同合作讓開發者與用戶，在現有 Java 投資與 OpenJDK 參考實作的基礎上進行開發和創新。更明確地說，Oracle 和 IBM 將在 OpenJDK 社群中協同合作以發展此一最主要的開放源碼 Java 環境。

在記者會上，IBM 和 Oracle 官方表示這次的合作將以 OpenJDK 專案和相關的 Java 開發套件(Java Development Kit，JDK )與 Java 執行時期環境(Java Runtime Environment，JRE )為中心。在此同時，Java Community Process (JCP) 將持續作為 Java 規格工作的主要標準組織，兩家公司也將持續加強 JCP。

Oracle 的 Oracle Fusion Middleware 和 Java 資深副總裁 Hasan Rizvi 表示，對於 IBM 和 Oracle，這是透過 OpenJDK 社群進行合作以加速 Java 發展的重要一步。

Oracle 和 IBM 將支持不久前公開的 OpenJDK 發展佈局，提升  Java 標準版本(Standard Edition，Java SE )在開放源碼社群的可用性。Rizvi 指出，Java 社群是 Java 平台演進不可或缺的一環，Oracle 與 IBM 之間的合作將建立在 OpenJDK 成功作為 Java SE 主要開發平台的基礎之上。

IBM 新興技術副總裁 Rod Smith 表示，IBM、Oracle 和其他在 OpenJDK 共同合作的 Java 社群成員將加速 Java 平台的創新。Oracle 與 IBM 的合作也是向企業用戶顯示，他們可以持續仰賴 Java 社群提出更為開放、更具彈性且創新的新技術，來協助他們的業務繼續成長。

Smith 指出，IBM 和 Oracle 在 OpenJDK 上的合作，已經開始掃除圍繞在 Java 未來周圍的不確定性。他們對 OpenJDK 的承諾有利於整個 Java 社群。

Eclipse 基金會 (Eclipse Foundation) 執行董事 Mike Milinkovich 表示，這項宣示由於整合了 Java 社群，對 Java 來說是件好事。有了 Oracle 和 IBM 直接參與該平台的發展，不僅有助加快創新步伐，Java 產業體系中的其他主要參與者是否因此跟進，更是值得關注。

此外，Milinkovich 認為此舉也有利於 Eclipse 社群。首先，一般來說對 Java 有利的對 Eclipse 也有幫助。擴展 Java 普及率和動力的此一宣示將有助於 Eclipse。其次，這兩家公司作為該專案主要貢獻者，是 Eclipse 最大的支持者。他們在 OpenJDK 上的合作增加，也預示了未來社群全面協同合作的可能。

Tasktop Technologies 執行長以及 Eclipse Mylyn 專案創始人 Mik Kersten 認為，伴隨在圍繞於今年 JavaOne 大會的失望氣氛之後，開發者之間經常討論起 Java 分支的話題。對於擔心分支可能帶來的分歧化，IBM 與 Oracle 的宣告可說是 OpenJDK 成為 Java 周遭廠商未來開放源碼合作的重要信號。

OpenJDK 是由 Sun 在 2006 年所成立，在許多方面，OpenJDK 可視為 Sun 遵守對於 Java 開放源碼的承諾。在此同時，也有其他專案，例如 Apache 的 Harmony，企圖建構自己的 Java SE 6 開放源碼版本。IBM 和多家公司以及開放源碼組織也都表達過對 Harmony 的支持。但是反觀 OpenJDK 也有來自紅帽 (Red Hat) 和 Canonical 的支持。問題是 IBM 為何改而支持 OpenJDK？

根據 Smith 的說法，IBM 將持續支持 Harmony，不過該公司的重心會放在 OpenJDK。在部落格中，IBM 的 Linux 與開放源碼副總裁 Bob Suitor 解釋，他們認為這是個務實的選擇。顯然過去的 Sun 和現在的 Oracle 都無意將 Java 的重要測試和認證測試 Java SE TCK，提供給 Apache。儘管 IBM 並不贊同此一決定。所以與其持續推動 Harmony 作為非官方且未經認證的 Java，他們決定調整方向投入 OpenJDK。

此外，Smith、Rizvi 和 Oracle Fusion Middleware 副總裁 Adam Messinger 表示，他們預期 JCP 的變化和改進將有利於開發者。Milinkovich 表示，Oracle 和 IBM 部落格文章的評論中暗指近期將發佈有關 JCP 管理和 Java7 的 Java 規格提案 (Java Specification Requests, JSRs) 的消息，也引起他的關注。假如他們能讓 JCP 往正面方向推進，那麼 Java 產業體系將能夠真正地重拾其活力。
