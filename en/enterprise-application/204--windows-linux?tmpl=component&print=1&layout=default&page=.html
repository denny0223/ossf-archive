從 Windows 遷移到 Linux（上）

Created at Friday, 09 June 2006 08:00      Last Updated on Friday, 03 February 2012 14:40 Written by 陳貴成
前一陣子，某超級大軟體公司的廣告大概嚇壞了所有有在使用電腦的企業主，主要內容是希望所有的員工協助該軟體公司來抓盜版，獎勵員工提出道德勇氣來揭發老 闆在公司使用盜版軟體的罪行，主要目的當然是希望企業主可以乖乖的購買軟體版權在使用。果然此一廣告一出嚇壞一堆公司的老闆，尤其公司越大是越害怕，紛紛 的去補買了一定數量的軟體授權，先不說有沒有真的把軟體的授權數量買齊，但至少該超級大軟體公司是因為這樣收到不少應該要補的授權費用，想想這效果真是驚 人。 也因為這樣，許多的公司開始在尋找可以降低成本的替代方案時，自然將工作平台從 Windows 轉換成 Linux 變成主要的選擇之一了。

所以在下參考了國外關於 Windows 移轉到 Linux 的相關文章後[註]，再跟台灣的現況稍稍的整合了一下，提出來跟大家分享一下。

許多的公司都會因為下面的幾點考量而希望將現有的 Windows 環境改成是 Linux 的環境：

  1. Linux在普遍上來說仍然是一個較為穩定及可靠的作業系統
  2. 針對授權費用而言幾乎是免費的
  3. 可以取得軟體的程式碼進而進行客製化及修改的動作
  4. 有較佳的安全性
  5. 是依據開放性的標準進行系統的發展
即便有這樣多的好處，但在遷移前仍有許多的規劃要進行，因為會有許許多多不同的問題會產生，所以在決定遷移之前良好的規劃是非常重要的，還有一重點是必須要確定遷移是必需的且對組織是有益的，並不是意識型態在作遂，不然遷移計劃終將招至失敗

◎ Windows 遷移到 Linux 前的重要步驟

  1. 取得老闆的全力支持：
    若可以取得高層的支持，所有事情的進行將可以事半功倍。
  2. 確認公司的環境適合進行 Windows 移轉至 Linux 的計劃：
    一間公司的環境包含了許多不同的電腦使用者族群，所以在進行系統遷移的計劃之前，必須要確認並且評估過公司目前是適合進行系統遷移的環境。
  3. 全盤的檢查及了解現有環境：
    針對要進行遷移環境進行所有需要的檢查及了解，包含了硬體的支援性。
  4. 選擇最可以接受的 Linux 套件：
    目前有許多受歡迎的 Linux 套件可供選擇，例如商業支援最為豐富的 RedHat、 SUSE，或是社群支援較為豐富的 Debian，都可以依據單位特性及能力的不同進行選擇。在台灣當然因為是繁體中文的環境居多所以中文化的程度也變成是考量的項目之一。
  5. 確認那些人適合先進行移轉的動作：
    一間公司裏面可以先針對每種不同的職務需求來進行評估，看看那一個單位比較適合運用 Linux 桌面進行日常的工作，並由他們開始做為 Linux 桌面的使用者。
  6. 充份的教育訓練：
    若沒有充份的教育訓練，所有的遷移計劃成功的機會將會微乎其微。教育訓練不僅僅是電腦技能的教育訓練，員工心理層面的溝通也是非常重要的，因為改變的會是他們一天接觸八小時以上的環境。
  7. 評估外聘顧問的可行性：
在組織/公司本身的技術及專業並不足以滿足遷移需求時，可以考慮向外進行顧問的聘用，顧問的計價方式可以以時薪、月薪、或是專案的方式聘用。這邊可以強調的是專業的指導可以減少許多不必要的資源的浪費，例如：時間及人力。
