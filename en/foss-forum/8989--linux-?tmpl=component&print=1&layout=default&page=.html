企業開發者與系統管理者的最佳 Linux 工具

Created at Tuesday, 28 May 2013 11:01      Last Updated on Tuesday, 28 May 2013 15:07 Written by 謝良奇／編譯

從最佳軟硬體清單上，處處可見 Linux 的蹤跡，即可證明該自由與開放源碼作業系統的普遍性與泛用性。例如，光是 Linux.com 過去幾個月內，就刊出過「2013 最佳 Linux 伺服器」，與「2013 最棒的 7 個 Linux 散佈套件」這些文章。現在應該是企業 IT 人士登場的時間了，就讓我們看看企業系統管理者與開發者的最佳 Linux 應用程式與工具。

在 Google+ 上對此問題進行非正式調查後，出現了一些入圍者，包括 Qt Creator、KVM、Puppet、Red Hat Satellite、Nagios、Archipel、Terminator、Samba、Cobbler、Git/Subversion、Perl、mrepo、Python、SaltStack、Wireshark、Clonezilla。RedMonk 共同創辦人與首席分析師 Stephen O'Grady 表示，這些選項並不出乎意料，其中可以合理地劃分成三個領域，Hudson/Jenkins 屬於持續建構 (continuous build)，Chef / Puppet 是組態管理與配置，Git 則是版本控制。

451 Research 的企業軟體資深分析師 Jay Lyman 表示同意，他說，談到 Linux 系統管理者與開發者的工具時，我會想到的主要軟體技術與社群，是 Chef、Puppet、CFEngine、Juju、Salt，它們各自由 Opscode、Puppet Labs、CFEngine、SaltStack 提供商業支援。

Lyman 表示，還有許多其他工具與框架，像是 Apache 的網路與應用程式伺服器、Git、Jenkins、node.js、PHP、Python、RabbitMQ、Ruby，和企業經常在 Linux 上使用到的其他軟體。此外，資料儲存與大數據工具也經常和 Linux 共同運用，其中包括 MySQL、PostgreSQL，MongoDB 與 Riak 等 NoSQL 資料庫，Cassandra 與 Hadoop 等資料管理技術。

他指出，當然，由於運用多種語言已經成為一種趨勢，熱門且優異的工具清單可以想見會不斷增長與改變，所以在此列舉的不過是其中一部分。不過，這也顯示出今日的企業開發者與系統管理者，傾向運用各種資源，更為熟練且有能力為工作運用最佳工具，這也是多語言編程的主要動力之一。
