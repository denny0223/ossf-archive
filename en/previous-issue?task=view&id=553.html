自由軟體鑄造場 誠徵網站程式開發工程師
OSSF    2009/10/22

【公司名稱】中央研究院資訊科技創新研究中心自由軟體鑄造場

【徵求時間】第一階段截止日期：10/31

【職稱】專任研究助理-網站程式開發工程師

【需求】一名

【工作內容】
主要為設計與維護 OpenFoundry 專案開發平台。
該平台使用 Ruby on Rails 網頁框架為主，架設於 FreeBSD 作業系統上，
並整合其它開放源碼的軟體模組。

【應徵資格 / 條件】
學歷要求：專科以上
工作經驗：不拘
必備條件：
1. 對於程式設計具備高度興趣與熱忱
2. 對開放源碼軟體具備高度興趣與熱忱
3. 對程式邏輯及學習能力有信心
4. 瞭解 HTML / CSS / JavaScript 等相關技術
5. 具網站程式開發經驗 (PHP Web Framework, Python Web Framework等，熟悉
Ruby on Rails 尤佳)，或熟悉 Linux / FreeBSD 作業系統

加分條件：
1. 熟悉 MySQL / PostgreSQL 資料庫設計與操作
2. 熟悉網路通訊協定及建置經驗
3. 熟悉資訊安全相關技能
4. 熟悉 Perl 程式語言

【工作待遇】
依資訊創新研究中心規定

【聯絡方式】
曾先生 yftzeng@citi.sinica.edu.tw

詳情請見：https://whoswho.openfoundry.org/case-job/84-2009-09-22-03-45-21.html

Attila    2009/10/25

※ 特別加映篇


第一場：快速改變投影片的順序

製作好簡報檔後，要進行放映是很簡單的事，不過放映前應該再檢查一下比較好。檢查什麼呢？首先是播放的順序是不是符合講解的邏輯。有時候在製作投影片過程中，常常出現事前規劃時所沒有的靈感，做好了之後，如果不從整體的角度去檢視，很容易和原本的規劃有所衝突（衝突不一定是壞，也許是更好），此時也許得重新安排前後的順序，甚至某些投影片必須加以捨棄，千萬不要因為某張投影片是難得的「傑作」而捨不得「砍掉」，否則和整體不合的結果，可是會毀了簡報的效果。

Netgear 路由器遭指控違反 GPL
謝良奇/編譯    2009/10/13

忽視授權條款，很可能使開放源碼成為企業產品的雙面刃。網路公司 Netgear 遭指控旗下某款以 Linux 為基礎的路由器出貨時，並未提供原始程式碼，因此違反了開放源碼授權條款。

這項指控來源是一位 Linux 開發者兼觀察者 Harald Welte 的部落格。Welte 告訴新聞媒體，為小型公司與消費者設計的 Netgear WNR3500L 路由器，出貨時僅附有二進位格式的核心模組。此舉違反了 GNU Public Licence (GPL) 的條款。根據 Welte 表示，GPL 並不允許開放源碼應用軟體僅連接到二進位的核心模組。

LINA 1.0 beta 釋出 允許 Linux 軟體跨越主要作業系統
謝良奇/編譯    2009/10/15

Lina Software 宣佈該公司的應用軟體移植性解決方案 LINA 1.0，進入 beta 釋出階段。此一開創性技術可協助 Linux 軟體開發者更有效地進入新市場，簡化終端使用者體驗，在無須更換現有硬體情況下，執行 Linux 應用軟體。

LINA 是一套企業級的開放源碼軟體，可允許 Linux 應用軟體在 Windows、Mac 與 Linux 各主要作業系統上執行，並提供原生介面外觀。LINA 讓開發者使用開放源碼的 Linux 工具撰寫並編譯程式碼，然後快速地轉換所產生的應用軟體到 LINA 的泛用安裝程式上。
