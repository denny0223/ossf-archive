AMD 加入開放源碼發展實驗室 加速企業 Linux 解決方案開發

Created at Friday, 23 April 2004 08:00      Last Updated on Friday, 09 March 2007 01:56 Written by 謝良奇
微處理器商 AMD 日前成為開放源碼發展實驗室（Open Source Development Labs，OSDL的一員，象徵以推動 Linux 應用為目標的開放源碼發展實驗室，持續在產業界獲得支持。AMD 將加入 Carrier Grade 及 Data Center Linux工作小組，以上均為開放源碼發展實驗室成立，各別以加強 Linux在電訊、資料中心基礎架構應用適用性的技術組織。

晶片製造商 AMD 在 64 位元的 x86 計算市場佔有領先地位，多種小型處理器在嵌入式設備中受到廣泛採納。AMD 於 2000 年推出的Hammer 64 位元架構，由 SuSE 和 Red Hat 迅速地發展 Linux 移植。針對嵌入式叢集應用，LinuxBIOS 也支援了 AMD64。開放源碼發展實驗室是一個專為加速 Linux 普及而成立的全球性協會，提出並維護 Linux 電訊應用的 Carrier Grade Linux 規格，為該實驗室贏得不少掌聲。開放源碼發展實驗室成員同時也是 Linux 創始者 Linus Torvalds認為，64 位元擴充代表另一個重要進展，開放源碼社群很快地已體認其潛力。AMD 嵌入式處理器包括 2003 年購自 National Semiconductor 的Geode，以及 2002 年 Alchemy Semiconductor 收購案所取得，基於 MIPS 的 Alchemy 系列晶片。AMD 並針對 PDA、網路存取設備、車載資訊服務平台（automotive telematics ）等設備，販賣 Alchemy的參考板（reference boards）及參考設計。開放源碼發展實驗室執行長 Stuart Cohen 表示，隨著 AMD 技術上Linux 部建的迅速成長，AMD 將會是該實驗室持續成功的重要助力。AMD 會為他們的工作小組帶入豐富的 Linux 解決方案經驗。AMD 微處理器事務部副總裁兼總經理 Marty Seyer 指出，對於 AMD和開放源碼發展實驗室，加速具延展性、可靠性、可負擔性的企業Linux 解決方案發展，是兩者的共同目標。透過加入開放源碼發展實驗室的產業工作小組，AMD 將可貢獻其可觀的微處理器平台技術，推動 Linux 成為策略性作業系統。
