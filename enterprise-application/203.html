從 Windows 遷移到 Linux（下）

Created at Friday, 09 June 2006 08:00      Last Updated on Tuesday, 06 March 2012 16:58 Written by 陳貴成

上一期提到了從 Windows 遷移到 Linux 時的考量要點與重要步驟，接下來探討的是萬事俱備之後，應該如何進行下一步：

◎ 當做好充足的準備及規劃之後

1.選擇一個率先移轉的目標
　a.最好是一個部門
　b.而且是易於管理的單位
　c.這個部門對公司而言有明顯的核心價值
　d.而且有許多不同的電腦使用者型態

2.這樣的好處是可以避免掉
　a.若是策略需要修正，不會影響整個公司
　b.可以建立公司其他單位在移轉時的信心
　c.不建議由個別的部門中組成一個群組進行移轉，這樣群組的個體
　　會在部門的同事間發生一些資訊及知識交換的困擾。

 

◎ 提醒事項

1.選擇適合的 Linux：
在作這個決定之前要先確定內部有那些人才，也就是進行專業的 Linux人力資源盤點，這樣在遷移的過程中才可以把痛苦指數降低。因為雖然都是 Linux，但是不同的套件之間還是會有許多不同的差異，最明顯的例子就是 Debian 跟 Redhat 的套件管理方式就有許多的不同。

2.硬體需求的更換：
目前大多數的 pc 相關週邊都可以找到 Linux 支援的產品，但是在原本的資訊硬體環境中並不一定會有這樣完全支援的環境，所以在遷移之前硬體的評估也是必須的，因為這也會是成本之一。

3.選擇一種合適的視窗管理套件：
KDE、GNOME 是目前最多人在使用的桌面管理套件，但是他們通常較為消耗硬體的資源，所以若是希望在效能較差的硬體上進行 Linux 的安裝時，例如：icewn 或是 xfce 都是可以選擇的視窗管理套件。
當然還有許多的不同的視窗管理套件，但以目前在台灣的 Linux 社群使用習慣來看，應該上述的四種可以取得較好的技術支援，尤其是Gnome 或是 KDE。

4.應用程式的遷移考量：
目前在 Office 的選擇上大多會選用 OpenOffice.org 做為 Microsoft Office 替代方案，其他不同的應用軟體通常都已經有可以替代的方案在進行，在台灣可以在下面的幾個政府推動的計劃中找到一些資源：
中研院資訊科學研究所 自由軟體鑄造場計劃
教育部委辦 校園自由軟體應用諮詢中心
經濟部工業局自由軟體入口網站

5.郵件、行事曆、個人管理工具的選用：
在目前的 Linux 環境中，以 Novell 的 evolution 跟 outlook 的功能最為相近，但若並不要求與 outlook 的功能相近，那麼 KDE 上的 Kontact 也是一個不錯的選擇。但同樣的舊有郵件的移轉必須要有良好的規劃。

6.瀏覽器的選用：
目前在 Linux 應該都是以 Mozilla 家族的產品為主要選擇，例如：
firefox

7.即時通訊軟體 (IM) 的選用：
目前 Linux 上的 IM，例如 Gaim 或是 Kopete 幾乎可以支援目前市面上所有主流的通訊協定，包含 msn、icq、yahoo messenger、google talk 等，就連 skype 也有出 Linux 的版本，所以若要執行 IM 的基本功能，在 Linux 不是問題。

8.資料庫的遷移：
因為資料庫會牽扯到許多重要的營運資料，所以這段其實要非常的小心，若沒有 Open Source 資料庫，例如：MySQL / PostgreSQL 的管理經驗，通常會建議交給專業的公司協助進行。

9.檔案共享的方式：
這部份通常在 Linux 上可交給 SAMBA 服務就可以作不錯的處理了。

10.多媒體的應用程式：
目前在台灣，自由軟體應用在多媒體領域開發用的軟體並不多，但若是要以娛樂用的就有不少好用的選擇，例如 vlc 跟 mplayer 都是不錯的播放軟體，還有 realplay 也有出 Linux 的版本。但若是以辦公室應用來說，上述的軟體應該都夠用。畢竟辦公室應該不會成為視聽娛樂中心。

11.備份的策略：
單純的異地備份通常可以用 rsync 來處理，但是若是有要備分資料庫的需求，通常會 建議採用商業的備份軟體，例如：CA 或是 veritas都有支援 Linux 的備份軟體，但是他們同常都只支援商業版本的 Linux，例如：redhat 或是 SuSE，所以在資料庫平台的 Linux 選擇上，這點必須要考量進來。 ◎ 其他可以進行的目標 1.易於管理的統合性環境在 Novell 的環境中 SuSE Linux 可以整合 edirectory 的技術做到集中試的管理，甚至是跟 Windows 的 AD 進行部份的資源整合，例如人員帳號。

在 openSource 的環境中，一樣可以運用 OpenLDAP 的技術進行系統帳號的整合及帳號的集中管理。

2.可以在 Linux 上執行的商業應用軟體例如：erp 軟體、會計軟體、進銷存軟體……。這是目前遇到的遷移主要問題之一，若是仔細評估過，可以用 remote desktop protocol來將 Linux 連接到 Windows的Server 來解決部份的需求，但需要更多廠商加入開發的行列，並不一定是為了 Linux 開發，可以以跨平台為訴求，目前已經有愈來愈多的軟體開發業者在進行這樣的動作。 所以經過上述的一些評估及說明之後，企業組織其實是可以好好地考量一下資訊環境移轉的必要性，再來決定是不是要進行遷移。當然有一件事情要避免 ，就是未經詳細的規劃就進行移轉的動作，因為這樣通常就是註定失敗。當然移轉成功後，再也不用被特定軟體公司抽稅收費的感覺應該也是不錯的。

各位看倌，您準備好了沒呀？？
