微軟、Amazon 簽訂相互授權協議 範圍包括 Kindle、Linux 伺服器

建立日期 2010-02-23 08:00      最近更新在 2010-06-07 22:37 作者是 謝良奇/編譯

微軟和 Amazon 已經簽訂了專利相互授權協議，協議內容包含運用開放源碼和 Amazon 私有軟體元件的 Amazon 電子書閱讀器 Kindle，以及 Amazon 對以 Linux 為基礎的伺服器使用。Amazon 依據協議將支付微軟一筆未透露其數額的金錢。

微軟過去也曾和其他銷售或使用 Linux 的公司簽訂類似協議，因而惹惱自由與開放源碼社群。微軟宣稱 Linux 和其他開放源碼元件侵害其專利，如今 Amazon 的動作看似承認此說法正當性，可能又會引來 Linux 陣營的怒火。雖然長久以來微軟堅稱部份 Linux 技術使用了該公司專利，不過該公司卻從未具體指出是哪些專利。與 Amazon 之間的協議再度讓外界猜測微軟的意圖何在。

ITIC 首席分析師 Laura DiDio 指出，曾經公開反對開放源碼的微軟，近年來開始採取較為務實態度，其中最常見的就是類似的相互授權協議。她說，微軟了解到高成本的法律動作往往得不償失，其中最明顯的就是 Sun 與微軟間的多年纏訟。DiDio 表示，簽訂這些協議的另一個理由，則是為了爭取歐盟執行委員會 (European Commission) 的支持。

不到一年前，微軟曾經向 TomTom 提告，TomTom 也以反訴回擊。由於微軟舉出的 3 項專利都是基於 Linux 之上，許多自由與開放源碼支持者相信，微軟只是利用法庭，悄悄對 Linux 發動攻勢。

DiDio 宣稱 TomTom 的案例不應作為微軟敵視開放源碼的證據，反而是支持了該公司致力於相互授權策略。她認為，微軟將此視為合理的侵權行為，但 TomTom 並不願和該公司進行協商。微軟則宣稱在採取法律行動一年多前，曾試圖與 TomTom 進行授權談判。

在這次的協議中，微軟可以取得某些 Amazon 技術的授權，包括 Kindle 與 Amazon 的 Linux 伺服器應用，而透露不多的協議內容，不禁讓人猜測這是否意味著包括在 Amazon Web Services 的 Linux 映像檔案。

在 Amazon 方面，很顯然地該公司可以獲得承諾，微軟不會因為使用 Linux 而控告該公司。

Techdirt 部落客 Mike Masnick 指出，相互授權只是微軟的敲詐手段。他說，每過幾年微軟就會宣稱 Linux 違反其專利。這種如果你不告我們，我們也不會告你的協議，在技術領域已經是司空見慣。Masnick 認為，最終除了授權自有專利之外，Amazon 仍然必須支付金錢給微軟，當一家公司只是因為要使用 Linux，就必須付費或授權專利給微軟，看來似乎很有問題，不是嗎？或許是時候重新定義微軟稅。

這意味微軟可以突然出現在某些以 Linux 為基礎的大型技術公司，然後說，該公司想使用他們的技術，只要授權該公司使用，該公司就不會在 Linux 議題上控告他們。這是一個非常有效的策略，畢竟如果微軟控告他們，他們可能會損失一大筆錢。這些公司很可能會接受微軟的提議，只為了為未來的 Linux 專案買個平安。SCO 當年就曾經讓某些公司相信他們擁有 Linux 程式碼而付費。

IDC 的應用程式開發軟體計畫主任 Al Hilwa 表示，選擇相互授權協議並不代表微軟在開放源碼議題上已經舉白旗投降。令人驚訝的是，到目前為止，開放源碼社群對此還沒有太大的反彈聲浪出現。

Linux 基金會 (Linux Foundation) 執行董事 Jim Zemlin 就在部落格上寫到，相互授權協議不是件新鮮事。還是寫程式比較重要。

微軟智財與授權副法律顧問 (Deputy General Counsel) Horacio Gutierrez 表示，與 Amazon 之間的協議顯示，兩家公司對於不論是私有或開放源碼軟體，都有針對智財議題達成務實解決方案的能力。
