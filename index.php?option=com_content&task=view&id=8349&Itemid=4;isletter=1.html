Red Hat 推出 PaaS 方案 OpenShift 而 Ubuntu 雲端平台採用 OpenStack

建立日期 2011-05-17 00:00      最近更新在 2011-05-23 17:06 作者是 謝良奇／編譯
Red Hat（紅帽）正跨越其雲基礎 (Cloud Foundations) 計畫並提供新的平台即服務 (Platform-as-a-Service, PaaS)、架構即服務 (Infrastructure-as-a-Service，IaaS) 解決方案。另一方面，Canonical 則宣佈 Ubuntu Cloud 未來版本採用的架構即服務方案，將以 OpenStack 取代過去的 Eucalyptus。

在美國波士頓舉行的 Red Hat 大會 (Red Hat Summit) 上，Red Hat 公開了特別為開放源碼開發者打造的平台即服務解決方案 OpenShift，OpenShift 支援多種開發環境，以供在雲端上部署應用軟體。除了 OpenShift，Red Hat 也在大會上宣佈基於超過 60 種開放源碼專案的架構及服務方案 CloudForms。

OpenShift 是以 Red Hat 在去年 11 月購併的 Makara 公司所開發的技術為基礎。該雲端服務背後包含許多開放源碼產品，Red Hat 更計畫將包括使用者介面等若干尚未開放源碼的元件加以釋出。

OpenShift 中兩種不同類型的服務，已經提供為數不多的用戶群在註冊後免費使用。"Express" 版本類似於 Google 的 App Engine，提供給只希望將應用軟體轉換到雲端的使用者。"Flex" 版本則讓用戶可以完全控制認證雲端供應商（目前僅有 Amazon）的虛擬伺服器 (instances)。

上述兩種版本目前都處於開發者預覽的階段，預計今年稍晚將正式推出。此外，最強大的服務版本 "Power"，預料屆時也將問世。與其說是平台即服務，該版本較接近架構即服務解決方案，將提供使用者完全控制雲端部署的功能。

Red Hat PaaS 大師 Isaac Roth 表示，OpenShift 是 Red Hat 針對平台即服務提出的開放源碼方案。OpenShift 針對 Java、Python、PHP、Ruby 提供了多種開發框架的支援，包括 Spring、Seam、Weld、CDI、Rails、Rack、Symfony、Zend Framework、Twisted、Django、Java EE。

儘管 Red Hat 將 OpenShift 定位為開放源碼解決方案，但目前並不完全是如此。Roth 在大會中承認，OpenShift 某些部份並未開放源碼。Roth 表示，OpenShift 某些部份尚未開放源碼，例如使用者介面，他們計畫將 OpenShift 開放源碼，雖然目前並無時間表，但他們正在努力進行。

介紹 OpenShift 的同時，Red Hat 也宣佈其 OpenShift 夥伴計畫 (OpenShift Partner Program)。包括 Appcelerator、Bitnami、Cotendo、Couchbase、EnterpriseDB、eXo、MongoDB、Zend Technologies，多家合作夥伴正計畫提供產品以允許自家軟體在 OpenShift 上使用。OpSource 則是第一個加入此一計畫的架構即服務廠商。

另一方面，日前同樣在雲端平台上做出重大決定的，則是 Ubuntu 散佈套件的商業贊助者 Canonical。Canonical 與 Ubuntu 開發者日前在 Ubuntu 開發者大會 (Ubuntu Developer Summit) 上，決定將以 OpenStack 作為 Ubuntu Cloud 平台的基礎技術，放棄過去採用的 Eucalyptus。

Canonical 在聲明中指出，Ubuntu 團隊與 OpenStack 專案的緊密合作，將成為未來 Ubuntu Cloud 產品的堅實基礎。這項決定闡明了 Ubuntu Cloud 未來 12 個月的方向。

過去 Ubuntu 採用 Eucalyptus 作為核心雲端技術。Eucalyptus 從 2009 年起納入 Ubuntu 9.04 成為 Ubuntu Enterprise Cloud (UEC) 產品的基礎。該技術複製了 Amazon EC2 的 API。

2010 年 7 月從 NASA 與 Rackspace 的開發成果中浮出台面的 OpenStack，則是以打造開放標準、社群推動的雲端平台為目標。目前受到包括 Dell、Intel、AMD、Citrix、NTT、RightScale、Puppet Labs、Canonical 在內超過 60 家廠商支援。

Canonical 今年稍早時便宣佈在 Ubuntu 11.04 中提供 OpenStack 的技術預覽，並整合 OpenStack 的最新 Cactus 釋出版本到該散佈套件中。這意味 11.04 同時支援了 OpenStack 和 Eucalyptus 兩套雲端專案。

Ubuntu 表示 Canonical 將持續提供 Eucalyptus 下載與支援。由於針對 Ubuntu Server 10.04 LTS 版本的支援將延續至 2015 年 4 月，因此已經部署 UEC 的企業沒有擔心的必要。Canonical 的 Robbie Williamson 表示他預期 Ubuntu 11.10 將會包含 Eucalyptus 3.0。而 10 月預期釋出的 Ubuntu Server 11.10 版中，將提供轉換至 OpenStack 所需的技術工具。
