各國政府接連宣佈採用開放源碼軟體

建立日期 2011-07-12 00:00      最近更新在 2011-07-26 11:15 作者是 謝良奇／編譯

最近各國政府單位不約而同做出採用開放源碼軟體的宣示，大有開放源碼普及於政府機構主流應用之勢。首先是巴西政府決定投資 OpenOffice 與 LibreOffice 的開發，從開放源碼軟體的使用者，更進一步成為開放源碼的貢獻者。美國聯邦通信委員會 (Federal Communications Commission，FCC) 宣佈將運用開放源碼技術翻新其 FCC.gov 網站，希望抓準機會成為聯邦政府中的專家技術機構。印度政府在最新的電子化政府政策草案中宣示，所有新的專案都必須在開放源碼作業系統上運作。澳洲聯邦政府則針對如何應用開放源碼推出更新過的指南。


關於使用開放源碼軟體的組織，是否適當回饋開發社群的問題，過去有過不少爭論。儘管某些觀察家表示使用與貢獻之間的懸殊現象無關緊要，也有其他人強烈認為使用開放源碼軟體的組織，應該協助開發或投資其開發。基於對 OpenOffice 與 LibreOffice 這兩套開放源碼辦公室套件軟體的使用，巴西政府日前決定加以投資，可說是從使用到回饋的範例。

日前到巴西參加全球最大軟體自由會議 FISL 的開放源碼組織 (Open Source Initiative) 董事會成員 Simon Phipps 表示，文件標準社群中的重要人士 Jomar Silva 在會議中宣佈，Apache 和文件基金會 (The Document Foundation，TDF) 雙方面的社群代表 Jomar Silva 以及 Olivier Hallot，已經和巴西政府簽署了意向書，巴西政府未來不再只是開放源碼的使用者，將開始直接參與該辦公室套件軟體。

Phipps 認為 OpenOffice.org 程式碼庫的開發者社群成長一直以來處於被扼殺的情況，儘管其中有不少出色且具經驗的開發者，但多數都來自 Sun 或 Oracle。隨著 Oracle 的撤退，程式碼維護的責任正從組織贊助者轉交到 Apache 和 TDF 支持的社群管理人手上。然而這兩個專案必須面對的是，義務工作者之中僅有少數開發者，足以取代 Oracle 所雇用的資深開發者。

因此巴西政府宣示投資 OpenOffice 與 LibreOffice 所意味著開發者社群將會成長，根據 Phipps 表示，正是他們在此一時刻所需要的好消息。

美國則是要使用開放技術應用到政府網站。FCC 常務董事 Steven VanRoekel 表示，他們正努力重新設計 FCC.gov，定義現今的公民與企業特質，建構資料基礎架構，做出首次的網站分析與使用者調查，並且在線上與現實生活中與使用者對談，以了解他們如何重新想像 FCC.gov。

FCC 新線上平台將借重與 WhiteHouse.gov 相同的開放源碼技術，FCC 也計畫積極參與該開放源碼社群。他們發現到開放源碼技術成為了最大化 WhiteHouse.gov 延展性與親和性 (accessibility) 的絕佳方式。

印度政府在名為電子化政府硬體採購設備驅動程式 (Policy On Device Drivers For Procurement Of Hardware For e-governance) 的政策草案中提到，印度政府將努力提供技術中立、廠商中立、具成本效率與相互操作性的電子化服務，而關於開放標準的政府政策正是為達成此一目標所踏出的一步。該政策將適用於所有新的與現有的電子化政府專案。

這項政策的出現是因為在實作開放源碼的過程中，並非在所有通用作業系統上，都有可用的設備驅動程式。正因如此，該草案中指出，印度政府有必要針對電腦與相關周邊的採購提出政策，以便實現對於所有通用作業系統中立的電子化政府專案。

印度政府資訊技術部 (Department of Information Technology，DIT) 針對 OEM 業者提出若干要求。首先，電腦應該能夠執行所有的通用作業系統。第二，OEM 廠商應該為電腦和周邊裝置提供能在所有通用作業系統上運作的驅動程式。第三，針對 GNU/Linux 的電腦與周邊驅動程式應該滿足若干條件：驅動程式的原始程式碼、建構程序、安裝指令的授權方式，必須依照任何一種該政策所列舉開放源碼軟體授權；使用這些驅動程式的電腦與周邊裝置必須能在最新的穩定 Linux 核心中運作。此外，驅動程式的原始程式碼、建構程序、安裝指令必須符合 LSB-3.0 或更高版本。在僅提供二進位格式驅動程式時，這些驅動程式必須相容於 Linux，而核心相依二進位檔案必須符合最新穩定 Linux 核心的規格。

無獨有偶地，澳洲聯邦政府繼 3 月釋出尋求公開意見的草案後，日前發表了最新的 2.0 版開放源碼軟體指南 (Guide to Open Source Software)。澳洲政府資訊長 Ann Steward 表示她非常樂見澳洲政府資訊管理辦公室 (Australian Government Information Management Office，AGIMO) 針對該草案提出意見徵求所獲得的回應。

以專案管理人與採購團隊為對象，這份指南預期能在軟體授權與風險有關資訊上，補足澳洲政府的 2011 年 1 月開放源碼軟體政策 (January 2011 Open Source Software Policy)。該政策是針對資通訊軟體的聯邦採購指南。

這份新釋出的指南力促組織在私有與開放源碼軟體間作選擇時，應該將成本、客製化能力、終端用戶訓練、可靠性、廠商依賴 (lock-in)、授權責任納入考量。
