Ubuntu 9.10 伺服器版增加私有雲端功能

Created at Thursday, 22 October 2009 08:00      Last Updated on Tuesday, 03 November 2009 18:23 Written by 謝良奇/編譯
Canonical 在其 Ubuntu Linux 作業系統最新升級版本中，大打私有雲端功能。先前代號為 "Karmic Koala"，即將釋出的 Ubuntu 9.10 整合若干新的雲端技術。10 月 29 日公開下載的 Ubuntu 9.10 伺服器版本這次納入了 Ubuntu 企業雲端 (Ubuntu Enterprise Cloud，UEC)，一項開放源碼雲端計算環境，採用與 Amazon EC2 (Elastic Compute Cloud) 相同的 API。Canonical 表示，企業可以藉此運用私有雲端。

Canonical 支援與服務主任 Steve George 表示，基本上，這次釋出版本主要是以雲端運算為目標。透過 9.10 版，使用者可以取用雲端運算的核心理念，例如敏捷性、資源集中共用與虛擬化，並且在自有的硬體上部署雲端。

George 指出，使用者可以設定 10 到 15 部的私有伺服器，並且使用與 Amazon EC2 相同的 API，發揮與 Amazon 同樣的功能。應用軟體可以從 Amazon 雲端取回至使用者擁有的雲端，反之亦可。

在 4 月推出的 Jaunty Jackalope 釋出版本中，UEC 首次以預覽方式與使用者見面。幾個月後，Canonical 跟著推出的完整的企業支援服務。在這次的 Karmic 釋出版本中，UEC 與伺服器的整合變得更為緊密，同時更新增了應用軟體的 UEC 商店。此舉不但加強了 Ubuntu 的雲端策略，並且可能為該 Linux 廠商帶來合作夥伴與營收。

UEC 是一系列雲端技術的總稱，其中包含了 Eucalyptus 專案，該專案提供使用現有軟硬體，實作就地部署 (on-premise) 與混合式雲端的開放源碼系統。Canonical 表示，UEC 可以讓雲端環境部署與管理變得更為簡易。Eucalyptus 吸引了 RedMonk 分析師 Stephen O'Grady 的注意。O'Grady 表示，就此次釋出的伺服器部份來看，納入 Eucalyptus 可說是最引人注意的發展。

Ubuntu 9.10 也將在 Amazon EC2 環境上提供 Amazon Machine Image。Canonical 表示，UEC 映像檔與 Ubuntu AMI 是相同的，在其中一個環境可以運作的映像，將可以上傳到另一個環境上使用。

UEC 中提供的商店功能，將允許使用者存取在 UEC 環境中立即可部署的應用軟體。George 表示，當使用者安裝完 UEC，下一步要思考的，就是在雲端上要執行的應用軟體，因此他們希望將 Ubuntu 的映像放置在易於存取的地方，讓使用者能下載然後立即送到雲端上執行。

除了讓 Ubuntu 雲端用戶快速簡易地取得雲端應用軟體外，此一商店概念也讓 Ubuntu 確保雲端應用軟體的品質與安全性，能夠符合 Ubuntu 的安全與維護政策。儘管 UEC 商店有潛力成為 Canonical 營收來源，但這一點並非 Karmic 釋出的主要焦點。George 指出，目前這並非 Canonical 的市場策略，而只是一項 Ubuntu 技術，也是他們的首次嘗試。他們希望在將如 UEC 這類新技術提供給合作夥伴前，先確保該技術已經準備妥善。

9.10 中另一項特色是 MySQL 5.1 資料庫的加入。此外，核心上的改進對於 Xen 和 KVM 虛擬化提供更好的支援，並改善了快取效能。USB 3.0 協定支援使得超高速傳輸速率成為可能。OpenLDAP 的整合則允許 Ubuntu 納入現有 LDAP 網路中。系統管理支援的特色是 WBEM (Web-based enterprise management) 協定，可經由普遍的系統管理工具支援 Ubuntu 環境。
