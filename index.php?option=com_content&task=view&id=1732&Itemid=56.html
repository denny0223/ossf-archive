紅帽購併虛擬化軟體開發商 Qumranet

建立日期 2008-09-09 08:00      作者是 謝良奇/編譯

隨著購併以色列專長為伺服器與桌面虛擬化 (virtualization) 軟體的 Qumranet 公司，紅帽 (Red Hat) 藉此擴展該公司對於虛擬技術的支援。據紅帽表示，購併金額為 1 億 7 百萬美金。

紅帽計畫將 Qumranet 的 KVM (Kernal Virtual Machine) 整合到旗艦產品 Red Hat Enterprise Linux 當中。由於紅帽目前在 Red Hat Enterprise Linux 中，嵌入了由 Citrix Systems 掌握的開放源碼 Xen Hypervisor 做為伺服器虛擬化解決方案之用，因此雖然這次的購併給了紅帽進入桌面虛擬化軟體市場的切入點，外界不禁對紅帽未來如何看待 Xen 伺服器虛擬化技術的應用。

紅帽技術長 Brian Stevens 表示，KVM 是 Linux 核心的延伸元件，可做為原生 hypervisor (bare-metal hypervisor)，直接在硬體上執行，並管理 guest 作業系統。KVM 已被建置於 Fedora，也就是紅帽贊助的社群支援版 Linux 中。

紅帽產品與技術總裁兼執行副總裁 Paul Cormier 指出，該公司視此為進入下一代虛擬化技術的機會。Cormier 認為紅帽與其競爭對手微軟相同，將虛擬化視為作業系統的一部份，而非 VMware 與 Citrix 等廠商，將虛擬化軟體做為獨立銷售的產品。

今年 6 月，紅帽宣佈正在開發輕量級的可嵌入式 Linux hypervisor，其技術將以 Qumranet KVM 為基礎，用以運行 Red Hat Enterprise Linux 與微軟的 Windows 環境。如今掌握 KVM 將有助紅帽推行此一計畫。Stevens 日前向媒體表示，透過此一購併，紅帽可確保 KVM 維持開放。Cormier 透露該 hypervisor 目前處於 beta 階段，將在 2009 初推出。

根據紅帽在網站上的說法，該公司會持續在 Red Hat Enterprise Linux 5 中建置 Xen，也會至少持續支援 Xen 直到 2014 年。然而該份文件也表示，目前對於由 Xen 轉換至 KVM，仍未決定時間表。

Cormier 強調由於紅帽支援 Libvert 虛擬化 API 抽象層，針對整合 Xen 的 Red Hat Enterprise Linux 所撰寫的管理工具與應用軟體，未來仍可在改採 KVM 的作業系統版本上執行。

除了被紅帽視為下一代虛擬化技術的 KVM 之外，紅帽此次透過購併也取得了 Qumranet 開發的 SolidICE，這是一套針對虛擬桌面所發展的高效能、具可延展性的桌面虛擬化解決方案。SolidICE 可以讓使用者的 Windows 或 Linux 桌面，在運行於中央伺服器的虛擬機器上執行。該產品有賴於領先業界的 SPICE (Simple Protocol for Independent Computing Environments) 技術，以連接虛擬桌面到 KVM 虛擬機器。儘管 SolidICE 並非開放源碼產品，但 Cormier 表示，紅帽有意將 SPICE 開放源碼。

Qumranet 銷售副總裁 Craig Bauman 指出，Qumranet 的 SolidICE 與如 VMware、Citrix 與微軟等其他公司的桌面虛擬化產品相較，最大差異在於，以上廠商皆以虛擬伺服器環境為主，並使用如 RDP、ICA 等協定，加入桌面虛擬化。這些協定足以因應伺服器環境管理者之用，對於習慣於高速桌面效能的使用者，卻力有未逮。SPICE 技術提供卓越的使用者經驗，有助克服虛擬桌面架構 (virtual desktop infrastructure，VDI) 的此一關鍵障礙。

Bauman 表示，這一點是桌面虛擬化的弱點。如果終端使用者接收到不好的使用者經驗，就不會接受此一環境。而該公司能讓使用者以為使用的仍是跟過去一樣臃腫的桌面環境。

和技術同等重要的，是紅帽藉購併也獲得 Qumranet 優秀的專業團隊，同時包括 Qumranet  在開放源碼 KVM 專案的社群領導人。

紅帽執行長兼總裁 Jim Whitehurst 表示，紅帽用戶喜愛具高度反應力、彈性且高成本效率的 IT 基礎架構，此一購併進一步擴展該公司能力，擴大了開放源碼與私有基礎架構軟體間的距離，簡單來說，Qumranet 的 KVM 與 VDI 技術是下一代虛擬化前端技術。他們象徵了提高虛擬化解決方案水準並滿足市場需求的機會。
