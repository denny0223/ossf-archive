OpenOffice 3 釋出 首次提供原生 Mac 支援

建立日期 2008-10-23 08:00      作者是 謝良奇/編譯

OpenOffice 商業生產力套件推出最新 3.0 版本，首次在 Windows 與 Linux 之外，提供對 Mac 的原生支援。新版本支援微軟 Access .accdb 檔案，其文字編輯軟體可建立 Web 2.0 XHTML 與 MediaWiki 文件。當 OpenOffice 3 從 OpenOffice 2 帶來如此多的改進之時，有效的推廣策略將能進一步讓更多人了解此一開放源碼軟體。

可用以作為微軟 Office 替代方案的開放源碼軟體 OpenOffice，3.0 版本整合了文字編輯器、試算表、簡報軟體、圖形、方程式與資料庫功能，已經提供 Windows、Mac 與 Linux 版本的下載。

OpenOffice.org 社群經理 Louis Suárez-Potts 表示，3.0 版本提供與 MS Office 更高的相容性，功能更強也更具延展性，可免於桌面環境受制於單一廠商。

OpenOffice 3.0 有一系列的改善之處，足以吸引先前用戶以及考慮從微軟 Office 做轉換的使用者。特別是 OpenOffice 3.0 首次對 Mac 用戶提供原生支援，可用以開啟 Microsoft Office 2007 或 Microsoft Office 2008 的檔案。

此外，新版軟體對於 Visual Basic for Applications 巨集的使用做了改進，甚至支援微軟 Access 2007 .accdb 檔案，並完全相容於越來越多國家官方指定使用的最新 ISO 文件標準 Open Document Format 1.2 (ODF)，以及微軟的 Office Open XML (OOXML) 格式。

作為對 Web 2.0 趨勢的因應，OpenOffice 3.0 也提供相關功能。OpenOffice.org 行銷專案領導人 John McCreesh 表示，透過 OpenOffice 3.0，使用者無須學習難懂的 XHTML 與 MediaWiki 等語言，即可搭上 Web 2.0 列車。藉此，使用者可在 OpenOffice 的文字編輯器 Writer 內，建立網路與 Wiki 文件。

為了減輕微軟 Office 等生產力套件常見的功能膨脹問題，OpenOffice.org 特意將許多補充功能置於線上的延伸套件庫。第三方的開發者提供了更多其他功能，例如 PDF 匯入、建立 Hybrid PDF 文件，以及簡報軟體 Impress 的簡報畫面支援。

透過文字編輯器 Writer 最新的語言選擇工具，使用者可對不同段落指定不同語言，降低多語言文件編輯與拼字檢查的困難。試算表軟體 Calc 中單一表格從 256 行擴展至 1024 行。OpenOffice 3.0 也允許多使用者在試算表上進行協同合作。

過去以來，OpenOffice 的推廣一直依賴四個因素，包括免費、具份量的企業贊助者、對微軟 Office 的不滿，以及口耳相傳。儘管這四個因素都有其限制，然而很重要的一點是，許多人並不知道 OpenOffice 的存在。即使他們知道了，也因為不熟悉而不敢貿然使用。ZDNet 的 Paula Rooney 就建議 OpenOffice 主要贊助者 IBM 與 Sun，能在 OpenOffice 行銷之餘，提供免費的企業級訓練課程。

除了 OpenOffice 之外，Mozilla 近期也發表多項 alpha 與 beta 釋出。像是 Firefox 3.1 beta 1，首次納入 Mozilla 高效能的 TraceMonkey JavaScript 引擎，並支援 W3C Geolocation API，並引入網路開發者樂見的許多新功能如嵌入式網路字形。

Mozilla 的行動 Firefox 瀏覽器，代號 Fennec，也有新的發展。該瀏覽器已移植到 Windows Mobile 並於日前首次釋出 alpha 版本。此外，Mozilla Messaging 也釋出第三個 Thunderbird 3 alpha 版本。
