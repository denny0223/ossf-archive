Adobe 積極推動 Flex Ajax 框架分歧多面臨競爭壓力

Created at Friday, 05 January 2007 08:00      Last Updated on Monday, 05 March 2007 19:15 Written by 謝良奇

Adobe 積極為自家的多樣化網路應用軟體 (Rich Internet Application) 開發技術 Flex 部署，Forrester Research 預測 Flex 與 Ajax 競爭的結果，勢必憾動多樣化網路應用軟體開發工具的市場。同時，Forrester Research 更表示，私有 Ajax 框架的爆炸性成長已經對 Ajax 的開放性、標準化與效能等優勢帶來負面的影響。


企業有鑑於多樣化網路應用軟體為開發者與用戶帶來的優點，在急於踏入這塊領域前，首先要考慮採用的技術。其中，應用層面（如瀏覽器支援度、plugin 的普及度）、開發時間（如部署與測試）與社群基礎是最重要的考慮項目。

Forrester 警告市場上已有數十個常見的 Ajax 框架，表示軟體開發者在計劃納入特定框架前，必須先確認該框架在安全性、可用性、服務整合與資料存取上，能否滿足組織需求。

Ajax 一向以開發者人數眾多而成為企業考量上的首選，考慮到產品要儘快投入市場，企業很容易就能找到擁有 JavaScript 與 HTML 經驗的開發者，這一點也是新興網路公司大量使用 Ajax 的原因之一，推波助瀾下，Ajax 很快地成為趨勢。然而 Ajax 框架市場的分岐過大，卻是問題所在。

此外，不利於 Ajax 的另一項因素在於，Forrester 認為企業 IT 的多數開發者較偏好視覺編輯工具，這類 Ajax 工具幾乎都是商業框架，而非開放源碼工具。Forrester 資深分析師 Jeffrey Hammond 就認為，選用商業 Ajax 解決方案意味著採納私有框架與開發工具。在這一點上，Adobe 才是商業 Ajax 供應商的競爭對手，而非開放源碼 Ajax 工具套件。

Adobe 正積極讓自家技術整合到開放源碼的 Firefox 瀏覽器中。Hammond 表示，Adobe 藉由修改其開放源碼策略，將為這塊市場的激烈競爭開始加溫。Adobe 將透過整合客製元件到 Flex 與推動 Mozilla Tamarin 專案納入 Adobe Flash 9 所使用的 ActiveScript 語言，進一步縮減商業 Ajax 供應商的空間。

儘管 Flex 的學習曲線較高，開發者必須學習 ActionScript 及 MXML，對於期待產品儘早上市的企業，的確成為頗難以接受的跨入門檻，然而在報導指出 Adobe 大力投資 Flex 並為其建構社群以便提高 Flex 開發者人數後，這個問題的重要性可能會逐漸降低。同時，Adobe 更持續為 Flex 的開發性做出努力，如 David Mendels 與 Duane Nickull 日前的文章便是一例。此外 Adobe 已提供免費的 Flex 軟體開發套件與編譯器，希望提高開放源碼社群對於此一平台的接受度。

Forrester 表示開放源碼與商業 Ajax 工具數量已過於飽和，預料用戶將延後其投資決策，直到市場情況明朗，Forrester 也指出 Dojo、GWT 與 Openlaszlo 等開放源碼工具因為使用的機會成本較低，將可因此受惠。
