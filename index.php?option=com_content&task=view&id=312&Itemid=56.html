IBM、Zend 共推 PHP 整合開發解決方案

Created at Friday, 25 March 2005 08:00      Last Updated on Friday, 09 March 2007 01:32 Written by 謝良奇
IBM 為進一步擴大開放源碼開發策略，日前與 Zend Technologies 宣佈合作，推出採用了 PHP 語言的整合程式開發解決方案 "Zend Core for IBM"，企圖在 Java 之外，吸引 PHP 社群背後的廣大開 發者。

Zend Core for IBM 此項合作計劃，囊括兩家公司的開放源碼技術， 包括 IBM 的 Cloudscape 資料庫，Zend 的 PHP 開發環境。Zend Core 可以提供現成的 PHP 開發環境，讓組織用戶開發、部署並管 理開放源碼應用軟體。 IBM 與 Zend 表示，他們將在資料庫整合框架與 PHP 的網頁服務（ Web Services）標準上共同合作。兩家公司希望 PHP 能在組織資料 庫及 網頁服務協定上有更佳的表現。IBM 更計劃在其開發者網站上 設置 PHP 專區，提供技術資源如技術白皮書等。Zend Core for IBM 預計於今年下半年起提供免費下載。 以 IBM 在技術與行銷上的資源，該公司對 PHP 的公開支持，相信可加速此一開放源碼產品的應用，過去 IBM 對 Linux、Java 的投資， 對於該類技術的接受度皆有關鍵性的作用。 雖然 IBM 新興技術（emerging technology）副總裁 Rod Smith 表 示，IBM 會持續投入Java 產品標準及 WebSphere 產品線，一位不願 具名的產業人士認為，IBM 推動 PHP 反映出 IBM 對 Java 標準化程 序以及將 Java 易用化努力的夢想幻滅。該人士表示， IBM 一直在 找尋替代方案，他們希望用戶能快速建構與 IBM 後端接軌的應用軟體，Java 無法滿足其需求。Smith 則表示，Java 和 PHP 各有其領 域，IBM 會持續推展 Java。 Smith 認為，PHP 是 IBM 吸引更多開發者的方法，特別是中小企業。 Burton Group 分析師 Anne Thomas Manes 表示，IBM 投入 PHP 社 群會讓其發展更為專業化。Thomas Manes也認為，命令稿語言一直就很有吸引力，IBM 的動作很合理。 Meta Group 分析師 Thomas Murphy 表示，最初開發 PHP 網站應用 軟體的小型企業，往後因交易需求升級至 IBM WebSphere 伺服器， 也能為 IBM 帶來好處。Murphy 認為，大家開始體認到命令稿語言在市場中自有其空間，IBM 為了打入命令稿語言應用頻繁的中小企業，必得 在其中佔有一席之地。 The Selling Source 的首席開發者 Chris Stephens 表示，他們認 為針對資料庫應用軟體發展與部署的即用解決方案相當重要，而此一 合作對於以 PHP 建構企業系統的組織，可說是一大保證。 此外 IBM 日前也表示將把約 30 項專案發布至 SourceForge.net 上， 貢獻給開放源碼社群，其中包含了 Jikes，一套以 Java 為基礎的參 照編譯器，能加速開發時程。另外還有 Life Science Identifier，可以協助醫療領域的開發者，建構適用於該產業的應用軟體。
