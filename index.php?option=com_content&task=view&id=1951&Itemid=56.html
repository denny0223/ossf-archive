EMC 購併 SourceLabs 增添開放源碼動力

建立日期 2009-01-06 08:00      作者是 謝良奇/編譯

企業資訊儲存公司 EMC 正試圖打入開放源碼界，日前購併 SourceLabs 的部份資產並獲得該公司的主要人才。據外界猜想，SourceLabs 優秀的開發者團隊可能是購併主要理由之一。

位於西雅圖的 SourceLabs 是家開放源碼新創公司，所開發的軟體技術為系統管理者、開發者與品管經理，簡化 Linux 與開放源碼 Java 相關問題的解決過程。該公司的 SourceLabs Support Suite 運用開放源碼協助用戶發現、定義並解決和 Linux 與 Java 有關的系統錯誤、系統當機、缺陷等等，其資料庫包含 1600 萬已知的 Linux 與 Java 問題。

SASH 則是 SourceLabs 的 Java 網路應用軟體快速開發開放平台。此外，SourceLabs 還經營 SWiK.net，這是社群為導向的開放源碼軟體資源，包含有關數千個開放源碼專案相關資訊與新聞。

EMC 發言人表示，該公司已購併 SourceLabs 的部份資產，並聘用了其部份人員，購併的資產與人員將歸於 EMC 的 Decho 事業部之下。

Decho 於去年藉由整合 EMC 的 Mozy 線上備份產品與位於西雅圖的 Pi 而成立。Pi 是 Paul Maritz 創立的個人資料管理新創公司。Decho 運用雲端儲存與資訊處理服務，提供技術給企業與客戶。Maritz 過去曾是微軟平台與策略集團的集團副總裁，在 EMC 購併 Pi 後加入 EMC。

當 EMC 執行長 Joe Tucci 指派 Maritz 為 VMware 新任總裁與執行長後，過去也曾是微軟高階主管的 Harel Kodesh 則被聘用來主導 Decho。Kodesh 同時也是 EMC 雲端架構事業的總裁，該事業部包含了 Atmos 雲端儲存產品。

由於 EMC 發言人表示，此項購併行動與 EMC 的財務報告並無太大關連，因此該公司不會提供進一步細節，如購併的資產項目等。根據該發言人指出，SourceLabs 購併後將以獨立事業體持續運作。

SourceLabs 發言人除了確認購併外，更表示 SourceLabs 執行長 Byron Sebastian 以及 SourceLabs 架構長 Will Pugh，將雙雙加入 EMC。而加入 EMC 的人員都擁有雲端運算相關經驗。該發言人說，SourceLabs 此後將專注在推動 SWiK.net 業務上。

從 EMC 近來一連串購併行動，看出該公司正專注在建構其雲端運算業務。包括該公司在 2007 年 10 月購併了 Berkeley Data Systems 與其 Mozy 線上儲存技術，以及在 2008 年 2 月購併 Pi。

要將 SourceLabs 產品與服務和 Decho 聯想在一起需要一些想像力，畢竟由前 BEA 高階主管 Byron Sebastian 創立的 SourceLabs，其目標在藉由開發產品與服務協助開放源碼軟體管理與支援，幫助 Linux 與開放源碼開發者與管理者發揮其 Linux 與開放源碼軟體的最大價值，並不容易直接連接到雲端運算上。

EMC 或許有意為 Decho 創造出開放源碼開發圈，或許 Support Suite 的技術平台能讓 Decho 用來協助用戶描述需求並搜尋資訊。另一方面，受到經濟蕭條影響的 SourceLabs 在岌岌可危之中，對於 EMC 而言，這是一個用合理價格取得優秀開發團隊的絕佳機會。別忘了 SourceLabs 與 Decho 同位於西雅圖。
