台灣部落格之父－Jedi

Created at Monday, 25 June 2007 07:29      Last Updated on Friday, 06 July 2007 23:17 Written by OSSF 工作小組

在自由軟體社群中，「人」是最重要的資產，一個有才華的開發者、推廣者、或是使用者，都可以對自由軟體專案做出許多貢獻，除了自由軟體社群需要這些人才，其他各界如教育、法律、商業界等等，也需要這些人才的交流，以加速自由軟體的發展與創新。

自本期開始，每個月將為您介紹一位自由軟體人才資料庫中的名家，以增加外界對自由軟體人才的認識，初次登場的是本電子報 [開放原力] 專欄作家，有「台灣部落格之父」之稱的 Jedi。


Jedi 精通網頁親和力 (Web Accessibility) 設計，熟悉 XHTML 與 CSS，長期關注部落格網站與網頁設計相關技術，著有《BLOG 架站實務：使用 Movable Type》一書，在部落格剛在台灣開始發展的時候，嘉惠不少想架站的使用者。

除了以上最為人所知的成就，Jedi 也積極投入自由軟體中文化與文件翻譯，曾參與過 GnuPG、GPGShell、WinPT 等跟公開金鑰加密法有關的軟體中文化，亦參與《Perl 學習手冊第三版》、《Perl 網路程式設計》、《架設 Slash 社群網站》、《FreeBSD 完全探索》、《深入親和力》等書的翻譯。對於以上眾多經歷，Jedi 謙虛地表示，自己其實從事的都是一些基礎建設相關與扎根的工作，同時在一些國外行之多年、但在台灣沒什麼人注意的領域去做第一線的開拓。

Jedi 覺得，自己較常扮演類似「橋樑」的角色，會去接觸使用者端，但是也混開發者社群，兩邊的想法、需求、思考模式都會接觸到，所以他常常會思考如何把兩邊對在一起，讓需求可以被滿足。Jedi 也認為，透過過去參與中文化社群專案，讓他更瞭解使用者操作軟體或網站的需求，「對於使用者來說，真正需要的不是一個應用軟體，或一個系統，而是一個解決方案，所以在社群的經驗以及所能接觸到的資訊就會很有用」Jedi 說。

此外，當 Jedi 在翻譯軟體專案的訊息時，他發現經由瀏覽該軟體的所有使用訊息，就可以很快地了解該軟體所提供的功能，接下來就可以把該軟體能做的事，變成若干「句子」，然後就可以使用對「語言」的感受能力，把「用很多軟體組成一個解決方案」變成「拿很多句子來組合成作文」一般容易的事，這是他覺得做軟體中文化有趣的地方，也幫助他在日後規劃解決方案時，更能從 end user 的角度來看待，也比較有助於稍後的客製化，當然 internal/external spec 要能對的上也很重要。

Jedi 稍早曾任職於中央研究院資訊科學研究所，為「台灣創用 CC 計畫」技術部門首席，並在聯合新聞網數位文化誌撰寫專欄，以深入淺出的方式向普羅大眾介紹自由軟體技術相關議題。目前 Jedi 為全職的自由工作者，有鑒於台灣電子商務與個人網站發達，但是目前為止沒有任何公司有 accessibility team，市面上也很少專門探討網頁親和力的專門書籍，因此他目前正在撰寫一本以網頁親和力主題的新書。關於 Jedi 的經歷，只要您登入中研院資訊所自由軟體鑄造場計畫的自由軟體人才資料庫中，都可以在資料庫中獲知。
