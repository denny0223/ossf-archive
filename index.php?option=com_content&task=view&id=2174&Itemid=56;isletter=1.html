Linux 2.6.32 第一次 RC 版本釋出

Created at Tuesday, 29 September 2009 08:00      Written by 謝良奇/編譯
隨著 Linux 2.6.32 首次 RC (release candidate) 版本日前釋出，Linus Torvalds 完成了 Linux 下一版本在主要開發分支的主開發階段。由於核心黑客們在稱為 merge window 的這個階段中，已經將新核心版本的大多數更動整合到原始碼管理系統，從 2.6.32-rc1 上面已經可以看出預計於 12 月初釋出的 Linux 2.6.32 最重要的新功能。

核心開發者針對核心的 Direct Rendering Manager (DRM) 做出若干改進，包括 3D 支援和Radeon series 2000、3000 與 4000 顯示卡以核心為基礎的模式設定 (KMS)。不過，為了使用這些新功能，必須使用到 Libdrm、Mesa[]3D 和 X.org Radeon 顯示驅動程式的開發者版本。

若干 Wi-Fi 驅動程式和微軟最近以 GPL 釋出的 Hyper-V 驅動程式，已經移入為仍須進一步開發的驅動程式準備的待命 (staging) 區中。待命分支的維護者日前曾因為微軟開發者似乎中斷該公司 hypervisor 驅動程式開發之故，揚言要將該驅動程式自核心 2.6.33 中移除。該公司開發者隨即回復至活躍狀態暫時避免了驅動程式遭移除的下場。若干處於待命區的未維護驅動程式將自 2.6.32 中移除。

一如預料的，今年 Linux 研討會 (Linux Symposium) 上曾詳細解釋的 Kernel Samepage Merging (KSM) 功能，如今整合到核心中了。此外，總是帶來驚奇的核心黑客，也將被戲稱為 Devfs 2.0 的 devtmpfs 整合了進來。儘管 devtmpfs 整合前後曾受到多位知名核心黑客的猛烈批評。另一方面，DRBD (Distributed Replicated Block Device) 複製解決方案儘管有 Andrew Morton 的大力支持，結果並未納入 2.6.32 之中。

另一個新增到核心的，是 Intel Trusted Execution Technology (TXT) 的支援，該技術過去稱為 LaGrande Technology。下一個核心版本將支援處理與避免記憶體錯誤的若干技術，這些技術將隨著 Intel 具備 Nehalem EX 核心的處理器一同推出。負責分配處理器時間給應用軟體的核心排程器，也有數項改進。其他更動則修改了追蹤基礎架構與最近引入、如今稱之為效能事件 (performance events) 的效能計算器 (performance counters)。此外，回寫 (writeback) 架構經過重新撰寫，改善了在 SMP 系統上的 I/O 處理效率。
