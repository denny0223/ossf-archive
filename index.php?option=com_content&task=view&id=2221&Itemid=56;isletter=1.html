Dell 釋出客製化 Chromium OS

Created at Thursday, 03 December 2009 08:00      Written by 謝良奇/編譯
Dell 釋出了可在該公司 Inspiron Mini 10v 小筆電 (netbook) 上執行的 Google Chromium OS  實驗版本。首度承認 Dell 有可能採用這套作業系統。Chromium 是上個月由 Google 授權釋出的 Chrome OS 開放源碼版本，該作業系統以 Linux 核心為基礎，並且看準小筆電市場為其目標。

儘管 Chrome OS 最終產品預計的推出時間，將是明年下半年，Google 決定將其原始碼公諸於眾，讓所有人都能建構自己的客製化 Chromium OS 版本。Dell 的 Linux 技術策劃師 Doug Anson 於是修改了原始程式碼，在 Dell Mini 10v 小筆電上安裝該作業系統，並能無線存取網際網路。Anson 過去曾經與 Canonical 合作，在 Dell 的 Mini 9 小筆電上，實現 Ubuntu 的介面。

在部落格寫到對 Chromium OS 的感想，Anson 表示，沒有網路連線的話，Chromium OS 表現平平，有了網路連線，Chromium OS 才會開始發光。Anson 指出，Chromium OS 是小型的最佳化作業系統，極度簡化且易於瀏覽網路是其目標。開機時間快速，從按下電源鈕後大約需 12 秒。Chromium 瀏覽器非常快速，並且創造出以網路為中心的豐富瀏覽體驗。

Anson 準備的 7.5GB USB 映像檔案，可以從 Dell 的網站上下載。不過這個 Chromium OS 映像檔，完全只針對熱血的 Linux 實驗玩家，而非一般的用戶。Anson 表示，使用這個映像檔請自負風險，該映像檔案完全不受支援，且只經過最小程度的測試。系統不包含關機與重開機的選單選項，要重新啟動，得按下 Mini 10v 的電源鍵。

Anson 表示，目前主要遭遇的問題，是該系統的連線管理程式與底層元件 (wpa_supplicant)，很容易當掉或卡住。如果遇上這些問題，得重新開機或者多試幾次才行。

此外，Anson 說，Chromium OS 的網路連線管理程式，可能會花 5 到 10 分鐘以上時間，才能找到網路存取點。不過另一方面，有線網路連線則是運作正常。

聽取開發者回應是 Google 決定及早釋出 Chromium OS 原始碼的原因之一。Google 軟體工程師 Glen Murphy、Martin Bligh 與 Will Drewry 表示，其目的是為了讓有興趣的開發者貢獻程式碼、想法與設計。

Google 是在 7 月宣佈 Chrome OS 的消息，並且在 11 月中釋出其開放源碼版本。Google 的集團產品經理 Caesar Sengupta 與工程主管 Matt Papakipos 表示，這麼作的原因是，他們希望與合作夥伴、開放源碼社群和開發者們及早建立關係。仍有許多工作有待完成，而他們很高興能與開放源碼社群合作。他們過去從 GNU、Linux 核心、Moblin、Ubuntu、WebKit 等專案上獲益良多，未來將貢獻其程式碼，並與這些專案緊密合作。

開發者們指出 Chromium OS 原始碼目前版本，已經在許多消費者等級機器上成功地進入 Wi-Fi 模式，其中包括 ASUS Eee PC700 與 1008HA、Lenovo ThinkPad T400 與 Toshiba 的 mini NB200。

然而，Gartner Research 副總裁與知名分析師 Michael Silver 指出，在企業端，Google 以 Chromium OS 切入利基的機會十分微小，因為大多數組織執行的應用軟體，超過半數以上都需要 Windows。Silver 表示，假如有利基存在，Linux 已經可以切入，然而桌面 Linux 事實上並不普及。Chrome OS 需要組織擺脫更多的 Windows 需求，並且改善連線可用性，才能獲得成功。
