GNOME Shell 延伸套件網站推出 降低 GNOME 3 客製化門檻

建立日期 2011-12-12 09:51      最近更新在 2012-05-09 14:56 作者是 謝良奇／編譯

對於一時之間無法接受 GNOME 3 的用戶， GNOME 近期借鏡 Linux Mint 12 的作法，藉由延伸套件 (Extensions) 讓他們逐漸熟悉新使用者介面。GNOME 專案日前公開了找尋與安裝 GNOME Shell 延伸套件的新網站 extensions.gnome.org，協助 GNOME 3 用戶利用現有的延伸套件，對其桌面環境進行客製化。

 

GNOME Shell 延伸套件一般是由第三方開發者所撰寫，並未獲得 GNOME 社群支援，是一些簡短的程式碼，可以用來改變 GNOME 的運作方式。

GNOME 開發者 Jasper St. Pierre 表示，目前進入 public alpha 階段的 GNOME Shell Extensions 網站，允許執行 GNOME Shell 3.2 版的用戶從其網路瀏覽器中安裝新的延伸套件。這一切都有賴於運用了 GNOME Shell Integration 瀏覽器 plug-in， St. Pierre 表示此一 plug-in 很可能早已安裝於 GNOME 3.2 用戶的系統當中。

不過 GNOME Shell Integration 目前僅支援Firefox，在建基於 WebKit 的瀏覽器如 Epiphany 和 Chrome 上會有一些問題。

St. Pierre 指出，針對未來 GNOME 版本可能產生的新功能，延伸套件提供了進行原型開發的機會；對於不一定相容於 GNOME 整體設計理念，但仍有實用性的功能，進階用戶也得以透過延伸套件進行客製化。

St. Pierre 也表示，因為這些延伸套件會成為核心作業系統的一部份，所有上傳到這個新網站的延伸套件都要經過程式碼檢視流程才會提供給用戶下載。他說，該網站目前提供的延伸套件數量，和現今所有延伸套件總數相比，僅是一小部份，不過該專案預期未來數週內將加入更多延伸套件。

目前該網站似乎僅將所有延伸套件列表呈現，除了不提供任何搜尋功能，似乎也沒有提供成功的安裝回應。這些地方顯然會在該網站完成 alpha 測試，並增加其延伸套件數量後逐步修改。

全球最受歡迎的作業系統之一 Ubuntu，近來持續因為新介面 Unity 缺乏客製化能力與選項飽受批評。反觀 Linux Mint 12 運用延伸套件作為 GNOME 3 敲門磚，卻受到不少好評。許多不滿 Unity 的用戶紛紛改用 LinuxMint 或嘗試其他以 Linux 為基礎的作業系統。Distrowatch 上的評比雖然不能代表實際的裝機量 (install-base)，然而 LinuxMint 與其他散佈套件取代 Ubuntu 的榜首地位，應該可作為給 Ubuntu 開發者的警訊。但是，至今似乎還沒有出現任何來自 Ubuntu 的官方 Unity 延伸套件專案。
