Dell、HP x86 伺服器銷售 Oracle 作業系統 OpenSolaris 與 Oracle 溝通受挫揚言解散

建立日期 2010-08-16 19:40      最近更新在 2010-08-16 19:43 作者是 謝良奇/編譯
Oracle 宣佈Dell、Hewlett-Packard 將在各自的 x86 伺服器上，認證與經銷 Oracle 的 Solaris、Enterprise Linux、Oracle VM。在此合作協議之下，Dell 和 HP 的 x86 伺服器用戶將可購買 Oracle 的技術支援服務，並取得未來的更新。如此一來將可快速而精確地解決用戶的問題，並減少企業作業環境之中的風險。

購買 Oracle 技術支援服務的用戶，將可受惠於 Oracle 未來對 Oracle Solaris、Oracle Enterprise Linux、Oracle VM 的持續投資，以及未來更新中的創新之處。

Oracle 總裁 Charles Phillips 表示，Oracle Solaris 是產業第一名的 UNIX 作業系統，其需求跨越多重 x86 平台。此外，越來越多用戶運用 Oracle Enterprise Linux 和 Oracle VM，在 x86 平台上建構虛擬環境。今天他們宣佈的消息顯示了 Oracle 的開放承諾，並且將提供 Dell 和 HP 用戶不同層次的支援、深入的產品專門技術，而且減少 IT 環境中的風險。

Dell 全球聯盟副總裁 Joyce Mullen 指出，Dell 藉由在其 x86 伺服器上供應 Solaris、Oracle Enterprise Linux、Oracle VM 以及其他作業系統，提供用戶許多選擇和彈性。他們的用戶將能夠依靠該公司備受讚譽的伺服器，以及來自 Oracle 的軟體資產，打造出穩固、可依靠的最佳化 IT 平台，在協助用戶更具競爭力的同時，最大化其技術投資的投資報酬率。

HP 解決方案和策略聯盟、企業伺服器、儲存和網路副總裁 Paul Miller 認為，用戶必須針對動態商業需求立即調整腳步，但是許多用戶綁定的應用軟體堆疊和基礎架構難以迅速更動。Oracle 基礎架構軟體和 HP ProLiant 伺服器的組合，能夠在 x86 伺服器上達成出色的效能、延展性、虛擬化能力。他們共同的用戶因此在擴展其業務時，能夠對掌控其成本擁有足夠的自信。

然而在此同時，對於 Solaris 此一 UNIX 作業系統的開放源碼散布版本 OpenSolaris，外界的疑慮卻逐漸升高。受挫於與 Oracle 之間的溝通不足，如果情況短期內未見改善，OpenSolaris 的理事會成員揚言將採取解散一途。
