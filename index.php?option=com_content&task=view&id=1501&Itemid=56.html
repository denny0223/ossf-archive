金融產業 Linux 部署焦點轉向多作業系統統整、即時應用軟體

Created at Thursday, 10 April 2008 08:00      Written by 謝良奇/編譯
隨著銀行等金融服務產業部署 Linux 的情況越見普遍，企業端扮演的角色重要性趨向顯著。華爾街 Linux/Open Source 大會 (Linux/Open Source on Wall Street) 的某座談會程就以開發者端與商業人士間的合作為主題。此外，成本議題過去是金融業先期導入 Linux 與開放源碼軟體的主因，如今情況似乎有所轉變，多作業系統統整與即時應用軟體成為產業關注的焦點應用。 不久前，金融產業仍以降低成本以及客製化元件，當成轉向 Linux 與開放源碼的主要原因。在日前的華爾街 Linux/Open Source 大會上，會談者開始將焦點放在大型主機上多作業系統伺服器統整與 PC 伺服器的即時 Linux 應用軟體開發等主題。這個趨勢反映了金融等企業軟體環境愈趨複雜的現況。

以 Citigroup 為例，資深架構與技術工程副總裁 Aaron Graves 表示，公司計畫在最近於 IBM Z10 伺服器上完成統整的一系列核心 Unix 與大型主機應用軟體中，增加 Linux 應用軟體。

Graves 指出，不論應用軟體執行平台為何，也不論應用軟體是否為開放源碼，對他來說最為重要的，是獲得軟體廠商或散佈商的技術端支援。Graves 表示，Linux 的優勢在於 Linux 應用軟體可以透過一般或虛擬化方式，在 X86 PC 與大型主機等各式硬體系統上執行。

對於 Credit Suisse 的 IT 研究與開發資深副總裁 Head Bubba，即時應用軟體多樣化是 Linux 的主要優勢。他正在從事交易商的即時 Linux 應用軟體開發。另一位講者 IBM Linux Technology Center 的計畫主任 Keith Bright 認為，即時應用軟體原有特性，使得開發者較能輕易限定其行為，並支援服務水準協議 (Service Level Agreement，SLA) 規範。

Bright 指出，過去數年間，紅帽 (Red Hat) 與 Novell 兩大 Linux 散佈廠商，也都各自增加對即時應用軟體的支援。由於即時應用軟體一般而言在效能上較為差強人意，因此 Bubba 表示 Credit Suisse 正在設法改善 Linux 即時應用軟體的效能問題。

在這場大會上，企業人士與開發者的協同合作成為熱門關鍵字，Linux 開發者突然間發覺周遭充斥企業主管。

該大會座談會程由紐約梅隆銀行 (Bank of New York Mellon)、軟體自由法律中心 (Software Freedom Law Center)、Alfresco 專案 (Alfresco Project) 與協同軟體機構 (Collaborative Software Institute，CSI) 多位講者上場。

紐約梅隆銀行技術風險管理主任 Stan Rose 認為政府與法規規範需要，使金融產業界的商業與技術人士得以共同聚首。舉例而言，根據 Rose 表示，約有 100 家主要金融服務企業的技術與管理階層透過 Bits 專案共同合作，目前他們正著手在 Linux 上執行終端使用者應用軟體。

軟體自由法律中心董事兼共同創辦人 Eben Moglen 認為，在 Linux 與開放源碼軟體上執行企業應用軟體的漸增需要，可說是促成技術與商業端合作的主要動力來源。

對此，Alfresco 企業發展副總裁 Matt Asay 提出他們的例子，Alfresco 的開放文件管理解決方案，正是商業與技術經理的共同推動成果。
