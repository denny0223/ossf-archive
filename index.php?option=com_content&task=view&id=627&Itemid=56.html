自由軟體產業推動策略會議之後

Created at Friday, 05 December 2003 08:00      Last Updated on Thursday, 21 July 2011 17:40 Written by 涂敏怡

從社群發展角度看台灣人才培育及產業與社群的互動

2003 年 11 月 19-20 日，由經濟部工業局主辦，資策會資訊市場情報中心（MIC）執行的「自由軟體產業推動策略會議」在台北萬里舉行。本次會議的目的主要是請廠商及與會人員評 估預測自由軟體產業推動的五年計畫 （2003-2007）到 2007 年為止規模與組成，以及如何制定工作項目。


會中 MIC 一開始首先對針對產業產值分析、授權方式及其他國家（例如德國慕尼黑市政府、美國、中國、印度） 的產業政策等作出報告。並由與會者在接下來的一天半議程中，提出產業推估及建言。會議分嵌入式 （Embedded）應用系統及軟體服務兩組進行。

這場會議沒有開放媒體採訪，因此我們在這場會議結束後訪問了三位參與會議的人士，分別為中研院資訊所副研究員莊庭瑞、晶慧資訊總經理劉政、以及基智科技總 經理梁力基，請教他們參與這場會議後的心得與建議，同時訪問到本次會議的主辦單位資策會MIC的資深產業分析師童啟晟，對於與會者的建議做回應。

◎ 初步共識與具體結論

與會的廠商與工業局、MIC 等人員經過一天的討論，得到了一些積極與正面的結論，根據參與軟體服務議題討論的中研院資訊所副研究員莊庭瑞說明，會中初步結 論，軟體服務未來五年的產值初估可達 100 億，同時與會者均希望政府能認定軟體與服務的價值，重視人才培育，強化市場與商機的開闊。政府招標的方式應鼓勵自由軟體的開發與應用。另一位參與軟體服務 議題討論的基智科技總經理梁力基（梁楓）表示，該議程初步的結論是政府需要在加強軟體方面國際行銷，讓世界知道，台灣不只能夠發展硬體，也有十分強勁的軟 體發展能力。同時也需要培養更多的專門人才，否則國內的人才恐怕不足以滿足業界的需求。

參與另一個嵌入式（Embedded） 應用系統議題討論的晶慧資訊總經理劉政則表示，相較於軟體服務的 100 億產值，嵌入式應用組推估出的結論是產值約 1000 億，嵌入式應用議程同時做出了以下的結論：(1) 積極建立嵌入式自由軟體開發平台。它需要一個很方便開發的開發平台。 (2) 選定旗艦型開發計畫 (3) 積極參與國際標準組織 (4) 培養嵌入式自由軟體人力。

從以上的結論來看，兩個議程均認為自由軟體專門人才的培育是重要的，此外應該加強市場與商機的開拓，國際交流，以及建立嵌入式自由軟體的開發平台。然而對於自由軟體人才應如何培育以滿足業界需求，以及如何進行自由軟體開發以擴大市場的商機，因時間因素則未有進一步的討論。

◎ 促進軟體開發與人才培育

在嵌入式產業的部份，劉政則鼓吹國內應發展由國人自己維護或自己開發出來的作業系統，也就是發展出國人自己的自由軟體堆疊（Stack）， 例如國產 IC、本土版的 Linux等。劉政同時也提出了一個嵌入式系統發展平台的構想，他說這個平台可以因應各種應用之不同以二維方式整合。比方縱向是上下游 Alliance，可能有 AP、CPU、OS、Compiler、IC 等，而 AP 可能又有開發、debugging、porting 等，這是 project base；橫向則可能是各家廠商，各項都有多家廠商。開發這個平台的人應該要有這個能力去分辨並分門別類提供這些 special foundry，與橫向來作交叉組合。劉政認為發展一個開發平台是很重要的，有了這個開發平台，整個產業、社群才能有互動。

不 過，如何培育嵌入式系統開發平台的人才，以振興國內的嵌入式產業？梁力基認為國內的嵌入式產業相當成熟，已經走在政府的腳步前面，但是如何培育往後產業 界所需要的人才，卻是非政府不可。當務之急應該是分為完整的教育計劃，由校園體系（大學、大專）、補教體系（補習班、人才培育班）、其它體系（社區大學 等）規劃出完整的教育方式，培訓業界所需要的精英。他在此強調國內目前的教育，培養出來的跟業界所需的有相當大的差異。目前針對自由軟體人才培育方面著重 於「網管人員」，最多只能培訓出「Linux 的伺服器架設人員」甚至End User（一般使用者）；卻不能教出國人產業所急需的「程式開發人員」。政府應該針對目前產業所需的人才進行調查，並在校園、補教等體系中，做出較符合國 人企業所需人才；而不是僅盲目的開設人才培育班，造成國內的「資訊人才」很多。但因培養方式的失調，而造成各層面無法銜接，甚至發生這裡人才過剩，那裡卻 人才不足的窘況。

在國內，社群的發展與自由軟體人才的培育息息相關，政府若能重視人才培育，將可進一步促進自由軟體市場的擴大與活絡，以促成自由軟體整體的良性循環，社群 也才跟著有興盛的可能。對於政府的自由軟體人才培育政策，本次會議的主辦單位，資策會MIC資深產業分析師童啟晟表示，政府推動自由軟體的部會很多，經濟 部工業局僅是其中之一，重心放在產業推動的部分，不過工業局會聯合自由軟體產業上下游做整合的動作，因此包括數位內容、數位學習亦將是整合的重點。

童啟晟補充，在本次會議中他們亦針對在場人士舉行一項小型調查，詢問與會代表最需要哪一方面的人才，結果與會者認為最缺乏的是嵌入式系統的開發人才、以及 OS 開發工具的人才，因此 Open Source 的開發人才的培育可以說是目前產業界所重視的項目，他認為身為自由軟體產業推動計畫分項計畫之一的自由軟體鑄造場，本身亦具有培育人才的目標，未來可以朝 著這個目標努力發展。

◎ 增進產業與社群的互動

台灣目前自由軟體產業，莊庭瑞表示，政府與廠商的出發點還是以利用與應用著手，國內現行的思考點多偏向使用現成的自由軟體，以及客製化是否容易，經濟上的 互動較薄弱。其實廠商可以有中文化的 Linux、或是現成的自由軟體，這些都是社群投入許多時間精力的成果，都為產業或廠商帶來助益。在國外，社群與產業的互動就很密切，例如 Apache 的開發社群就是由 IBM 的開發者來開發的網路伺服器，Mozilla 的前身是 Netscape等，但是在國內這樣的互動較少。莊庭瑞同時表示，自由軟體產業如果也能加深本地人才的參與，對於產業的生根是很重要的，對於整體水準的提 升也會有所助益。自由軟體對於本地整合廠商是個很好的商機，已釋出的自由軟體元件授權，使得要整合出符合本地所需的客製系統變得容易。梁力基也認為社群能 夠激發各項專業能力的交流。

對於扶植自由軟體社群的實際做法，梁力基認為非常簡單，「不要去重新打造已經有的輪子」。國內的社群其實動能很強，也有許多架構已經相當完整的專案，如自 由軟體總藍圖（各類於 Linux 相對應於 MS 的解決方案訊息）、ask-linux（自助式的 Linux 知識管理網站）、自由軟體鑄造場（提供國內社群各類專案的文件、程式碼集散地）、 以及各種 Linux 資訊傳播管道。只要能以政府的資源甚至業界的資源，讓這些原本由國內社群所自發性的熱情組織，可以更無後顧之憂地發展（如必要的經費、網站頻寬的供應等 等）及善用這些專案所帶來的資訊及便利性。比起政府一直花費大筆的經費；卻無法做到符合台灣目前環境的方式來得更有經濟效益。這也是扶植台灣社群的其中一 種最好的方式。

◎ 與會感想

參加完本次自由軟體產業推動策略會議，莊庭瑞認為自由軟體的推動目前侷限於自由軟體的「產業」或 Linux，其實其推動未必立即遷涉到產業面。舉例來說報稅系統只能用於 Windows，如果能是 Web-based，即使尚未成為自由軟體，也已經能對自由軟體的相關系統已經有鼓勵作用；另外對整理出來的中文字碼能善加利用，也是一種鼓勵。在教育方 面，可以發展自由軟體課程教學，或是國科會補助以自由軟體的方式釋出。政府招標文件也可以自由軟體授權釋出。如此，自由軟體自然而然就能茁壯。

梁 力基參加完本次會議後認為，若可以凝聚業界與政府的共識，讓大家能看重國內社群的力量，比如人才的素質、能力，專案的開發、資源、組織的能量，對自由軟 體社群的社態會有很大的助益。而相對的，國內社群所產生的效益，最獲得其利益的，當然也是國內產業界。他對於本次會議也有兩個期許：希望政府能針對國內的 人材培訓也來一 個正式的策略會議；同時希望政府能夠多了解國內的自由軟體到底做了些什麼，他們需要什麼？政府有哪些事其實只是在做社群以經做的東西，而且通常社群所發展 出來的，能比政府花下大筆經費的專案來得更好，影響更深遠。

劉政對於本次會議嵌入式應用系統的結論感到滿意，但他也另外指出，從 11 月 6-7 日的阿里山會議 [1] [2] [3] 到這次在萬里召開的自由軟體產業推動策略會議，兩個會議的與會者均具有高度共識要推動自由軟體產業，然而這兩個會議的結論是否具有連續性與一致性，以及受 邀與會者是否具有普遍性與公平性，他認為這是日後政府在舉辦類似會議時，所需要重視的項目。童啟晟回應，這兩次會議的召開是兩件不同的事情，MIC 是算是工業局的幕僚單位，本次會議結果將提供給政府作為參考，阿里山會議由台北市電腦公會指導，討論的部分較偏向由業者廠商執行的方案，兩者的出發點是不 同的，因此無法談論一致性的問題。

關 於邀請與會者的代表性，童啟晟說這點在本次會議中確實引起了不小的爭執，因為本次會議主辦單位經濟部工業局的調性比較偏向產業的推動，因此邀請的多半是 廠商，此外亦邀請了中研院的何建明研究員，並由莊庭瑞研究員代為出席。如果想要邀請除了廠商以外的各界代表與會，應該要由更高層級的單位（如行政院國家資 訊通信發展推動小組NICI）舉辦類似的會議時來邀請較為適合。

綜 觀三位各自代表不同領域的受訪專家（與會者）的會後感想，意見或有不同，然對於舉辦這樣一次會議，除了看出政府的積極作為外，其必要性應該已受到與會者 一致認同，且與會者也一致提出人才培育的重要性。而政府的後續政策應如何制定、如何提升自由軟體社群與產業的互動程度，將更受到與會者的期盼與關注。

（編後語：本次的訪問，受訪人士的意見主要呈現了本土社群的觀點，其實還有許多產業代表參與此次的會議，在截稿前不及採訪。因此我們歡迎關心此次會議的各 界人士，在閱讀完本文後，能到 OpenFoundry 電子報網站 https://www.openfoundry.org/archives/cat_19epaper.html 發表您的意見與看法，與我們做更進一步的討論與交流。）
