調查報告顯示開放源碼主導雲端運算領域

建立日期 2011-06-14 00:00      最近更新在 2011-06-28 14:16 作者是 謝良奇／編譯

一份由多家開放源碼雲端廠商釋出的調查報告顯示，開放源碼在雲端運算中的普及與優勢地位。這份詳盡的調查報告是由開放源碼雲端運算軟體供應商 Cloud.com 及其合作夥伴 BitNami、Zenoss 共同釋出，其目的是為了找出採納雲端技術的關鍵 IT 目標與阻礙因素。報告指出僅有 3% 的雲端使用者完全不採用開放源碼軟體，反之，有 69% 的使用者表示會儘可能地使用開放源碼軟體。

這份報告乃是針對 BitNami、CloudStack、Zenoss Core，上述各家公司中的知名開放源碼專案，其周圍的社群成員進行調查。調查對象包括超過 500 位 IT 專業人士，其中 40% 是來自政府、教育、商業機構的技術長、架構師和 IT 經理人。

Cloud.com 社群副總裁 Mark Hinkle 表示，雲端運算正獲得前所未有的關注，然而和其他位於發展初期的技術一樣，產業使用者與解決方案供應者仍在試圖掌握雲端運算帶來的優點和部署選項。這份報告中的發現能有助辨識組織的需要，以及他們對於虛擬基礎架構的部署偏好，好讓該公司能發展出最能符合雲端運算用戶需求的解決方案。

該報告揭示了開放源碼在雲端運算上的廣泛運用，有 69% 的雲端運算用戶表示他們會儘可能地使用開放源碼軟體，僅有 3% 的用戶完全不使用這類軟體。調查對象中的所有政府單位或多或少都使用了開放源碼軟體。有趣的是，在不使用開放源碼軟體的使用者中，有 58% 沒有任何雲端運算策略。

受訪的 IT 專業人士中，有 83% 表示他們計畫在雲端部署 Linux 作為 guest 作業系統，使得 Linux 成為雲端上主要的作業系統。也有 66% 的受訪者表示會部署 Microsoft 的 Windows。

除了揭示開放源碼在雲端運算中所扮演的角色，這份報告也顯示 61% 的組織正就雲端運算進行資訊收集與規劃階段，或者已經通過雲端運算策略但尚未實作。20% 的組織則已經有雲端運算實作，有 20% 目前沒有雲端運算的計畫。

高達 68% 幅度的硬體成本節省的比率，成為採用雲端運算的第一大理由，其次則是加速基礎架構部署和降低系統管理負擔。在受訪的技術長中，延展性是採用雲端的最普遍原因，隨之在後的則是根據資源需求變動而進行調整的彈性需要。

此外，該報告也指出，70% 的資料中心管理者選擇在專用資源上部署基礎架構，僅有 12% 傾向部署在公有雲 (public cloud) 上。
