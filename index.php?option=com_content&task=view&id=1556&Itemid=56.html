昇陽推出開放源碼儲存技術工具與擴展服務

建立日期 2008-05-01 08:00      作者是 謝良奇/編譯
昇陽 (Sun) 宣佈推出一系列新的開發者工具，並擴展專業服務，以協助開發者為基於 Solaris 作業系統上的開放源碼儲存軟體，發展相關應用。免費為開放源碼開發者，提供其儲存技術，昇陽有意藉著開放源碼推動儲存技術走向更為開放的局面，試圖增加自己在儲存市場的佔有率。 開放源碼軟體允許終端使用者、製造商與應用裝置設計者，以私有軟體無法提供的方式，開發新的產品。昇陽過去幾年內陸續將 Solaris、Java Enterprise System 軟體堆疊、Java 以及各種儲存相關軟體，加以開放源碼。

日前昇陽自豪地表示其 OpenSolaris 可做為全球首套開放儲存平台核心。暫且不論過去 Linux 與各種 BSD Unix 系統上皆有開發者投入儲存技術，昇陽透過 NFS (Network File System)、ZFS (Solaris Zettabyte File System)，的確在儲存技術上做出不少貢獻。

如今昇陽的開放儲存社群，根據昇陽的說法，已經有超過 3 千個會員，以及超過 30 個專案。昇陽的儲存服務產品主任 Graham Lovell 表示，OpenSolaris 擁有相當強勁的成長動能，社群看出其中蘊含一套商業模式。

這套商業模式，以昇陽的眼光看來，有如開放源碼過去促成便宜的伺服器環境，開放源碼技術同樣能提供價格更為低廉，成本效率更高的儲存基礎架構。Lovell 認為，未來一年中，儲存技術的經濟面會有相當大的改變，獲益來源將因為開放源碼而有所轉變。

昇陽此次推出的開發者工具與專業服務，可供開發者在 OpenSolaris 平台上，運用產業標準伺服器建構低成本的儲存裝置。昇陽為此特別將 OpenSolaris 為儲存伺服器加以最佳化，有如微軟將 Windows 修改後推出 Windows Storage Server，做為低價儲存裝置用作業系統。
Lovell 表示，解決方案供應商或其用戶可下載 OpenSolaris 的儲存系統版本，並安裝於各種伺服器上，在 10 分鐘內即可完成 NAS 應用裝置的架設。他說，軟體完全不須成本，當然能夠購買 Sun x86 伺服器是最好的，不過用舊的伺服器也可以。下載軟體後，用戶只需依照他們提供的步驟，以 NAS 來說，安裝完成加以執行只需 10 分鐘，從私有儲存技術轉到開放儲存技術一點都不困難。

Lovell 表示從私有儲存技術到開放儲存技術，對於協助企業在合理成本下快速提高儲存容量，是相當重要的一步。他認為，有許多企業在儲存技術上需要不同的經濟規模，以支援其企業模式。這些企業越來越需要有如 Amazon 或 Google 同樣經濟的儲存基礎架構，這表示每 1 gigabyte 要少於 1 美元。

Lovell 指出，EMC 與 NetApp 這些傳統儲存廠商將儲存應用裝置以一般標準組裝的 x86 伺服器硬體為基礎，但價格卻比標準組裝伺服器高出許多，雖然都是標準化的硬體元件，不過用戶還是必須出高價購買。他認為，透過開放源碼軟體與標準元件，可以為用戶節省 90% 的成本，這對於資料中心的爆炸性擴充有相當大幫助。

昇陽這次提供的新工具包括線上教學指南與影片，用以協助運用 Solaris 做為儲存平台。此外，新工具軟體可供開發者建構應用軟體，直接與 OpenSolaris 的儲存技術如 ZFS、NFS、CIFS (Common Internet File System)、COMSTAR (Common Multiprotocol SCSI Target) 等溝通。目前例如 DigiTar 與 Nexenta 等公司，皆以 Solaris 10 和 ZFS 為基礎，建構儲存陣列 (storage arrays)，允許 JBOD 裝置轉換為先進的 RAID Z 陣列，並具有資料保護、先進檔案管理、資料複製 (replication)、快照 (snapshotting)、分群 (clustering) 與其他功能。

此外，為協助加速開放儲存應用軟體發展，昇陽亦擴展了專業服務，幫助用戶轉換至開放儲存基礎架構上。昇陽會協助用戶開發開放儲存架構設計，並執行自私有系統到 OpenSolaris 儲存環境的資料轉移服務。

然而，昇陽針對儲存技術提出的開放源碼策略，專家倒是不怎麼買帳。部份專家認為昇陽的策略只是為了拉攏更多生意的行銷手法，其他人則表示此舉將有賴於企業對開放源碼儲存有可觀的經濟規模需要。

Forrester Research 分析師 Andrew Reichman 表示，由於儲存產品買家多採保守心態，開放源碼的空間並不大。像 EMC 這類的領導廠商仍然佔有優勢。由於儲存產品的決策對企業影響重大，用戶較願意購買過去使用過的解決方案。

盡管如此 Lovell 仍相信許多新創公司使用此類技術，以及用戶逐漸熟悉開放源碼環境後，將會推動開放源碼的接受度。除此之外，圍繞在此開放源碼儲存平台上，還有昇陽方面對此策略的授權決策。由於 OpenSolaris 目前採用 CDDL 授權，外界相信一旦 Solaris 採用的是 對開放性有更高要求的 GNU 授權，來自產業界的回應將有所不同。

今年 2 月昇陽剛剛把 Sun StorageTek 5800 System 原始碼貢獻給 OpenSolaris 儲存與  Java.net 社群。至今昇陽已將多種儲存相關軟體加以開放源碼，如 Solaris ZFS 檔案系統、IETF NFS 協定第 2 至第  4 版的伺服器與用戶端實作、YANFS 檔案存取協定等等。

Pund-IT 分析師 Charles King 認為，昇陽的最新工具可說是另一次促使更多開發動能，並吸引開發者目光的動作。昇陽的目標希望讓 OpenSolaris 開發更為簡易，藉此滿足開發者需求。
