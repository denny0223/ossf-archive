Oracle 放棄 OpenOffice 商業版本 LibreOffice 重申發展方向

建立日期 2011-05-05 00:00      最近更新在 2011-05-09 16:52 作者是 謝良奇／編譯
僅僅幾個月前 Oracle 才清楚表達其控制 OpenOffice.org 的意圖，並導致社群人士出走成立文件基金會 (Document Foundation) 與建立其分支專案 LibreOffice。日前 Oracle 突宣佈不再銷售 OpenOffice 生產力套件的商業版本，並且要將 OpenOffice.org 轉變為純然以社群為基礎的開放源碼專案。如此一來不禁引發外界揣測，兩個專案是否要重新合併？

文件基金會不久後發表聲明平息了外界的揣測，在簡短但堅決的聲明中，Charles-H. Schulz 表示該基金會將按計畫持續進行。文件基金會是一個獨立、自治的功績式基金會，由 OpenOffice.org 的主導成員所成立。或許是作為對 Oracle 的示意，Schulz 說他們很願意接納新的成員與夥伴。

聲明中也指出，文件基金會將持續建構於 OpenOffice.org 社群 10 年來致力完成的基礎上，這個基金會的成立來自一個信念，那就是出自於獨立基金會的文化，不僅最有利於貢獻者，也能為市場帶來最佳的軟體。

代表該基金會指導委員會 (Steering Committee)，Schulz 表示，該基金會已經為 OpenOffice.org 社群與源碼庫安排了未來的發展途徑，此一未來發展正如該基金會於 2010 年 9 月成立之時所公佈的一樣。

自從 OpenOffice.org 社群出走至今，分支專案 LibreOffice 在此期間發展迅速，獲得來自 Canonical 與紅帽 (Red Hat) 的青睞與支持，分別將其納入各自 Linux 散佈套件中。就在 Oracle 發佈消息數小時之後，文件基金會釋出了 LibreOffice 3.4 Beta 1。接著在 1 週後，文件基金會跟著釋出 LibreOffice 3.4 Beta 2。

在 Oracle 企業架構長 Edward Screven 宣佈 OpenOffice.org 將成為純然以社群為基礎的開放源碼專案，並不再提供 OpenOffice 商業版本後，Oracle 並未就 OpenOffice.org 源碼庫的著作權或商標，給予詳細的計畫說明。

Screven 表示，鑑於自由生產力應用軟體的廣泛利益，以及個人計算技術的快速演進，他們相信 OpenOffice.org 最好能專注於非商業基礎上，服務廣大用戶的組織來管理。他們希望立即與社群成員合作促進 OpenOffice 的持續成功。Screven 說，Oracle 將持續支持採用以開放標準為基礎的文件格式，例如開放文件格式 (Open Document Format)。

此外，Screven 也表明 Oracle 對 Linux 和 MySQL 等其他開放源碼技術的承諾。他說，Oracle 之所以專注於 Linux 與 MySQL，是因為這些產品已經在商業與政府客戶中，贏得了廣泛基礎的接納。事實上，Oracle 的決定顯示出該公司購併 Sun Microsystem 後，在銷售 OpenOffice 授權上並未獲得太大進展。該公司網站上若干 OpenOffice 相關連結已經失效。

儘管 Oracle 並未言明，該公司近期發佈的 Cloud Office 產品，其未來顯然也隨之出現問號。有關 Cloud Office 的網站連結也已經失效。

雖然 Oracle 宣告了對開放源碼產品的支持，許多開放源碼支持者對該公司卻是抱持不信任的態度，特別是在該公司於 2009 年購併 Sun Microsystems 之後。該次購併為 Oracle 帶來了知名的開放源碼資料庫 MySQL，以及包括 OpenSolaris 在內的其他開放源碼專案。

Oracle 雖然取得了 OpenOffice，許多人對於該公司是否有意貢獻開放源碼社群持存疑態度，這些顧慮使得一群 OpenOffice 開發者選擇離開，在相同的程式碼基礎上成立獨立的辦公室套件 LibreOffice。隨著 Google、Novell、紅帽、Canonical、開放源碼促進會 (Open Source Initiative，OSI) 表態支持 LibreOffice，OpenOffice.org 過往的風光不再。

這整個故事發展反映出一個事實，那就是具有適應力的自由與開放源碼軟體，甚至能抵抗最頑強的商業公司。如今在社群一致地走向 LibreOffice 後，Oracle 的宣告或許已經太遲。例如，Canonical 發言人 Gerry Carr 就表示，儘管其軟體庫仍提供 OpenOffice，在可預見的未來 Ubuntu 將持續以 LibreOffice 作為預設套件。

不論動機為何，外界仍然樂見 Oracle 將 OpenOffice 移交給社群，不過既然有了 LibreOffice，OpenOffice 是否還有容身之處，值得後續觀察。其實，不論 Oracle 是否移交，社群早已經擁有了 OpenOffice。

鑑於 Oracle 在繼承自 Sun 的開放源碼專案上連連犯錯失誤，包括過去的 OpenSolaris、Hudson，和現在的 OpenOffice.org，不禁讓外界好奇該公司接下來是不是會再次搞砸購併 Sun 取得的最重要資產 Java。畢竟 Oracle 到目前為止的成績，例如去年 Apache 軟體基金會 (Apache Software Foundation) 退出 Java SE/EE 執行委員會 (Executive Committee)，以及透過控告 Google 而為 Java 建立嚴格的保護主義式智財權立場，對於 Oracle 的社群關係並無多少幫助。
