MeeGo 與 Limo 合併為新作業系統 Tizen 獲 Intel、Samsung 支持

建立日期 2011-10-17 10:11      最近更新在 2011-10-25 11:04 作者是 謝良奇／編譯

Linux 基金會 (Linux Foundation) 與 Limo 基金會 (Limo Foundation) 正在重整對抗 Apple 和 Android 陣營的計畫，他們把 MeeGo 與 Limo 合併成新作業系統 Tizen，這已獲得 Intel 與 Samsung 的支持。Tizen 是以 Linux 為基礎的開放源碼行動作業系統，平台的應用軟體堆疊與第三方開發者框架將根基於標準的 Web 技術。Tizen 以 Web 為中心，是針對智慧型手機、平板電腦、智慧電視、小筆電、車載資訊娛樂系統的作業系統。由 Linux 基金會主導，計畫於 2012 年第一季釋出第一個版本，第一部裝置則於 2012 年中上市。

此一新平台將取代失敗的 MeeGo 專案。MeeGo 是去年推出的開放源碼行動作業系統，由 Intel、Nokia 在 Linux 基金會協助下，整合各自的行動 Linux 平台。當 Nokia 選擇放棄 Linux 轉向 Microsoft 的 Windows Phone 7 時，MeeGo 便走向分崩離析的局面。

現今多數行動作業系統著重於原生應用軟體的執行，而 Tizen 的目標則是開發可在瀏覽器上輕易執行的應用軟體。Intel 開放源碼技術中心主任 Imad Sousou 指出，未來這類以 HTML5 為基礎的應用程式會變得很常見。

HTML5 將在此新作業系統中扮演關鍵角色，這也是為何需要一套新作業系統，而非單純升級現有系統。Sousou 表示，轉向 HTML5 不僅是在現有的 Linux 上增加 web 執行環境，也意味著隱藏在 HTML5 程式開發者之外的 API 可以保持彈性，隨平台技術演進並依市場區隔而有所不同。

除了 HTML5，Tizen 將整合給應用程式批發聯盟 (Wholesale Applications Community, WAC) 的 web 開發環境支援。WAC 是一個由營運商支持的計畫，該計畫的訴求是讓開發者能夠在單一平台開發、散佈行動應用軟體，這些軟體將以 web 為基礎，且可在眾多手機上執行。該計畫也謀求讓營運商能在應用軟體商店市場中佔有一席之地。以 WAC 為基礎的應用軟體將於電信業者營運的商店銷售。

Sousou 指出，在未來幾個月裡，Intel 將致力確保 MeeGo 用戶能夠順利轉進 Tizen，並更進一步把 MeeGo 開發者也導入 Tizen。

然而，Tizen 究竟是一個全新的開始，或者只是為了減低 Nokia 離去的衝擊，而對 MeeGo 進行的品牌重塑，目前仍未完全明朗。Linux 基金會執行董事 Jim Zemlin 指出，MeeGo 現有的部分技術將由 Tizen 接手採用。Zemlin 還表示 MeeGo 將在有限範圍內，與 Tizen 持續並存。

Tizen 以 HTML 主的策略在概念上類似於 HP 的 webOS 平台，與 Mozilla 實驗性的 Boot to Gecko 專案。儘管相當受人注目，但是否能夠打造出具競爭力的主流行動平台，仍有待觀察。

自從 Nokia 於 2 月時選擇 Windows Phone 而放棄 MeeGo，Intel 便缺少主要的硬體夥伴，如今在 Tizen 上該公司有了 Samsung 的加入。根據 Tizen 官方網站，Intel 與 Samsung 將帶領 Tizen 的技術指導團隊。Intel 在 MeeGo 上差強人意的表現，顯示該晶片製造商不足以開發出優秀的使用者互動軟體 (user-facing software)，鑑於 Samsung 製造消費者電子產品的記錄，該公司的加入將為 Tizen 專案增添不少可信度。

Canalys 分析師 Pete Cunningham 表示，隨著 Android 主導者 Google 購併 Motorola Mobility、Nokia 與 Microsoft 結盟，Samsung 不得不找尋除了自有的 Bada 作業系統之外，其他可能的替代方案。

Zemlin 表示，Tizen 將採取開放管理模式，並與 MeeGo 一樣，和上游開發者緊密合作。在廠商中立的環境中，開發開放平台儘管看似大有可為，卻導致現有局面的一連串的錯誤與失望，這將是 Tizen 的主要障礙之一。

此外，在 Windows Phone 和 Research In Motion 環伺下，要與 Apple 和 Android 陣營對抗，看起來是一個不可能的任務，MeeGo 與 Limo 都宣告失敗，許多人認為 Tizen 這次也不會成功。而且Tizen 在使用者堆疊上從頭開始，並轉換到新的開發者工具，拋棄了現有 MeeGo 的社群與眾多開放源碼開發者，要引導這些人再次轉向 Tizen，恐怕是一大挑戰。

另外一則有趣的消息則來自 Nokia：Nokia 正在為其 S40 手機開發建基於 Linux 的替代品，名為 Meltemi。4 月時此代號首見於 Nokia 的一份內部通訊上，儘管當時外界猜測是跟 Windows 有關的專案，但顯然並非如此。
