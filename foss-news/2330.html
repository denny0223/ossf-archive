微軟簽署 Joomla! 貢獻者協議

Created at Thursday, 29 April 2010 08:00      Last Updated on Monday, 07 June 2010 22:38 Written by 謝良奇/編譯
本週微軟簽署了 Joomla! 貢獻者協議 (Joomla! Contributor Agreement，JCA) 並提交了部份程式碼，這表示該公司成為了該 GPL 專案貢獻者的一員。事實上，在即將釋出的 Joomla! 1.6 版本中，已經可見來自於微軟的程式碼。接下來將可見到微軟與 Joomla! 更緊密的合作。

Joomla! 是內容管理系統 (content management system，CMS) 也是一套 MVC 框架。採用 GPL 第 2 版 或更新版本授權，Joomla! 是全球第 2 大的 PHP 應用軟體。根據 Joomla! 網站，其用戶包括 MTV Networks Quizilla、IHOP、Harvard University、Citibank。

微軟簽署 Joolma!貢獻者協議，是由 Open Source Matters 發佈的消息。Open Source Matters 是 Joomla! 背後的非營利組織。微軟和 Joolma! 之間的協同合作關係，成果已可見於程式碼基礎之中。Joomla! 將因此獲得在微軟 IIS 上執行的最佳化，而且將可透過 Web Platform Installer 散佈。

Open Source Matters 總裁 Ryan Ozimek 表示，微軟簽署 JCA 對 Joomla! 社群而言是一件大事。不僅僅因為該公司是同意該貢獻者協議條款且向其貢獻程式碼的最大組織，同時這也意味Joomla! 用戶將可在微軟 IIS 上，體驗到更好的效能與可用性。Ozimek 補充說，該社群一直以來就有部份用戶採用 IIS 平台，但是他們提供的 IIS 支援與整合卻不夠理想。藉由新的協議，Joomla! 將不只能提供與 IIS 更緊密的結合，該社群也能夠與微軟更加合作做出改進，或解決未來的工程問題。他們很樂見微軟成為其 GPL  開發社群的一部分。

最近，似乎開放源碼所在之處，都可以見到微軟的蹤跡。本月稍早之時，該公司曾派人參與 DrupalCon 討論有關 SQL Server Driver for PHP 的釋出事項。上個月微軟的代表也曾在開放源碼商業大會 (Open Source Business Conference) 上討論將開放源碼納入其體系中的重要性，並且宣佈開放源碼的微軟生物基金會 (Microsoft Biology Foundation) 計畫。

有趣的是，在此同時微軟表示設計用以執行 Android 的手機侵犯了該公司的專利。熟悉微軟對 Linux 敵意態度的人，對此應該不會感到意外。例如微軟貢獻程式碼給 Linux 核心，但同時也控訴 Linux 違反了微軟的專利。

微軟的架構顧問Josh Holmes 在部落格上，表達了對該公司貢獻程式碼給 Joomla!　的興奮。他表示，顯然地這意味微軟的員工可以貢獻給 Joomla! 了。對他而言，這代表著微軟法律部門批准了撰寫 GPL 授權的程式碼。此舉清楚地展示了微軟對開放源碼專案的投入程度。

在　Joomla! 部落格上，儘管關於微軟簽署協議一事的文章中洋溢興奮之情，但也間接提到了微軟和社群尖的問題。文章中表示，此事對於　Joomla!　和社群的意義為何？當然，他們永遠歡迎新的程式碼貢獻者，特別是當他們可以帶來新的技術。從　Joomla! 應用的角度看來，支援　IIS　意味著打開新的應用領域，其中包括許多企業。此外，他們樂於見到微軟成為 GPL 的貢獻者。如果這個協議可以成為微軟和自由與開放源碼軟體世界建設性關係的開端，那不是再好不過了嗎？
