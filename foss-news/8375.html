OpenOffice.org 進入 Apache 育成中心

建立日期 2011-06-23 00:00      最近更新在 2011-06-28 16:19 作者是 謝良奇／編譯

OpenOffice.org 已經被接受成為 Apache 育成中心 (Apache Incubator) 專案，即將以見習 (podling) 身分，也就是 Apache 育成階段中的專案，開始其發展。在 Apache 育成郵件列表上的公開公眾投票，不論是約定 (binding) 或非約定 (non-binding) 投票，都可見對接受該專案的多數支持。

Oracle 日前宣佈將 OpenOffice.org 轉移給 Apache 軟體基金會 (Apache Software Foundation)，此舉除將該專案置於 Apache 育成計畫之下，更將轉換該專案的原始程式碼為許可式 (permissive) 的 Apache 授權。

OpenOffice.org 在 Oracle 購併 Sun 之後落入該公司手中。在 Oracle 未能解決該社群成員長久以來對於 Sun 管理不當的不滿後，社群的重要成員轉而成立文件基金會 (Document Foundation) 這個獨立於廠商之外的組織，並開發 OpenOffice.org 的分支 LibreOffice。

在投下贊成票的人當中，引人注意的包括前 Sun 的開放源碼長也是開放源碼促進會 (Open Source Initiative) 的現任董事會成員的 Simon Phipps。他對於 Oracle 貢獻 OpenOffice.org 給 Apache 基金會一事採取批評態度，因為他覺得此舉會製造分裂並且忽視 LibreOffice 社群與文件基金會。

至於為何投下贊成票，Phipps 表示 OpenOffice.org 重構為包括開放文件格式 (Open Document Format) 匯入與匯出程式在內，辦公室套件元件參考實作的前景，使得他投下了贊成票。這樣的參考實作可以被除了 OpenOffice.org 和 LibreOffice 之外的其他專案所使用，協助開放文件格式更為廣泛地運用在開放源碼與私有軟體中。

Phipps 認為，假如從 Oracle 重新授權並交由 Apache 維護的程式碼中，能夠為每一種開放文件格式的子格式，建立可重用的開放文件格式編輯器參考實作，對於所有人來說其正面影響不可限量。他也希望該專案不要再使用 OpenOffice.org 名稱與 LibreOffice 競爭。

他表示，假如這個 Apache 見習專案成為了 LibreOffice 的競爭者，以 OpenOffice.org 的名稱嘗試與 LibreOffice 一爭高下，那麼這對軟體自由來說將是一個不良發展，到時候他會在他處強烈表示其反對意見。假如情況演變至此，他將會投票反對該專案進入上層 (top-level) 階段。

曾就此一專案貢獻提案，與自由軟體支持者進行熱烈論辯，IBM 的 Rob Weir 表示樂見此一投票結果。他在部落格發文表示，87 位開發者與 8 位 Apache 導師 (Apache mentor) 已經簽署了這項專案，強調這並非針對數量有限的開發者所進行的競爭，並認為反對此案的少數人士犯了資源短缺的謬誤 (the fallacy of scarcity)。

Weir 表示，開放文件標準能夠且應該，成為競爭應用軟體套件之間相互操作性的基礎。

Apache OpenOffice 作為其他辦公室套件軟體參考實作的概念，似乎成為了批評者與支持者雙方面的共同基礎。至於有關 Apache OpenOffice 應該如何發展，以及其目標為何的爭議，如今將進入新成立的見習階段。

依照設計，Apache 流程要求見習專案擬定並針對目標、管理、所需程序制定文件，並且同時針對新近貢獻的程式碼重新開發，以便成為 Apache 上層專案邁進。
