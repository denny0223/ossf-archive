Dell 筆電的隨開即用 Linux 作業系統： Latitude-On

Created at Thursday, 01 October 2009 08:00      Written by 謝良奇/編譯
出自於 Dell 的新筆記型電腦 Latitude Z600，搭載了 Dell 的筆記型電腦輔助作業系統 Latitude-On 而成為話題。Latitude-On 作為基於 Linux 核心所建構的獨立作業系統，加上在單獨的 ARM 晶片上執行，能延展筆記型電腦的電池時間，從數小時延伸至數天。

Z600 搭載的 Latitude-On，可以讓使用者在不開機情況下，提供快速的 Web 電子郵件、連絡人、行事曆與其他網際網路內容與服務存取。根據 Dell 所作的調查，使用者有 70% 的時間會花在此一隨開即用的 Linux 環境中。

從 Latitude-On 過去的代號 Blacktop，可以看出該作業系統希望帶給筆記型電腦，和 BlackBerry 同樣快速的功能性，一年前外界曾經一窺 Latitude-On 的面貌，如今該作業系統隨著 Dell 的新 Latitude Z 產品問世。此外，Latitude-On 也提供作為 Latitude E4200 和 E4300 筆記型電腦的額外升級選項。

與過去預先開機 (pre-boot) 環境如 Splashtop 等不同，Latitude-On 是一套小型的系統晶片 (System on Chip) 電腦。Dell 則稱之為次處理器與次作業系統。這套作業系統是以 SUSE Linux Enterprise Desktop 10 為基礎，並且附有改自 Firefox 並加上客製化電子郵件、日記、連絡人管理的瀏覽器，這一切通通塞在一個快閃記憶體晶片中。

這一整套系統採用了 TI OMAP3430 ARM 處理器去載入 Linux 環境，藉此避免 Windows 緩慢的開機流程。Dell 表示之後將遵循 General Public License 的要求，釋出 Latitude-On 程式碼。

iSuppli 主分析師 Matthew Wilkins 表示，第一眼看來 Latitude-On 似乎代表了 ARM 微處理器與 Linux 作業系統聯手攻進 Wintel 主導的個人電腦市場，然而，事實上 Dell 只是將這套基於 TI OMAP 處理器的系統，運用在相當狹窄的用途上，也就是提供特定形態資訊的傳輸管道而已，並未取代 Intel 與微軟作業系統平台的關鍵功能。

TI 的 OMAP 處理器一般應用於手機和行動網際網路裝置 (mobile Internet devices) 上，並且稱霸獨立應用軟體處理器市場，根據 iSuppli 的資料，其第二季全球營收佔有率為 24%。

Wilkins 指出，Dell 的解決方案確實代表了 PC 市場，針對 iPhone、BlackBerry 等智慧型手機資訊存取方式的一種反擊。Latitude-On 允許使用者拿起尚未開機的筆記型電腦，迅速地上網查詢班機資訊、瀏覽重要電子郵件。Wilkins 表示，Dell 顯然在獲取重要用戶回饋意見後，實作出這樣一套快速開機的有限環境系統。然而，在對此做出成功與否的結論前，iSuppli 將會觀察用戶對此系統的反應，而 Intel 與微軟的回應也會是關注重點。

iSuppli 的報告指出，雖然短期內不可能出現以 ARM 為中心的 Windows 個人電腦，但是以 ARM 為中心的 Linux 電腦卻可能成真，對於那些在個人運算上尋求 Wintel 之外替代方案的使用者來說，這將會是一個具有吸引力的選項。

Latitude-On 提供了 Exchange 2003、Novel Groupwise、POP 和 IMPAP 伺服器的支援能力，其他的組織平台如 Exchange 2007，Dell 表示將於日後以韌體更新的方式加入支援。 Latitude-On 也內建一套透過 VPN 存取桌面應用軟體的 Citrix 用戶端。
