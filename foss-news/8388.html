經理人惹惱 Firefox 企業用戶 Mozilla 部落格發文平息怒火

Created at Tuesday, 05 July 2011 00:00      Last Updated on Tuesday, 12 July 2011 11:22 Written by 謝良奇／編譯

看來 Mozilla 經營公共關係的速度有點跟不上其快速釋出時程。經過媒體上一星期的混亂之後，Mozilla 終於在一篇部落格文章中，試圖平息其業務騷動並向企業用戶重新保證他們會認真看待其需求。這場騷動起因自一位 Mozilla 經理人在部落格上過於激烈的回應，引發 IT 專業人士的不滿。

Firefox 專家與顧問 Mike Kaply 日前的部落格文章，指出 Firefox 快速的釋出時程，對於企業用戶其實有很大的負面影響。最好的例子是 Firefox 4 今年 3 月才釋出，3 個月後又推出了 Firefox 5.0 穩定版，Mozilla 並且停止了對 Firefox 4 的支援。Kaply 指出如此快速的更新時程對一般用戶而言沒什麼問題，但在組織環境下，特別像是銀行這樣的地方，卻行不通。期望一家公司會每 6 個星期就跑一遍完整的瀏覽器部署流程簡直荒謬。有許多人同意他的觀點，特別是支援成千名使用 Firefox 用戶的 IT 人員。

不過，Mozilla 經理人 Asa Dotzler 對 Kaply 的回應才是惹惱企業用戶的主要導火線。Dotzler 表示，企業從來，也不應該，是他們關心的重點。除非有一天只剩下需要系統管理者或企業部署團隊照料的人存在，否則他不知道為何他們要把重心擺在 Kaply 如此重視的環境上。

顯然為了平息此事，Mozilla 產品副總裁 Jay Sullivan 在部落格中寫到，Mozilla 社群將工作重點放在個人用戶的需要上，並依此排定產品藍圖與功能的優先順序。不過，正和其他技術一樣，Firefox 忠實用戶與其 IT 部門也會試圖將 Firefox 帶進工作場合。

他指出，企業的關鍵挑戰在於必須在 Firefox 每次更新時，確保其網站、應用程式、附加元件運作無誤。透過社群中的積極討論，包括來自 IBM 的重要回饋，他們正在規劃能夠平衡此一需求的解決方案。開放源碼軟體非常適合因應這些挑戰，因為有關方面可以共同建置所需方案。

事實上，IT 管理人員因為安全缺失、缺乏支援與文件等等他們所認知的原因，對開放源碼軟體採取不信任的態度。儘管近年來 Firefox 與 Google Chrome 等開放源碼瀏覽器變得格外風行，卻仍然沒有辦法讓 IT 管理人員對開放源碼軟體帶有信心。

此一現象或許也要多少歸功於 Microsoft。儘管其 Internet Explorer 瀏覽器市占率持續流失給開放源碼瀏覽器，該公司並沒忘記要抓緊許多組織用戶，其關鍵就在於讓 IT 管理人士認為開放源碼瀏覽器是不安全的平台。

比起之前的 Firefox 4，Mozilla 最近的 Firefox 5 釋出版本受到的批評要少了許多，這兩個新版本的釋出速度也為 Mozilla 寫下新的一章。多年來 Mozilla 的瀏覽器採取緩慢的釋出流程，相較之下 Google Chrome 的重要更新則遠多於 Firefox。今年初 Mozilla 才宣佈要採取新的快速釋出流程。

然而並非所有人都樂見此一釋出速度，正如 InfoWorld 所報導，許多企業 IT 管理者認為 Firefox 5 的迅速釋出造成了困擾。報導中指出，許多組織 IT 管理人對於 Mozilla 決定每 6 星期推出新版 Firefox 的計畫感到不滿。其不滿的焦點圍繞在 Mozilla 春天時決定停止對 Firefox 4 的安全支援。

正當 Dotzler 的爭議言論惹惱企業用戶時，Google 倒是趁此機會拉攏此一用戶群，強調他們對商業用戶的承諾，Google Chrome for Business 產品經理 Glenn Wilson 就表示，他們一直在聽取管理人士的意見，也在規劃產品藍圖上借重其協助。

不久前 Mozilla 在 MozillaWiki 上勾勒了對 Firefox 的願景。雖然出了 Dotzler 的事件，但其願景內容除了用以描述該組織的新產品藍圖，聲稱其在轉變中的網路瀏覽空間中的重要性，以及透露未來目標之外，並未提及商業或企業等字眼。Sullivan 的部落格文章或許能讓 Mozilla 扭轉外界的論述並結束對 Firefox 的這一場怒火。
