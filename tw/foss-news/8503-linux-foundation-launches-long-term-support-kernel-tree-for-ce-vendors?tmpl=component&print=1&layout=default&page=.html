Linux 基金會宣佈針對消費性電子產品廠商推出長期支援的核心樹

建立日期 2011-11-07 10:40      最近更新在 2011-12-02 13:41 作者是 謝良奇／編譯

LinuxCon Europe 大會日前在布拉格舉行，會中 Linux 基金會 (Linux Foundation) 宣佈推出新核心樹的計畫，目的是為設備廠商提供長期支援。這項計畫名為 Long Term Support Initiative (LTSI)，將可降低為行動與嵌入式系統開發客製化 Linux 平台的成本。

 

LTSI 專案由 Linux 基金會消費性電子產品工作小組成立，將提供每年一次的 Linux 核心釋出，並針對這些釋出進行為期兩年的定期更新。該專案已經吸引若干主要硬體廠商，例如 Hitachi、LG、NEC、Panasonic、Sony、Samsung Electronics、Toshiba。LTSI 將由 Linux 基金會負責管理，協調來自產業參與者的協同開發。

LTSI 的主要目標是產出穩定的核心釋出，足以讓使用期限為 2 至 3 年的消費性電子產品安心採用。該專案的構想是提供消費性電子產品廠商一個上游環境，可以在持續時間更長的核心上協同合作，包括向後移植功能與錯誤修改，以降低個別廠商維護自有核心樹的開發負擔。

圖片來源：Davidblog，CC-BY-NC-ND 2.0 Unported

Linux 基金會執行董事 Jim Zemlin 表示，消費性電子產品產業已經運用 Linux 來降低成本與縮短上市時程。現在他們進行的下一步，是減少在維護客製化核心上的重複工作。LTSI 的重要性在於，能讓廠商共享一份針對消費性電子產品產業的通用核心，藉此增加 Linux 在該產業中的使用，並且提高該產業製造商對 Linux 進展的貢獻。

Linux 基金會行銷與開發者服務副總裁 Amanda McPherson 指出，LTSI 核心樹對於多數嵌入式系統、半導體廠商、軟體元件商、散佈商與其他供應商，都會是有用的基礎。

在帶領行動計畫上，Linux 基金會各有成敗的記錄。例如，與 Nokia、Intel 合作的 MeeGo 專案，因為 Nokia 在今年放棄該平台轉向 Microsoft 的 Windows Phone 7 而告失敗。

取代 MeeGo 專案的新 Linux 平台是 Tizen，其空降過程並未徵詢社群的意見，使得 MeeGo 的獨立貢獻者感覺權利被剝奪，這打擊了 Linux 基金會的信譽。然而 MeeGo 的失敗也使得 Linux 基金會開始對硬體廠商的實際需要做出回應。

從去年的 MeeGo 大會可以看出，許多硬體廠商對 MeeGo 堆疊的底層部份感到興趣，這些廠商希望提供自有的客製化使用者體驗，而非標準化的使用者空間環境。為了縮短上市時間並加速客製化嵌入式 Linux 平台，MeeGo 專案提供了一些工具，這些工具是廠商對 MeeGo 的興趣所在。

為了迎合這些需求，名為 Yocto 的 Linux 基金會專案於是成立。該專案提供了打造客製化嵌入式 Linux 平台所需的工具與基礎架構。和 MeeGo 與 Tizen 不同的是，Yocto 並非散佈套件，也沒有標準化上層堆疊的打算。

相較於失敗的 MeeGo，LTSI 與 Yocto 這類計畫能更務實得應付硬體廠商的技術需求。這些專案的規模並不宏大，也不設計堆疊上層，使得這些專案更加符合產業在產品區隔化的作法。
