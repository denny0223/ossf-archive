Zmanda 推出 Windows 環境的開放源碼備份回復解決方案

建立日期 2008-10-14 08:00      作者是 謝良奇/編譯
開放源碼備份與回復獨立軟體開發商 Zmanda，為該公司快速擴展中的開放源碼備份回復套件，加入兩個新成員。首先是 Zmanda Internet Backup (ZIB)，是針對 Windows 環境的網際網路備份解決方案，可將檔案備份至 Amazon 知名的 Simple Storage Service (S3) 上。另一項新產品是 Zmanda Recovery Manager 3.0 版本，新版本增加對 Windows 主要伺服器的支援。ZRM 3.0 Windows 解決方案成為 Zmanda MySQL 資料庫保護解決方案的下一代升級。

ZIB 為 IT 業界第一套 Windows 備份與回復解決方案，將 Windows 伺服器與 Microsoft SQL Server 或 Exchange Server 等執行中應用軟體，即時備份至 Amazon 高可靠、可延展且低成本的線上儲存服務 S3。對於需要 Windows 伺服器與應用軟體備份及災難回復的中小企業而言，ZIB 可說是兼顧成本考量與易用性的資料保護解決方案。

ZIB 主要針對 2 至 5 部 Windows 伺服器的環境。與 Zmanda 的 Amanda Enterprise  等產品不同，ZIB 是封閉原始碼產品。Zmanda 執行長 Chander Kant  表示，ZIB 與 Exchange 及 SQL 伺服器之間的整合功能，和 Amanda Enterprise 相同，然而成本更為低廉。以 50 GB 環境計算，ZIB 第一年支援費用為 300 美元，若為 250 GB，第一年則為 775 美元。

根據 IDC Link 2008 年 8 月的報告，Zmanda 是唯一與 Amazon Web Services 整合，提供 備份至 Amazon S3 的企業級備份應用軟體。企業可應用 Zmanda 備份軟體，透過網際網路，隨時輕易且安全地備份、保管與回復有價值的資料。

ZRM 3.0 則是 Zmanda 針對 MySQL 回復解決方案的首次 Windows 版本，運用了 Windows 的 VSS (Volume Shadow Copy Service) 快照，以及 ZRM 內建的時間點 (point-in-time) 回復功能，提供 Windows 上 MySQL 資料庫的即時熱備份。

ZRM 3.0 支援 Windows XP、Windows Server 2003 與 Vista 伺服器，此外，3.0 版也增加對 Debian Linux 與 Ubuntu Linux 的支援。

ZRM 3.0  用戶可在純 Windows 環境下執行 Zmanda Recovery Manager，透過 VSS 技術對線上 MySQL 資料庫進行即時熱備份。Zmanda Recovery Manager 為開放源碼的企業級備份與回復軟體，專門為協助 MySQL 資料庫管理者在人為失誤與系統或資料庫發生錯誤等事件發生時，預防資料損失造成的嚴重傷害。

ZRM 乃專為 MySQL 資料庫而設計，納入獨特的企業級功能如 MySQL 儲存引擎 (storage engine)、智慧快照 (snapshot intelligence)、副本支援 (replication support)，以及整合 MySQL 二進位日誌的備份作業等。ZRM 附帶的 Zmanda Management Console，是強大的 Web 圖形使用者介面，資料庫管理者可從任何支援瀏覽器的裝置上，控制所有 MySQL 伺服器的備份與回復作業。

Taneja Group 分析師 Eric Burgener 表示，由於 Zmanda 不支援多重主要伺服器的集中式管理，使得 ZIB 僅能侷限於中小企業用戶。但他也認為，該產品納入的許多功能與支援，或許能克服許多中小企業儲存管理者對開放源碼產品的不信賴。Burgener 認為，該公司產品所擁有的智慧型排程演算法，能在最適當情況下運用伺服器與網路資源進行備份排程，大公司的備份管理者需要備份與排程的控制權，然而小型企業不願意在備份基礎架構上耗費太多成本。

Zmanda 創辦人兼執行長 Chander Kant 表示，Zmanda 任務是確保用戶的資料不受到損傷，同時讓備份過程簡單、安全，成本也要低廉。而這次的新產品支援更多作業系統、備份更多應用軟體並運用新技術如雲端運算，使該公司向此一目標更進一步。
