法律專欄
獨樹一格的藝術條款：Artistic License 2.0
作者  葛冬梅
Artistic License 2.0（以下稱"Artistic 2.0"）這份條款並不像 GPL、LGPL、BSD 或 MIT 等條款如此地廣為人知，不過對於有在接觸 Perl 語言與相關軟體的開發者來說，Artistic 條款應該一點也不陌生，因為第 1 版的 Artistic（註一）就是由 Perl 基金會 (The Perl Foundation, TPF)（註二）所起草，並且適用於 Perl 程式語言與相關的軟體專案，不過由於第 1 版的規定有許多模糊不清的地方，並且不相容於應用普遍的 GPL，因此 Perl 基金會經過多年的討論與意見蒐集之後，於 2006 年正式發布 Artistic 2.0。

如同其他的自由軟體授權條款一樣，Artistic 2.0 可以讓使用者自由地執行、修改、重製與散布該軟體，不過 Artistic 2.0 並不像 BSD 這般的幾乎毫無拘束，也不像 GPL 或 LGPL 那般嚴格地拘束使用者的散布方式，Artistic 2.0 有著它獨樹一格的規定，這些獨樹一格的設計來自於編撰 Artistic 條款的基礎理想，那就是讓軟體專案的原始開發者可以用自由授權的方式來釋出專案，但同時讓後續的開發流程，也能夠達到「藝術控制 (artistic control)」（註三）的理想目的。

Broadcom 等四家公司加入 Linux 基金會
作者  謝良奇/編譯
Linux 普及率成長的同時，Linux 基金會 (Linux Foundation) 也隨之持續地增添新的成員。挾帶著 2010 年 Linux 基金會迅速地擴展成員名單的氣勢，2011 年剛開始，該基金會已經增加了 Broadcom、Timesys、Protecode、GoAhead Software 共四家新的成員企業。

2010 年 Linux 基金會增加了至少 23 個新成員，相比之下，2009 年僅有 8 個組織公開宣佈加入該基金會。如今 Linux 基金會約有 78 家成員企業。

Android 平板電腦多數未遵守 GPL
作者  謝良奇/編譯
近來眾家廠商相繼推出 Android 平板裝置，對於 Linux 來說似乎是件好事。然而這些平板電腦遵循 GPL 與否的問題，卻使 Android 平板市場籠罩上一層陰影。受雇於紅帽 (Red Hat) 的 Linux 核心工程師 Matthew Garrett，針對市面上 Android 平板裝置的 GPL 遵循程度，做了依次總體檢。結果顯示，許多推出 Android 平板的廠商並未切實遵守 GPL。

除了是一名 Linux 核心工程師外，Garrett 以檢視消費者裝置的 GPL 遵循性而聞名。這一次他公開了一份搭載 Linux 核心卻未遵守 GPL 提供原始程式碼的裝置清單。假如你手上的 Android 平板來自 Sylvania、Viewsonic、Smart Devices，或 Panda Pad 2、XPad，以及其他不知名廠牌的 Android 平板電腦，你很可能已經拿到了一部侵害 GPL 的平板裝置。

NASA 物件導向資料技術成為 Apache 頂層專案
作者  謝良奇/編譯
Apache 軟體基金會 (Apache Software Foundation) 擴展其開放源碼軟體專案的廣度，納入了由美國太空總署 (NASA) 開發的資料管理中介軟體套件。出自於 NASA 噴氣推進實驗室 (Jet Propulsion Laborator，JPL) 的 Apache 物件導向資料技術(Object-Oriented Data Technology，OODT)，日前已經成為了 Apache 的頂層專案。

OODT 專案副總裁 Chris Mattmann 表示，OODT 是一套開放源碼框架，可以被運用在多樣化的資料管理工作中。該專案可以用來設定移動資料至資料庫的工作流。Mattmann 指出，該軟體提供了建構資料庫、搬移資料庫、設定工作流以搬遷資料至資料庫，以及自資料庫中取出眾多不同內容的綜合性工具。

CPTN Holdings 申請案遭撤回
作者  謝良奇/編譯
日前傳出由微軟等四家公司組成的聯營企業 CPTN Holdings ，已經向德國監管機構撤回其申請案，此舉被不少人視為開放源碼支持者抗議後所發揮的影響力。然而，事實恐怕並非如此，CPTN Holdings 買下 Novell 專利的計畫恐怕不會因此中止。

去年 12 月微軟、Apple、EMC、Oracle 向德國監管機構聯邦競爭管理局 (Federal Cartel Office) 申請成立 CPTN Holdings，以購買 882 項 Novell 的專利，並引來開放源碼組織 (Open Source Initiative，OSI)、歐洲自由軟體基金會 (Free Software Foundation Europe，FSFE) 等開放源碼組織的抗議。日前此一申請案已被撤回。

Google 宣佈 – 擁抱開放格式 Chrome 揚棄 H.264
作者  陳瑞霖/編譯
Google 官方部落格中，有一則篇幅甚短但引來軒然大波的聲明文 (https://blog.chromium.org/2011/01/html-video-codec-support-in-chrome.html)，版主也罕見地針對大家的疑惑回文 (https://blog.chromium.org/2011/01/more-about-chrome-html-video-codec.html)，但仍然留下不少疑問。Google 宣佈對於 HTML 5 語法中 ＜video＞ 標籤，自家瀏覽器將不支援播放 H.264 影片。原本基於務實考量， H.264 和 WebM 兩種格式都支持，但現在Google 開始要完全支持開放標準 WebM 。Google 相信，以長遠的角度來看，採用開放格式這項創新所帶來的開放科技環境，將有助於整體網路環境，所以不得不捨棄受歡迎的 H.264 格式。
