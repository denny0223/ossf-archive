Hi Sonnenuhr,

簡要回覆您的問題如下，

1、USB Protector是以GPL-3.0授權的自由開源軟體，可以於企業內部合法使用，不過如果要再散布它、就必須主動的提供後手該程式的程式源碼。

2、Openfire與Spark都是以Apache-2.0授權的自由開源軟體，可以於企業內部合法使用，不過它的授權聲明裡特別表達程式裡的圖標與logo並不是Apache-2.0的授權範圍，其為Jive Software及INCORS GmbH所有，並採近似免費軟體(freeware)的授權方式釋出，原則上這些圖標與logo可以被Openfire與Spark程式的使用者所合法使用，但是並不允許被抽離出來另行利用、或是另行改作。

3、此外、Openfire與Spark所利用到的Java JRE並非自由開源軟體元件，其性質較屬於一般的免費軟體(freeware)與共享軟體(shareware)，其雖然亦容許被置於企業內部合法使用，但不得修改。更細部的說明，可以參照右列討論串的內容：https://www.openfoundry.org/tw/forum?func=view&catid=8&id=550#556

那麼、再將幾個重要的佐證資訊表列如下：

1、關於USB Protector

(1)其專案網址如右：https://www.openfoundry.org/of/projects/905

(2)該專案的主要開發者Ant，是之前亦服務於OSSF的合作夥伴，其對於自由開源軟體的開發很有理念與想法，個人profile可以參照右列連結頁面：https://www.openfoundry.org/community/userprofile/ant

2、關於Openfire與Spark：

(1)其檔案下載頁面如右：https://www.igniterealtime.org/downloads/index.jsp

(2)下載Openfire後，其授權說明頁面在檔案目錄下的路徑為：openfire/README.html，說明其為Apache-2.0授權的自由開源元件；下載spark後，其授權說明頁面在檔案目錄下的路徑為：Spark/documentation/README.html，說明其為Apache-2.0授權的自由開源元件。

(3)注意README.html亦同時說明其額外的logo授權方式如下，原則上就是這些圖標被容許隨著Openfire與Spark來被利用，但不得被另行抽離程式並進行編輯改作。

希望上述這些資訊對您有所幫助！如後續還有其他問題歡迎接續討論。

XML Notepad在Microsoft官方Codeplex網站的授權聲明頁面如右：xmlnotepad.codeplex.com/license

這裡顯示XML Notepad不但可能讓人免費下載，它還是一套Microsoft以Microsoft Public License(Ms-PL)向外釋出的開源軟體！

所以、XML Notepad是可以讓下載者以免付授權金的方式自由修改並自由散布的，使用目的也不禁止商業利用，也就是說、在企業內部安裝使用也是沒有問題的！

Ms-PL是一款近似BSD授權方式的開源軟體授權條款、更多的參考資訊可以參照葛冬梅撰寫的「來自封閉陣營的 Ms-PL 與 Ms-RL」專文：www.openfoundry.org/tw/legal-column-list/8263--ms-pl-ms-rl

Hi Victor,

依照您所提供的授權資訊，CCleaner與Defraggler這二套軟體都是Piriform這家公司所提供的免費軟體(Freeware)，

它的授權方式都是一樣，基本上有以下三大重點：

1、不負擔保，
2、權利人可隨時撤回該授權，
3、可在企業內部使用、但若員工有使用不當的行為，則企業主必須負連帶責任。

所以問題A與問題B的答案都是一樣的，那就是CCleaner與Defraggler都可以在企業內部使用，但使用上是毫無技術支援，如果需要它的技術支援，則可以找Piriform這家公司洽談商業支援版本的授權：www.piriform.com/business

那麼問題C、則比較有趣，事實上依照CCleaner的授權規定，這個軟體只能在Piriform網站與其他與他有合作關係的網站下載到，其他人若是沒有事先取得Piriform的書面同意，是沒有辦法自行散布這個軟體的，有可能軟體王這個網站有取得Piriform的書面授權，也有可能沒有取得，這個不可得知，但前提就是Piriform網站上的官方版本，本來就可以讓員工在辦公室環境下去安裝使用，只是若是沒有付費則不能得到Piriform的技術支援而已。

最後一個問題D，我認為Adobe這個授權聲明是要防堵伺服器取用Adobe Reader的功能來建置PDF文件轉檔機，那這個使用方式是逸脫它的預設的；如果是線上簽署Adobe Reader的企業內部散布聲明書，然後單純放到一台企業內部可以接觸到的網路硬碟，讓員工自行下載它來安裝，這便是比較符合Adobe公司這份聲明的預設的。

Distributor may install one copy of the Software on a computer file server within Distributor’s Intranet for the sole and exclusive purpose of allowing use of the Software from an unlimited number of client computers on Distributor’s Intranet via (a) the Network File System (NFS) for UNIX versions of the Software, (b) Windows Terminal Services, (c) Citrix, or (d) in the case of Adobe Reader, allowing printing within Distributor’s Intranet. Unless otherwise expressly permitted hereunder, no other server or network use of the Software is permitted. By way of example, the foregoing does not permit Distributor to serve the Software as an element of an Intranet or Internet hosted service.→此段顯示、如果是將Adobe Reader運用在網路傳輸上，Adobe Reader也僅容許列印方式的利用，也就是其他台電腦透過一個列印伺服器來列印PDF格式的檔案。

但是解釋上來說，MIS人員透過USB隨身碟為其他同仁安裝Adobe Reader，應該是符合此份條款「企業內部散布」的原則，基本上就是放置於公司內部一個網路硬碟進行倉儲，然後員工將其下載進行安裝，員工下載後進行安裝這個動作，並沒有規定「不允許員工為其他員工下載並進行安裝」，條款之所以沒有說明這樣的狀況，應為有意省略並精簡條文，所以您提到的MIS代其他同仁安裝的狀況，是被Adobe Reader使用者條款所容許的，如果可以的話、還是填寫[線上簽署AdobeReader散發合約]，預估一個內部可能安裝的機器台數，會更符合Adobe公司的授權政策。

而ERP-Server, Mail-Server上能不能裝Adobe Reader呢？如果您將Adobe Reader安裝在ERP-Server或Mail-Server的目的是要列印出PDF格式的文件，照上述條款的解釋，單純列印文件的需求、並且這個列印的服務僅供企業內網使用(Intranet)，是符合Adobe授權條款的設計的，但需要注意這樣的服務僅限於內部網路，關於Adobe對於Intranet的定義，則可參照下列這段文字：

“Intranet” means a secure internal website or server system that is accessible only to Distributor’s employees, contractors, or other persons granted access to Distributor’s internal networks in the furtherance of Distributor’s normal course of business.

大概是這樣，希望上述的資訊對您有所幫助！
