News!! OSSF2.0 is coming in late 2004!

建立日期 2004-11-01 10:00      最近更新在 2010-07-23 23:35 作者是 ossfepaper

What's new in 2.0?

1. Provide PROJECT_UNIX_NAME.openfoundry.org redirect service for projects with existing website. Don't allow project unix names as {www|mail|users|db|cvs|svn|lists|ftp|rt} (#4218)

2. Provide ability to invite more than one friend to join memberships. Enter emails separated by newline, semicolon, and comma. (#2236)

3. Allow project admin to set project members' rights in 6 areas: Basic, Release Plans, Forums, Tracker Fields, Help Wanted, and Statistics. Default setting: all members are read only.

4. New Release Tools: release management, auto roadmap generation, event management, calendar, Gantt for task and events.

5. Change files download policy. (#2235) Project admin can decide whether user needs to enter personal information before download or not. Moreover, project admin can choose the level of users' rights for downloading files. Implement as three levels, including "anybody", "registered", "member".

6. Provide Help Wanted for each project. Project admin can enter job listing to the system. User can browser job listing at each project's Basic page. User can also subscribe RSS of Help wanted for an interested project.

7. Provide project news editing tools. (#981) Project admin can add or modify project news, including release news, release notes, etc. Also, provide RSS feed of project news.

8. Provide Dynamic RSS links. (#986)

9. Enhance search ability.( #1884, #1885, #1183) Provide two types of search: simple search (on the top of all pages) and advance search. Simple search use google API and cover issue tracker, kwiki, sympa, user info, project info, site help documents. Advance search can be used for complicated requirement search.

10. Ontology browsing is added to this release.(#3196) Currently, we have three categories: embedded, chinese, and others.

11. Provide new repository service: Subversion. In other words, we support CVS, Subversion. If user has existing repository, they can use the original one or choose to mirror to rt.openfoundry.org.

12. Provide code commit notification service for project members.

13. Provide related citation and links for projects.

You will get the actual release date at Openfoundry.org
