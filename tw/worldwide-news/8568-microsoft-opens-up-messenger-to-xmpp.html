Microsoft 打開 Messenger 大門 開始支援 XMPP

建立日期 2011-12-26 10:19      作者是 謝良奇／編譯

Microsoft 日前宣佈，該公司旗下廣受歡迎的即時通訊服務 Messenger，如今可在 XMPP 協定下使用。這表示第三方通訊軟體與服務將可更輕易地與該服務連線。

之前 Microsoft 曾在 9 月舉行的 BUILD 會議上宣佈，Messenger 將可在 XMPP 協定上使用，並且曾提供開發者預覽。

XMPP 原名為 Jabber 協定，是一套廣泛應用於訊息交換之用的開放標準。Microsoft 已證實該公司正跟隨著競爭對手 Google 與 Facebook 的腳步，這兩家公司各自的 Google Talk 與 Facebook Chat 服務都已經支援 XMPP。

除了 XMPP，Messenger 服務也支援 OAuth 2.0。Microsoft Live Connect Platform 的首席計畫經理 Dare Obasanjo 表示，隨著 Messenger 的 XMPP 介面釋出，任何以 XMPP 為基礎，且支援 OAuth 2.0 認證的通訊軟體用戶軟體，將可連線至 Windows Live Messenger，確認朋友的線上狀態並與之即時交談。這意味著所有人都可以打造創新的傳訊軟體，並接觸到 Messenger 的 3 億活躍使用者。

根據 Microsoft 表示，Messenger 將支援核心協定 RFC6120、即時傳訊與在線狀態 RFC6121，以及針對電子名片 (vCards)、聊天狀態通知、延遲傳送的 XMPP 標準延伸。

宣佈這項消息後，Microsoft 也在其 Windows Live MSDN 網站為開發者提供了快速入門指引與程式碼範例。
