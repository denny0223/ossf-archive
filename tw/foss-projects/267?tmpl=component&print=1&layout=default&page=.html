SpaceChewingOSX 使用心得

建立日期 2007-01-23 19:35      最近更新在 2007-03-08 05:38 作者是 Mikeliu

我大約是在 2004 年 2 月買的 Apple iBook，鑑於 Apple 當初所附 的中文輸入法（漢音），十分彆腳而不順手，使用的快速鍵和使用方式與原本使用的 Windows 輸入法差異甚多，而且輸入速度與輸入法 反應也是十分的緩慢，所以就興起了是否有相關的替代方案的念頭。 一開始在使用者同好聊天的的 IRC channel 上，知道了 SpacingChewing 這個專案，也稍微瞭解它是從著名的 Unix-Like 系統上非常 常見的 GPL license 輸入法－酷音輸入法改寫而來，由 clkao 作一 開始的移植，有簡單而基本的智慧型選字功能，基本上跟 debian 或者是 freebsd 上面看到的酷音輸入法相比，完成度是算很高的。

不過後來 clkao 因故沒繼續維護輸入法，所幸在台灣 Open Source 圈很有名的 gugod 接手了這個專案，也對 SpaceChewing 作了功能 的增進與 bug 的修正，以往沒有 Installer 的幫忙安裝，得用「手 動」的方式安裝程式，也在 gugod 的努力下，在 SpaceChewing 0.6 版完成了以 Installer 安裝，對推廣 SpaceChewing 的角度來說，助益是非常大的。後來陸續有了 zonble 的幫忙，加上許多功能，例 如鍵盤配置的設定，還有更友善的設定介面（原生的 OSX UI 喔），還有更為美麗的選自視窗等等，算是完成度非常高的輸入法了。 我使用上的心得來說，當然是遠勝於原本 OSX 所附的漢音輸入法， 又因為程式簡潔的關係，輸入反應的速度非常的快，而且智慧選字的擊中率也非常高，一點也不輸 Windows 上面的商業輸入法，真的很 謝謝網路上的長輩加持，讓這輸入法更加方面而實用。後來 lukhnos 因為自己的使用上的需求，做出了香草輸入法框架出來，gugod 也陸 續將功能移植到香草輸入法上面，後來功能也幾乎是 100% 相同，然 後 gugod 與 lukhnos 更花非常大的心力利用 Objective-C 使用 COCOA framework，大幅度的改寫 UI 程式，對 OSX 的相容性就更高 了，也不會對一些軟體會有奇怪的相衝現象，所以滿建議讀者們可以 直接使用香草輸入法裡面的酷音，不僅功能 100% 相同，會有對 OSX 的環境有更好的相容性。
