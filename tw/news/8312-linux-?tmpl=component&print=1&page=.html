Linux 基金會宣佈成立高可用性工作小組

建立日期 2011-04-21 00:00      最近更新在 2011-04-25 12:01 作者是 謝良奇／編譯
致力加速 Linux 成長的的非營利組織 Linux 基金會 (Linux Foundation)，日前宣佈成立高可用性 (High Availability，HA) 工作小組。這個新的工作小組將加速 Linux 產業生態系統內的發展，讓該作業系統核心更適於建構高可用性系統。工作小組將連同各專案與參與者，共同定義開放源碼的高可用性軟體堆疊，並根據開發者、廠商、客戶的意見，制定功能的優先順序。

由 Linux 基金會主導的工作小組，參與的專案包括 corosync、DRBD、GFS2、hawk、Linux-HA、Linux Virtual Server、luci、OCFS2、Open Clustering Framework、pacemaker。最主要的企業 Linux 散佈套件如 Debian、Fedora、openSUSE、Ubuntu，都已納入此項技術並將為工作小組做出貢獻。

Novell 的工程師，也是該工作小組架構師的 Lars Marowsky-Bree 將在 Linux 基金會協同大會 (Linux Foundation Collaboration Summit) 對高可用性與 Linux 發表演說。Marowsky-Bree 表示，希望在開放源碼軟體上執行 Linux 叢集的人，都會是該軟體堆疊的使用者。儘管 Hewlett-Packard 提供了 Linux 的高可用性套件，紅帽 (Red Hat) 也為 Red Hat Enterprise Linux 提供了高可用性的預覽模式，使用者一直以來需要的是完全開放源碼、廠商中立的套件。

Linux 基金會執行董事 Jim Zemlin 表示，Linux 基金會的高可用性工作小組將帶領重要專案，在共同元件集與優先順序上進行協同合作，以支援企業中此一成長領域。此一協同開發模型將能加速關鍵技術的進展，他們預期高可用性工作小組將達成此任務。

高可用性意指電腦系統或網路中的資源可用性，由於內外部在架構與應用軟體上的需要，高可用性比起以往任何時候都更為關鍵。Linux 基金會將高可用性計算定義為維持至少 99.999% 運行時間的系統。換句話說，一年僅允許約 5 分鐘又 26 秒的故障停機時間。

當高可用性計算需求漸增之時，Linux 也逐漸受到發展此類系統廠商的歡迎。過去數年，高可用性系統建造者傾向使用一般商品化硬體，加上額外的備援設備，以確保系統在個別元件出錯時仍能持續運作。

Marowsky-Bree 表示，透過將伺服器叢集化，將可達到高於單一系統的可用性。因此上述的參與專案有助於叢集化伺服器之用。儘管許多技術已經達到成熟，仍須進行額外工作以確保這些技術能輕易共同運作。
