Talend 發表開放源碼整合方案 擴展雲端運算與大規模資料技術支援

建立日期 2011-12-12 09:59      最近更新在 2012-05-09 15:19 作者是 謝良奇／編譯

法國開放源碼整合公司 Talend 日前發表了其整合統一平台的第 5 版 Talend v5，成為針對包括資料、應用軟體、企業流程在內所有企業整合需求，提供全面技術產品組合的唯一開放源碼廠商。在此一新版的開放源碼中介軟體平台中，Talend 透過與 BonitaSoft 的 OEM 合作，為其產品線加入了企業流程管理 (BPM) 能力。

當前組織通常依賴其 IT 部門撰寫腳本以同步資料，但結果總是過於複雜且多變。 Talend 的 Talend Unified Platform 正是依賴開放源碼社群解決此問題的雲端服務。

Talend v5 提供了若干加強功能，擴展了對雲端運算、混合雲端系統，以及 Hadoop 等大規模資料技術的支援，其針對線上資料整合的雲端功能，不僅限於企業流程，更包括管理現有流程，甚至是運用圖形化工具規劃新流程。

在新版軟體中，Talend 提供了全面化的整合途徑，透過一套通用的產品、工具、最佳實踐，讓 IT 組織能匯集傳統上完全不同的整合嘗試與做法。Talend 表示，Talend Unified Platform 提供一套產品的共同基礎，可依據組織需求進行個別或協同運用。

此外，Talend v5 的設計支援現今對雲端和大規模資料的市場趨勢。Talend 認為，由就地部署 (on-premise) 系統、私有雲、公有雲、軟體即服務 (SaaS) 構成的混合雲端環境，所創造的獨特整合挑戰，僅能透過橫跨所有環境的整合平台才能有效解決。

Trader Media Group 開發經理 Philip Naughton 指出，英國最大駕駛市集 AutoTrader.co.uk 背後的公司 Trader Media Group，多年來一直將 Talend 使用在廣泛的任務關鍵資料整合需求上。如今隨著眾多應用軟體被部署到雲端，以及對整合即服務的需求漸增，他們認為 Talend 對於企業是相當有利的一項技術。

Talend 新版平台納入了新的企業流程管理產品 Talend Enterprise BPM，以補足其現有的資料管理與應用程式整合產品。此一產品利用 Talend 和開放源碼 BPM 廠商 BonitaSoft 的 OEM 合作為基礎。

在第 5 版的整合平台中，以開放源碼授權的產品被統一納入 Talend Open Studio 旗下，包括 Talend Open Studio for Data Integration、Talend Open Studio for Data Quality、Talend Open Studio for MDM、Talend Open Studio for ESB。

在資料整合上，Talend 早已對 Hadoop 提供某種程度的支援，這次改版則為 Hive、Pig、Sqoop 提供加強支援。Talend 資深產品管理主任 Ciaran Dynes 表示，組織近來最大的挑戰之一是改編其現有資料庫以便運用於 Hadoop 中。這並不表示他們要取代傳統關連式資料庫，只是認為大規模資料可以運用在今日遇到的某些問題上，其中之一便是資料歸檔。

Talend v5 的資料整合特點之一，就是運用 Hadoop 的 MapReduce 讓用戶在資料中發掘更多資訊。Dynes 預測，這將會改變組織歸檔資料的方式。

在資料品質方面，Talend Open Studio for Data Quality 新的嚮導精靈與備忘單，能協助找出重複資料與資料連接。在開發時期即時視覺化對應結果的功能，能立即顯示演算法或參數更動造成的影響。

Talend 的企業服務匯流排 (Enterprise Service Bus，ESB) 元件 Talend Open Studio for ESB，是一套用以建構、測試、部署 REST 網路服務的圖形化環境，讓整合後的資料得以線上存取。Dynes 表示，該元件能協助並未擁有純熟 Java 開發團隊，或不希望在整合工作上投入最佳 Java 開發資源的組織。他們希望用點擊方式，連接資料庫並公開作為網路服務。

對於主資料管理 (Master Data Management，MDM)，Talend Open Studio for MDM 針對主資料中心提供全球的高效能搜尋功能，其 Web 表單設計器可針對任何主資料化的項目，設計與部署客製化表單。Dynes 指出，Talend 的 MDM 工具改變了用戶打造 Web 表單以視覺化整合資料的方式。
