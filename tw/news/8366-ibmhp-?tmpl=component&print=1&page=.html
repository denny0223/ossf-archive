紅帽、IBM、HP 成立開放虛擬聯盟

建立日期 2011-06-09 00:00      作者是 謝良奇／編譯

日前的開放源碼商業大會 (Open Source Business Conference，OSBC) 發佈了若干重大消息，其中最重要的莫過於開放虛擬聯盟 (Open Virtualization Alliance，OVA) 的成立。

由 IBM、Hewlett-Packard、Intel、紅帽 (Red Hat)、Novell、BMC、Eucalyptus Systems 所支持，該聯盟以推展基於 KVM hypervisor 的開放源碼虛擬化堆疊為目標。此一團體的成立再次清楚顯示，開放源碼為基礎的堆疊，將在雲端運算的興起中扮演顯著角色。VMware 與其開放的雲端鑄造廠 (Cloud Foundry) 計畫，勢必會緊盯此一發展。

作為非營利組織的開放虛擬聯盟，其成立是為了作為鼓勵和協助開放虛擬化技術推廣的共同行動。該聯盟有若干理由吸引其他夥伴加入，首先是 KVM。KVM 是 Linux 核心的一部份，提供的效能驚人，又能達成全虛擬化 (full virtualization)、絕佳的延展性、高水準的安全性。這些特性使得 KVM 最新的企業虛擬化市場競爭者。

紅帽在 2008 年購併了開發 KVM 與相關 SPICE 協定的以色列公司 Qumranet，並取得 KVM。作為開放源碼軟體，KVM 被納入於 Linux 核心中，是 Linux 在具備虛擬化延伸 (如 Intel VT 或 AMD-V) x86 硬體上的全虛擬化解決方案。透過 KVM，使用者可在虛擬機器中執行未經修改的 Linux 或微軟 Windows 映像。

Linux 2.6.20 核心首度納入了 KVM，並與 Linux 核心同時進行開發。KVM 已能延展至支援 4096 個核心主機與 64 TB 記憶體。根據 OVA 表示，KVM 贏得了最重要的 SPECvirt 效能指標。

其次 OVA 目前的理事成員包括 HP、IBM、Intel、紅帽，都是開放源碼專案的主要貢獻者，儘管該聯盟並未提出制定虛擬化標準的意圖，但社群勢必會對該聯盟提出要求。目前該組織主要著重在教育、最佳實踐、技術指導、KVM 開發、推廣開放虛擬化技術上。

創立成員中的 IBM、紅帽都是主要的開放源碼支持者。紅帽是全球最成功的 Linux 與開放源碼軟體公司，IBM 則是將不少軟體專利釋出給開放源碼社群。HP 發起超過 100 個開放源碼專案，並貢獻可觀的智財權給開放源碼社群。Intel 也貢獻了 MeeGo、Xen、KVM、OpenSolaris、Eclipse、Java、GCC 等開放源碼專案。

此外，該聯盟將協助企業採納該虛擬化技術，作為其他商業技術之外的低成本替代方案。OVA 表示，部署以 KVM 為基礎的解決方案，相較於在私有解決方案上部署類似組態，可節省最多達 80% 的成本。

儘管 OVA 將協助推廣 KVM 為基礎的解決方案，發佈最佳實踐文件與藍圖，並且教育市場，透過行銷活動提高對 KVM 選項的接受度，但並不會取代開放源碼社群對 KVM 開發的管理。該聯盟表示，軟體、技術佈局、規格、開發將持續由社群程序主導。而此一程序大部分以紅帽為中心。

從另一方面看來，OVA 沒有明說的任務，是直接與 VMware 競爭。正如 Redmonk 分析師 Stephen O'Grady 表示，OVA 基本上是擁有共同利益關係廠商的結合。阻止 VMware 主導虛擬化與隨後的私有雲端市場，就是這個共同利益所在。

VMware 擁有虛擬化市場的最大市占率，並以雲端鑄造廠這樣的計畫，作為對於開放虛擬化策略興起的回應。由於 IBM、Intel、HP 都是 VMware 合作夥伴，OVA 有這些廠商的加入，無疑是對紅帽和 KVM hypervisor 最好的背書。

儘管紅帽副總裁 Scott Crenshaw 曾表示每個人都希望有 VMware 之外的開放替代方案。然而，事實是，VMware 目前掌握了大部分的市場。根據 Gartner 的資料，紅帽的 KVM 僅占不到 1% 的市場。換句話說，是紅帽希望有 VMware 之外的開放替代方案，而 OVA 的其他成員，則希望拉下獨霸的 VMware。

雖然參與 OVA 的廠商或許並沒有真正意願，把紅帽拱上開放源碼虛擬化技術的領先地位，但IBM、HP 等公司並沒有太多選擇。畢竟 VMware 的優勢地位顯而易見，紅帽是僅有的可能性。

紅帽希望 OVA 能夠比過去挑戰 VMware 的其他虛擬聯盟更加成功。但即使事與願違，對紅帽而言或許也沒太大關係。根據該公司每季財報顯示，紅帽的虛擬化業務正以飛速成長，如果 OVA 能成功，那最好，如果沒有，也無損紅帽的成長動能。
