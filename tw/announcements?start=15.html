完成升級Subversion模組

建立日期 2008-10-22 17:14      最近更新在 2010-07-23 23:07 作者是 ossfepaper

OpenFoundry已在台北時間 2008/10/22(星期三) 上午12:00 完成升級Subversion模組。

若造成困擾，在此表達歉意。

以下為新功能的說明。

1.讓專案可選擇subversion不開放給無權限的使用者來讀取資料，只有專案成員並且有給予權限者才可存取。

在專案的基本資料中選擇[版本控制系統]中選擇[Subversion Members only]。

使用站內提供的svn(subversion)版本控制系統，目前此類別的專案無法使用viewvc(專案的版本控制頁面)的web瀏覽。

如下二點才有存取的能力：

* 專案的Admin(管理員)，使用者是Admin這個角色。

* 於[權限管理]中替某角色(Role|群組)增加(核選)[Commit Changes]的權限，並於[成員管理]中將使用者加入此角色中

2.已將subversion從版本1.4.6升級至1.5.2版

升級後如果您遇到問題，或對新的版本有任何建議，請不吝與我們聯絡，謝謝！

我們的 Email 是： 這個 E-mail 地址已經被防止灌水惡意程式保護，您需要啟用 Java Script 才能觀看

OpenFoundry 系統升級 / System Upgrade

建立日期 2008-07-28 18:30      作者是 ossfepaper

我們將在台北時間 2008/08/04(星期一) 上午10:00 升級整個系統， 屆時 OpenFoundry 服務將暫停約24小時。

本次升級主要的變更有：

1. OpenFoundry(主系統): 將使用Ruby on Rails為系統主體，並與各模組整合

2. RT(待辦事項): 變成獨立的模組，並升級至 3.6.6 版

3. Wiki(共同筆記): 此版未做升級

4. VCS(版本控制): Subversion升級至 1.4.6 版

5. Sympa(通信論壇): 升級至 5.4.3 版

6. Web Hosting(專案網頁空間): 此版未做升級

7. Release(規劃釋出): 升級後使用FTP上傳檔案

如有任何問題與建議，請不吝與我們聯絡，謝謝！

We will upgrade our systems at 2008-08-04 02:00 UTC. All the services will not be available for about 24 hours.

The major changes are:

1. The new web front-end is writen in Ruby on-Rails

2. RT(issue tracking system): upgrade to version 3.6.6

3. Wiki: no upgrade

4. Version Control System: upgrade Subversion to version 1.4.6

5. Sympa(mailing list): upgrade to version 5.4.3

6. Web Hosting: no upgrade

7. Release: files can now be uploaded with FTP

If you have any questions, suggestions or concerns, please do not hesitate to contact us.
