使用自由軟體 ≠ 操作比較難

建立日期 2007-09-14 16:00      最近更新在 2011-07-12 14:24 作者是 蕭丞舜 (Attila)

許多人一聽到自由軟體，最先聯想到的就是 Linux 以及能在它底下執行的應用軟體，一想到 Linux，就立刻和操作不方便劃上等號，連帶著其他自由軟體也一併遭殃，即使這些軟體已經採用操作簡便的圖形介面。其次會說：與其浪費時間重新學習與適應，還不如就原有已經習慣的軟體來用就好。這樣的說法有一部份是對的，另一部份則有待商榷。

首先，自由軟體很多，並不只侷限於身為作業系統的 Linux，更不能說在 Linux 底下執行的應用軟體都是自由軟體（商業軟體也有 Linux 版本），自由軟體是一種概念，並不是特定陣營的軟體才叫做自由軟體，唯一的辨識標準，就是看它是否採用自由軟體的著作權和公共授權方式。（目前多以自由軟體基金會的 Copyleft 和 GPL 為主）

很多使用者在使用軟體時，並不太會特別注意這樣的問題，僅以先入為主的錯誤概念來辨別，才會把應用面的自由軟體和 Linux 混為一談，導致只要聽到自由軟體，就先聯想到 Linux，接著和「那是個自己不熟悉的作業系統」聯想在一起，於是認為若想使用自由軟體，就得把自己的電腦系統整個翻新才行。事實上，許多自由軟體的應用程式，是提供多種作業系統的版本（如 Windows 和 Mac），不見得一定要在 Linux 底下安裝和執行。

既然可以選擇在自己熟悉地作業系統下執行的自由軟體，那麼使用者介面的便利性，只取決於程式設計者如何設計該軟體的使用者介面和操作邏輯，跟作業系統所採用的介面沒有太大的關聯。

很多人之所以會認為自由軟體只能在 Linux 上執行，大概是因為早期自由軟體的開發，多以充實 Linux 應用為主有關。不過發展至今，情勢已然不同，著名的 7-zip、酷音輸入法等，都有 Windows 的版本可用，若這些自由軟體適合自己的需求，大家不妨考慮使用這些軟體來替代商業版本。

話雖如此，但總有些東西是只能在 Linux 底下執行的吧？是的，這時候如果還是想用，當然就必須要安裝 Linux 的作業系統才行啦！不過，對於這個自由軟體中最重要的作業系統，為何大家會覺得它的操作很困難呢？這跟它的用途和 Unix-Like 體質有關，相關的發展史在此就不多談，我們只要了解，它原本所設定的使用對象就不是一般的使用者，而採用的操作介面是最簡單也最不耗資源的文字介面。這對於已經習慣於圖形介面的人來說，簡直是難以超越的障礙。

以我個人為例，十多年前之所以不大願意用電腦，就是當時流行的 DOS 是文字介面的關係。因為想要好好操作 DOS 並且加以應用，就必須熟背那些文字指令才行，只要有個字母輸入錯誤（這些指令是以英文為主），螢幕上只會顯示「Bad command or file」，根本不知道真正不能執行的原因是什麼！要不是為了玩遊戲（就是鼎鼎大名的三國志 3），根本不想與電腦為伍。

不過當 Windows 3.1 出現之後，情況就不同了。到這個階段，常用的功能只要看圖說故事，根據圖示的樣子就能猜出它是作什麼用的，然後動動滑鼠點一點，多數的情況就會產生所要的結果了！這和過去得先熟背指令才能應用電腦的 DOS 來比，簡直是天堂與地獄的差別！所以圖形介面的低學習障礙，對於只想用電腦玩點遊戲，打打報告的我來說，真的具有莫大的吸引力。接下來出現的 Windows 各大系列會受歡迎，大抵上也都是以這個原因原主。所以，Linux 的文字介面，對於類似我這種只想用電腦做一般應用的人來說，當然是最不願意去碰的。我想所有的使用者都希望電腦的學習障礙越小越好，能夠用最短的時間讓電腦能幫忙做事，絕對是上上之選。

不過 Linux 發展這麼多年下來，圖形介面如 X-Windows、GNOME 等已經越來越成熟了，內含這些套件的 Linux 的版本也越來越多（如 Ubuntu），以個人應用的角度來看，它的學習障礙是相當低的。若是一般家庭或企業日常應用（文書、簡報、試算表、上網、影音娛樂等），不妨可以考慮採用這個，讓自己享受一下使用軟體的自由以及節省一些電腦系統的支出成本。

不過，原來使用其他作業系統的朋友，一旦決定改採 Linux，確實會有習慣上的問題要克服。畢竟不同的設計理念，操作邏輯總有不相同的地方，使用者是需要花一點時間來學習的（都是圖形介面，花掉的時間也不會太多）。但若只是因為習慣問題而不願嘗試其他的解決方案，這不代表自己缺乏學習的意願？因為所有的學習，或多或少都會改變過往的習慣。在這個需要終身學習的全球化的時代，希望大家別再把使用習慣當藉口，而應該以「需不需要轉換；轉換後所得到的好處值不值得」的角度來仔細評估才是。
