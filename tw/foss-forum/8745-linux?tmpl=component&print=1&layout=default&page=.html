Linux 將成為電腦遊戲的新平台？

建立日期 2012-07-06 11:50      最近更新在 2012-07-24 20:25 作者是 Kenneth Mak

◎ 本文原載 Linuxpilot 網站，原文章連結

遊戲廠商 Valve 早在 2010 年，已經表示過會推出 Linux 版的 Steam 平台，只是一直都沒有實現。但最近 Valve 卻一反常態，開始招攬 Linux 人材，務求把 Linux 打造成遊戲平台，原因是 Valve 開始受到微軟的威脅。

Valve 正在積極準備 Linux 版的 Steam 平台的消息，源於 2012 年 4 月底的一篇訪問。訪問中 Valve 的創辦人 Gabe Newell 稱讚 Linux 是一個有潛力的遊戲平台，相反 Windows 就沒有未來了，特別是 Windows 8 更加令人失望。這一番話如果出自一個普通人之口，一點也不稀奇。然而 Gabe Newell 卻是一位曾經在微軟工作了 13 年的工程師，而且是最早參與開發 Windows 的技術人員之一，這就有點奇怪了。更奇怪的是 Windows 目前是 Valve 和 Steam 的最大市場，究竟是什麼原因，令 Valve 不得不得罪自己的最大客戶？

Linux 的確有多方面的優點，但作為遊戲平台卻絕對比不上 Windows，這是不爭的事實，至少在過去十年以來的確如此。沒有幾家廠商積極地開發專門為 Linux 設計的遊戲，原因是在 Linux 上玩遊戲的人仍然屬於少數。當在 Linux 上有開發遊戲經驗的人為數不多，各種開發架構的成熟程度，自然給 Windows 比下去。當然假以時日，要令 Linux 成為真正遊戲平台亦非沒可能，只要積極開發就可以了。但在遊戲業界，「今天能夠」十分重要。要是停留在「下一年能夠」階段的話，就等於說「不能夠」了。

不論是 iPhone 還是 Android，都傾向用戶從唯一的官方渠道下載應用程式。如果大家是 Mac OS X 的用家，便會發現這個傾向已經延伸到電腦平台上了，未來的 Windows 8 也會利用同樣的方式，用戶只可以從 Marketplace 下載官方認可的應用程式。這本來其實無可厚非，從系統安全的角度而言，限制用戶的下載渠道，的確可以減少用戶下載有問題的程式，加強客戶端的安全性。

但這樣一來，卻會令用戶失去選擇的自由，「官方認可」亦大有可能成為打擊競爭對手的工具。Valve 正是因為感覺受到威脅，才會積極著手打造 Windows 以外的遊戲平台。畢竟微軟過去藉平台打擊對手的例子實在太多，IE 藉此把 Netscape 迫至末路、WordPerfect、Lotus 123 和DBase 亦因為缺乏平台優勢被 MS Office 取代，RealPlayer 也差不多被 Media Player 淘汰。不想成為下一個犧牲者的話，便要好好地想想下一步了。
