Linux.com 的問與答：ownCloud 提供企業安全的檔案分享

建立日期 2013-05-28 10:32      最近更新在 2013-05-28 13:42 作者是 謝良奇／編譯

使用用戶導向的雲端服務進行檔案儲存與分享，對希望避免機密資料外洩的企業來說，將構成安全上的問題。ownCloud 希望藉由提供安裝於企業自有資料中心的商業雲端服務，來解決這個問題。ownCloud 技術長 Markus Rex 表示，他們的開放源碼軟體建構於 Linux 上，多半由企業用戶部署在 Linux 上。在電子郵件中，Rex 談到 ownCloud 在開放雲端體系中的定位，開放源碼公司的意義為何，如何確保開放源碼軟體的安全性，以及最近他們為何加入 Linux 基金會。

  • ownCloud 在開放雲端體系中的定位
    ownCloud 核心是安全雲端引擎。企業與用戶透過整合 ownCloud 與其基礎架構和安全系統，能在他們的政策下，完全控制其機密資料。我們整合開放源碼平台如 OpenStack，並且在高普及率的開放源碼堆疊上運行，如 Linux、Apache、MySQL、PHP。

  • ownCloud 如何運用 Linux
    Linux 是我們的主要開發平台，這很重要，這一點不僅是技術聲明，也是社群聲明。我的意思是，開放源碼存在我們的 DNA 之中。我們的本質就是開放的，這不僅有助吸引人們加入我們的社群，也讓我們能貼近現有的 IT，並迅速且輕易地為用戶需求擴展我們的能力。
    我們的部署大多在 Linux 上，而散佈與取得 ownCloud 的便利途徑之一，就是透過 Linux 套件管理程式，不論是 SUSE、Red Hat、Debian，或是 Ubuntu。當然，我們也可以輕易安裝在 Windows 上，我們當然也為大多數平台提供終端用戶存取工具，不過許多 ownCloud 內部與相關人士都使用 Linux。

  • 如何確保開放源碼雲端軟體在企業應用的安全性
    談到安全性，軟體是否開放源碼並非重點。你是否信任軟體的運行環境才是重點。部署在企業資料中心的 ownCloud，給予 IT 人員強大而易用的工具，在其現有資料安全、監控、管理工具中，整合檔案同步與分享功能。透過控制 IT 環境，而非仰賴不知道位於何處的第三方伺服器，用戶才能控制資料如何分享。如果能夠信任執行環境，其中莫過於你自己的資料中心，開放源碼軟體並不會對此信任造成損害。

  • 為何加入 Linux 基金會
    我是該基金會的前技術長，所以我知道其價值所在，像是教育訓練到體系連接等等。由於經營與 Linux 高度相關的業務，加入該金會是理所當然。
