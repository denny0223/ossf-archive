歐盟執行委員會反對 Oracle 購併 Sun

建立日期 2009-11-17 08:00      最近更新在 2009-12-17 17:48 作者是 謝良奇/編譯
關注 Oracle 購併 Sun Microsystems 進度的人，或許對於歐盟執行委員會 (European Commission) 日前正式反對這個價值 74 億美元交易，並不感到十分驚訝。Oracle 隨即發出聲明，反駁歐盟執行委員會認定 Oracle 購併 Sun Microsystems 以及旗下開放源碼資料庫子公司 MySQL，將有損開放源碼領域的競爭。

歐盟執行委員會告知 Oracle 與 Sun，這項決定的主要理由，是該交易包含了可以免費取得且普遍的 MySQL 資料庫。該監管機構認為，由全球最大商業資料庫公司擁有最大開放源碼競爭者，可以看到重大的利益衝突。

歐盟執行委員會的決定，並不會完全關上這項購併案的大門，但其進度將受到延遲影響。Oracle 與 Sun 預料將會立即展開上訴。

Oracle 在發出的聲明中，批評該委員會對於資料庫市場與開放源碼軟體的了解不足，該公司指出，歐盟執行委員會的反對聲明，顯示對於資料庫競爭與開放源碼動態的徹底誤解。此外，該聲明表示，了解開放源碼軟體的人就知道，因為是開放源碼之故，任何人都無法控制 MySQL。這就是開放源碼的關鍵。

Sun 在 2008 年支付約 10 億美元購併 MySQL，當時美國與歐盟當局並未提出反對。

自從歐盟展開調查，已經開始造成 Sun 的損失。Oracle 執行長兼創辦人 Larry Ellison 表示，由於 Sun 的顧客在該公司的前途明朗前，擱置了新的採購，Sun 每個月要損失 1 億美元與數千個工作。當此一交易受阻而 Oracle 提出上訴之時，拉長的購併過程對於 Sun 的財報與顧客，都是一大挑戰。

此外，Oracle 的聲明宣稱，資料庫市場處於高度競爭狀態，至少有 8 家強勁的競爭對手，包括 IBM、Microsoft、Sybase 與 3 家開放源碼廠商。Oracle 與 MySQL 是非常不同的資料庫產品，這 8 家銷售截然不同產品廠商中的 2 家要進行合併，歐盟法律並無禁止此一合併的基礎。像這類的合併經常發生，數十年來並未受到美國或歐盟的反對。

歐盟競爭委員會專員 (EC Competition Commissioner) Neelie Kroes 表示過許多次，她希望看到證據證明，在 Oracle 擁有下，MySQL 仍會持續發展與創新。然而，她說，Oracle 並沒有提出證明。

第二大商業開放源碼公司 Ingres 執行長 Roger Burkhardt 同意 Oracle 的說法，Burkhardt 表示，歐盟似乎不了解資料庫市場的競爭狀況，也不知道開放源碼軟體的運作。Burkhardt 和前 MySQL 執行長 Marten Mickos 都同意，開放源碼領域的市場力量，並不在 Oracle 的私有業務領域之內。

開放源碼支持者也是前 MySQL 商業顧問 Florian Mueller 表示，Oracle 想要全贏，Sun 註定全輸，歐盟競爭委員會顯然沒有其他選擇，只能發出反對聲明。Mueller 指出，宣稱 MySQL 的開放源碼本質，就足以確保競爭的人，忽略了開放源碼只是配送車輛，MySQL 依賴的是使用相關智財權，產生營收並贊助進一步開發的一家公司。

Mueller 表示，為什麼 Sun 願意支付 10 億美元購併一個可以分支出去的產品？為什麼 Oracle 不願意捨棄 MySQL，僅僅購併 Sun 就好，然後再分支開發 MySQL？如果一切這麼簡單，不論是誰擁有這項資產都沒關係？

Ellison 在最近的 Oracle OpenWorld 大會上表示，MySQL 不會和該公司的資料庫競爭，而是擁有自己的市場與支持者，其主要競爭者是微軟的 SQL Server。 Oracle 在其聲明最後表示，由於缺乏競爭傷害的證據，他們有信心該交易最終將獲得無條件通過。
