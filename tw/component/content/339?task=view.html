Minimo 專案簡介

建立日期 2004-05-18 08:00      最近更新在 2007-03-09 01:18 作者是 謝良奇
Minimo （Mini Mozilla）專案是專為嵌入式設備發展高 Web 標準支援瀏覽器的計畫，此一可稱為極簡版 Mozilla 的嵌入式瀏覽器，日前已發表第一個釋出版本，獲得不少用戶及廠商的關注和青睞，可說是當前小型化設備上可帶來最佳瀏覽體驗的瀏覽軟體。

New Mobile Computing 上日前發表 Doug Turner 和 Chris Hofmann 的專訪稿。他們是極簡瀏覽器 Minimo 的專案領導人，由他們兩位來談談這套供 Linux 設備使用的嵌入式瀏覽器，目前的開發現況，以及該專案的動機和目標。Doug Turner 和 Chris Hofmann 表示他們已完成了大量有關記憶體空間縮減的工作，目前Minimo 的記憶體用量相當穩定，並不遜色於市場上其它瀏覽器。當然這一點也是 Minimo整個開發過程中相當大的挑戰之一。此外，Doug Turner 和 Chris Hofmann 認為移植到新平台是最困難的工作。由於小型設備上眾家軟體 IE、WebBrowser、WebPro、NetFront、Opera 等，競爭越趨激烈，面臨即將爆發另一場瀏覽器大戰，Doug Turner 和 Chris Hofmann 表示他們最需要的是更多設備製造商的支持。Mozilla 可說是開放源碼界的最佳瀏覽軟體，在不少桌面應用軟體如 Netscape 7、CompuServe、AOL for Mac、Ximian GNOME、Galeon，以及 Nautilus，都可見到 Mozilla 的蹤跡。相對於 Mozilla，Minimo 則是鎖定嵌入式領域網路應用的極簡化瀏覽軟體。Minimo 的表現，可稱得上是當前嵌入式設備上最佳的瀏覽器。Minimo 的誕生將使用戶得以在桌面、筆記型電腦和嵌入式設備，像是 PDA、智慧型手機上，擁有相同的瀏覽工具及網路內容閱覽體驗。針對小型設備顯示空間不足的問題，Minimo 提出稱為 Small Screen Rendering 的解決方案。此一技術利用 CSS（Cascading Style Sheets）把網頁調整為適合小螢幕顯示的格式。Minimo 專案的焦點擺在程式碼及執行時間的縮減，以利於小型消費性裝置的移植。執行 Minimo需約 25 MB 的記憶體，乍看之下顯得有些驚人，但 Minimo 以此取得了對目前 Web 標準的高度支援。其它的嵌入式瀏覽器在標準支援上則較為不足。該專案目前的執行平台以擁有 32 到 64 MB 記憶體，執行 Linux 及 GTK toolkit 的 ARM裝置為主。Minimo 瀏覽器架構在 Mozilla 的 "gecko" 引擎上。Minimo 本身運作需要約 25 MB的記憶體。目前官方釋出版本乃針對附有 GPE（GPE Palmtop Environment） 的 Familiar 0.72 Linux散佈套件。
