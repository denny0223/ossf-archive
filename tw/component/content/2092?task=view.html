Mac OS X 與 Linux 平台 Chrome 釋出開發者預覽版本

建立日期 2009-06-11 08:00      作者是 謝良奇/編譯

對於 Mac OS X 與 Linux 平台上的用戶來說，Google 終於釋出了這兩個平台的 Chrome 瀏覽器，是個好消息。壞消息是，該公司警告，除非你是開發人員，請不要下載它們。

Google 產品經理 Mike Smith 與 Karen Grunberg 在部落格寫到，為了獲得更多來自開發者的回應，他們釋出了 Google Chrome for Mac OS X 與 Linux 的初期開發者版本。除非你是開發人員，或者樂於使用不完整、難以預期且經常當掉的軟體，否則，請別下載它們。

這套 Google 的開放源碼瀏覽器過去僅提供 Windows 版本，許多 Mac、Linux 用戶一直都在要求屬於這兩個平台的版本。Google 的程式開發人員正在為 Windows 以外的平台，重新建立 Chrome 的部份元件，例如圖形使用者介面，以及用來隔離不同程序的 sandbox。

Google 提供了 3 種不同的 Chrome，stable、beta 與開發者預覽版 (developer preview)。現在釋出的 Mac OS X 與 Linux 版本屬於第 3 類，也是臭蟲最多，有待測試，完成度最低的版本。

例如說，這個預覽版本的 Flash plug-in 無法動作，沒辦法讓用戶觀看 YouTube 影片。列印、書簽管理也還沒實作，隱私設定、修改預設搜尋引擎設定等，也還沒完成。根據 Google 表示，其中包含超過 400 個臭蟲。

知名部落格 ReadWriteWeb，在測試 Chrome 的 Mac 版本數週後表示，該版本穩定度雖然持續改善，但仍無法播放 Adobe Flash 檔案。

儘管這只是為特定群眾釋出的版本，但其意義仍然重大。首先，該軟體已經掛上 Google 用來自動下載新版軟體的自動更新服務，而且更多使用者意味新版本的自動當機報告，將協助 Google 進行程式除錯。

不過，也不是每個 Mac 與 Linux 用戶都能嘗試這個新版本。Google 發言人 Eitan Bencuya 表示，Linux 版本僅支援 Debian 與 Ubuntu，Mac OS X 版本則只支援採用 Intel 架構的 Mac。

經過多年來微軟 Internet Explorer 近乎獨霸的局面，瀏覽器市場近來再次成為多方人馬競爭之地，Mozilla 正位於 Firefox 3.5 釋出之際，Apple 則有 Safari 4，Opera 10 處於 beta 階段，微軟正緩緩地開始加快釋出數週之久的 Internet Explorer 8。

根據 Net Applications 統計，2009 年 5 月，Internet Explorer 以 65.5% 仍維持市佔率第 1 名寶座，Firefox 則是 22.5％，Chrome 則佔 1.8%。暫且不論 Chrome for Mac 版本目前的不穩定性，Computerworld US 發現其效能表現不俗，在 JavaScript 吃重的網頁上，速度比最新 Firefox 版本快上 37%。

瀏覽器版本的多樣化意味，網路開發者也必須隨之測試其網站，確保在各種瀏覽器上運作無誤。由於 Chrome 在解譯與顯示網頁上，採用和 Safari 相同的 WebKit 引擎，Google 宣稱 Chrome 應該有相似的呈現。然而，Chrome 卻使用了不同的 JavaScript 引擎 V8，而 JavaScript 指令已經是，數量激增的複雜網頁與應用軟體背後的重要核心。
