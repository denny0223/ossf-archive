Sun 修訂 Java 執行環境授權 Linux 將可直接納入 Java 環境

建立日期 2006-05-09 08:00      最近更新在 2007-03-08 08:14 作者是 謝良奇
Sun 宣稱在本月稍後將舉行的 JavaOne 大會上，將宣佈修訂 Java 授權條款，削除開發人員在開放源碼作業系統內事先納入 Java 環境 的障礙。此外，預料 Sun 也將在該會上發表 Java Platform Enterprise Edition 5 的軟體開發套件 (SDK)。 授權條款修訂的對象主要針對 Java 執行環境 (Java Runtime Environment)，修訂後 Linux 等開放源碼作業系統的開發人員就可 以免於授權限制，將 Java 執行環境納入作業系統中。Java 平台團 隊副總裁 Laurie Tolson 表示 Sun 已朝著對開放源碼作業系統更 為有利的方向，特別是 Linux、OpenSolaris，修訂授權條款。

外界要求 Sun 將 Java 開放源碼的聲浪從未停止，開發人員認為如 此將有助於 Java 平台的發展，也能鼓勵開放源碼創新發展。Sun 一 直不願正式將 Java 開放源碼，取而代之的，是採取改變 Java 開發 流程，並修訂授權方式，以便開發者能更輕易地檢閱 Java 原始碼。

除了修訂授權之外，Sun 也會針對 Java Platform Enterprise Edition 5 發表軟體開發套件的釋出計劃。Java EE 5 是 Java 伺服 器標準的最新升級版本，這個軟體開發套件將可降低開發 Java 伺服 器應用程式的複雜度。

相較於先前版本，Java EE 5 為加速 Web 開發與 Web 服務做了許多 改善，支援 Enterprise Java Beans 3.0 標準，以及內建的 AJAX 支援。

Sun 工程師 Bill Shannon 表示，Java EE 5 簡化開發的新功能可以 讓開發者在更短時間內完成開發，同時減低程式碼的複雜度。

包括 BEA Systems、Oracle、JBoss 與 SAP 在內的公司宣稱已經採 用該軟體開發套件，相關產品將在未來 18 個月內上世。
