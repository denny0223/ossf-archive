微軟宣稱開放源碼侵害其專利 開放源碼廠商齊反擊

建立日期 2007-05-28 09:34      最近更新在 2007-07-04 17:32 作者是 謝良奇
微軟高層日前宣稱 Linux 作業系統、OpenOffice 桌面應用軟體與其它開放源碼共侵害了 235 項微軟專利。微軟近來試圖與開放源碼社群修好的努力，可能將隨此徒勞無功。

這回可不像 2001 年微軟執行長 Steve Ballmer 只是臨時脫口而出，用「癌症」一詞為 Linux 下註腳，微軟法律總顧問 (General Counsel) Brad Smith 與他在智財權與授權方面的副手 Horacio Gutierrez，面對 Fortune 雜誌專訪時，發表了這項指控。微軟隨後發佈的新聞稿也重申了這個說法。

去年 11 月微軟與 Novell 達成雙方協議，Novell 用戶未來可免於微軟的專利控告，同時微軟用戶也受到 Novell 的專利保護。微軟曾表示其它 Linux，特別是紅帽 (Red Hat) 用戶將可能面臨專利控告。

微軟並不急於進入法庭。微軟官方堅稱只希望簽署授權協議。微軟平台策略總經理，也是微軟開放源碼專案主任 Bill Hilf 表示，他們不希望打擊任何一家開放源碼公司。正是這種不確定感，將成為微軟未來談判時的籌碼。

Linux 基金會 (Linux Foundation) 執行董事 Jim Zemlin 警告微軟必須小心行事。微軟與 Novell 簽署的協議透露出微軟產品可能同樣也侵害了其它科技公司的專利。Zemlin 說，微軟必須小心處理這個有可能最後會反過來傷到自己的議題。

微軟的下一步會是什麼？許多評論家認為微軟不會指出該公司受到侵害的專利為何。Linux 之父 Linus Torvalds 表示，微軟不願意公開底牌正說明了一件事。微軟希望藉由這種模擬兩可讓 Linux 開發者的反擊找不到著力點。

Gutierrez 告訴 Fortune 雜誌說，這些侵害不是無心之過。這種強硬的說詞可以收到對其它廠商的警告效果。正如 Dell 已經加入微軟與 Novell 的合作，Gutierrez 表示微軟會連繫其它 Linux 用戶要求簽署授權協議。微軟可能的打算是，藉由雄厚本錢與陣容堅強的律師團製造出來的恫嚇效果，微軟讓越來越多開放源碼供應商自願支付授權費，當許多廠商支付微軟授權費後，外界將減緩轉移至開放源碼產品的步調。微軟則在此時爭取到時間，以服務形式釋出更多軟體產品。

正如同專利法律公司 Alston & Bird 律師 Todd McClelland 所說，微軟宣稱的 235 項專利爭取到的，是該公司的策略轉型的時間。

此外，Novell 與微軟合作後，將成為大廠簽署微軟智財權授權的指標。在 Wal-Mart 這個世界最大公開招股公司成為其最大用戶後，微軟撐開的這一面 Linux 訴訟保護傘，將吸引到更多用戶與科技公司。如此一來將使其它公司對開放源碼失去信心。

然而，事情也並非完全悲觀。Jerome Lemelson 曾擁有超過 600 項專利，在 2004 年時包括 Symbol Technologies 在內的多家公司，以 Lemelson 利用提出修改以擴大專利保護範圍為由，聯手要求法院撤銷 Jerome Lemelson 所擁有的多項專利。Linux 基金會董事會成員也是軟體授權律師的 Andrew Updegrove 表示，微軟主張 235 項專利延伸範圍涵蓋 Linux、OpenOffice 與 Linux 使用界面的做法，正如同為所有作業系統專利擴大適用時限。Linux 是 Unix 的一種版本，他說，想想看 Unix 的歷史有多悠久。

微軟並非通用型作業系統的發明者，IBM 才是，Torvalds 認為，微軟侵害他人專利權的機會比 Linux 大得多。微軟很有可能高估了手中的牌，忽略了 IBM 與其它廠商可能團結反擊的嚴重性。IBM 與 Sun 手中都有可以反告微軟的專利，當這場多方避免多年的專利大戰一旦開打，結果將是軟體版本凍結的荒涼戰場，Windows 會因為訴訟問題，面臨多年無法釋出新版本的窘境。

當微軟尋找該公司認為受到侵害的專利時，開放源碼開發者會採取社群模式來進行專利技術的前案檢索 (prior art)，多年的訴訟只會弱化 Windows 目前的優勢，最終可能讓微軟的專利主張僅流於虛張聲勢的恫嚇。
