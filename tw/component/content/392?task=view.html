Novell 積極出招 擴大 JBoss 合作關係

建立日期 2005-04-08 08:00      最近更新在 2007-03-09 00:44 作者是 謝良奇
Novell正積極充任開放源碼與私有軟體合併者的角色，企圖為資訊技術資產、使用者識別與儲存設備管理提供相關軟體產品。正如 Novell 在其 BrainShare 研討會上所言，只要是足以與微軟一拼的技術，該 公司都樂意購買、租用或開發。

同時，Novell 正在擴大與 JBoss 之間的合作關係，打算將該公司的 JBoss 企業中介軟體系統（JBoss Enterprise Middleware System， JEMS）受納為其識別管理技術中的一部份。在日前雙方公開的擴大策 略合作關係中，Novell 會擴充其支援，貢獻程式碼與工程資源到 JEMS 中。 一度為網路軟體領導商，如今是基礎建設軟體與作業系統技術供應商 的 Novell 將積極參與包括 JBoss Portal 在內的 JBoss 產品架構設 計與走向。該公司將以 JBoss Portal 做為服務建構中心。此外， Novell 表示也計劃為旗下 Java 伺服器軟體生產線 Extend，加入更 多開放源碼技術。 Novell 第一步將貢獻基於 Web Services for Remote Portlets 標 準的 portlet 容器與 portlet 程式庫，協助加速 JEMS 元件之一的 JBoss Portal 的開發。 Novell 與 JBoss 雙方希望能率先結合開放源碼軟體與廠商創新力，提供市場強大且支援豐沛的應用軟體平台，減少以服務為基礎、具識 別需求之應用軟體的成本與時間。 JBoss 總裁兼執行長 Marc Fleury 表示，Novell 的幫助加速了雙方 關係的動力，他們期盼與 Novell 共同為 JBoss的專業開放源碼平台 加入識別驅動（identity-driven）的安全方案。 Novell 的動作還不僅只於此。該公司也正擴展與 Veritas Software 的合作，企圖提供儲存管理與高存續性軟體。 IDC 系統軟體研究副總裁 Kusnetzky 認為，Novell 的最大挑戰在於 如何說服潛在客戶，其對開放源碼與私有軟體的結合是未來趨勢。 Kusnetzky 說，Novell 眼裡並非只有開放源碼，和分站私有軟體與 開放源碼兩方的微軟及 Red Hat 等 Linux 廠商完全不同，該公司處 於一個獨特的競爭地位。
