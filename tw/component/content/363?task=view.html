新版 Eclipse 釋出 集合十大開放源碼 Eclipse 專案

建立日期 2006-06-30 08:00      最近更新在 2007-03-09 00:59 作者是 謝良奇
Eclipse 基金會釋出了代號為 "Callisto" 的 Eclipse 開放源碼專案第 10 版，此次可說是單一開放源碼社群主導下的最大型多重專案釋出，由於同時整合了 Eclipse 底下多達 10 個不同專案的釋出，可說為開放源碼軟體開發方法立下新的里程碑。

Eclipse 社群的主要釋出版本 Callisto 囊括了 10 個專案團隊、260 個專案提交者與多達超過 7 百萬行的程式碼，並且為 Eclipse 連續三年來皆按時程準時推出的主要釋出版本之一。 Eclipse 行銷經理 Ian Skerrett 表示，這象徵對於採用多種 Eclipse 產品的用戶的重視，他們希望同時散佈並鼓勵使用多重 Eclipse 專案。 Skerrett 表示，Callisto 的模式原不為 Eclipse 基金會執行董事 Mike Milinkovich 所支持，然而基於對開放源碼軟體開發與 Eclipse 管理模式的信任，Milinkovich 最終受到說服，才有了 Callisto 的誕生。 Milinkovich 認為 Callisto 已顯示出開放源碼開發模式在發展軟體平台上的效率。去年當 Callisto 專案開始時，要統整這麼多的專案仍是一大風險，如今 Eclipse 專案已經證明多專案、多廠商的分散式社群可以有效率地準時釋出軟體。 Interarbor Solutions 首席分析師 Dana Gardner 表示，雖然開放源碼軟體開發有著開放通透的協同合作，試圖管理多重開放源碼軟體專案仍極為困難。由於投入者多半是自願參與，因此很難指揮他們， Eclipse 卻成功地整合軟體開發與釋出，封住了那些開放源碼軟體批評者的嘴。 Callisto 強調，要讓組織更輕易運用 Eclipse 作為軟體開發平台，受益於 Eclipse 的模組化架構與各種 Eclipse 專案建構軟體。目前計劃使用 Callisto 的軟體商包括了 Actuate、AvantSoft、BEA、Borland、Business Objects、IBM 等。 Callisto 將 10 大專案一次、且如期釋出給用戶，囊括了以下 Eclipse 專案的主要釋出：Business Intelligence and Reporting Tools (BIRT) 2.1、C/C++ IDE (CDT) 3.1、Data Tools Platform (DTP) 1.0、Eclipse Modeling Framework (EMF) 2.2、Graphical Editor Framework (GEF) 3.2、Graphical Modeling Framework (GMF) 1.0 等。 有 15 個獨立軟體商支援開放源碼開發者投入 Callisto，這些開發人員來自 12 個國家，包括加拿大、美國、芬蘭、土耳其、中國等，展現出 Eclipse 社群跨越多重廠商與地域的特質。
