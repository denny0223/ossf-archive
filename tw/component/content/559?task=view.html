開放源碼 2005 年度展望

建立日期 2005-01-07 08:00      最近更新在 2010-07-13 23:20 作者是 謝良奇

2004 年多少是叫人感到欣慰的，由於先前令人憂心的 SCO 訴訟案逐 漸明朗，加上 Firefox 等開放源碼軟體創造紅盤成績，站在 2005 年起點此時，似乎讓人不禁對接下來一整年度的開放源碼界，抱有不 少期待。以下是業界專家對 2005 年開放源碼可能走向，所做出的預測。

◎ Linux 桌面、伺服器成主流

Linux 儘管近年在伺服器領域上進逼其他對手，創造出相當亮眼的成績，一直到去年為止，外界仍視 Linux 不足以攻佔桌面市場。2005 年呢？情況可能相當不同。

首先，Firefox、 OpenOffice 等商用及常用軟體逐漸成熟，開放源碼 軟體在這樣的背景下，被襯托得較過去更加平易近人，開放源碼軟體 不再是黑客、技術高手才能碰的技術名詞，而是活生生、用得上，功 能甚至比相同用途私有軟體更強悍。2005 年裡不少廠商將會釋出類 似 Windows 介面的桌面環境，消費者將更能接受 Linux 作為 Windows 之外，另一項低價選擇。

若干美國政府機構開始採納 Linux 為桌面標準，許多歐洲政府組織 在去年，轉而使用 Linux 與開放源碼軟體作為桌面系統，進一步會 擴大這股趨勢。預期這股趨勢會在 2005 年，擴展到企業、學校與其 他政府單位。AT&T 已表示正研究是否將 Linux 桌面系統視為組織標 準，美國賓州州立大學（Pennsylvania State University）也表示 將從 Internet Explorer 轉而改用 Firefox。資訊部門將擁抱 Linux 桌面以節省成本，同時提高安全性。預估到今年底，Linux 將在桌面 作業系統銷售額中佔上 10%。

IDC 分析師 Al Gillen 表示，Linux 不再是邊緣文化，如今，它是 主流。IDC 的研究更預測，Linux 在桌面、伺服器、套裝軟體上的整 體營收，在 2008 年將可達 350 億美元。其中大部份是伺服器。HP 全球 Linux 行銷主管 Efrain Rovira 說，客戶對於伺服器的需求越 來越嚴格，有超過 50% 的客戶選擇用 Linux。

除了 HP 和其他 Linux 公司如 Red Hat、Novell 的支持，IBM、 Intel、Oracle 擁抱 Linux 的動作對 Linux 的熱度更為加溫。這一 切並非因為商業公司受到開放源碼文化感召，只是它們看出 Linux 背後呈現的龐大商機。

軟體稽查工具商 Coverity 研究發現，Linux 核心比起一般商用軟體 套件，安全弱點要來得少了許多。這多虧了開放源碼的軟體開發方式。 鑑於近年來資訊安全受重視的程度，低安全度系統需要花費更多後續 維護成本，很可以想見的是，辦公室等商業環境對 Linux 桌面系統 的接受度將越來越高，在這些場合，有 Novell 等商業公司支援做後 盾的 Linux 機器，將會是取代現有桌面系統的首選。

Novell/SuSE 與 Red Hat 兩大 Linux 散佈套件之間，預料會有更白 熱化的競爭動作。相信 Red Hat 將尋求其企業金援，Novell 將更強 調 SuSE Linux 背後的行銷及開發團隊。而這場競爭也意味著較小型 的 Linux 版本可能在商業市場上，逐漸被邊緣化。

◎ 開放源碼軟體進駐私有作業系統桌面

即使 Linux 仍無法在桌面作業系統上佔有絕對優勢，而這也是一般 的預期，Windows 還不會被 Linux 完全取代。形形色色的開放源碼 桌面軟體，是另一股難以忽視的趨勢。Firefox 已經展現出非作業系 統類型的開放源碼桌面軟體，究竟有多大影響力。在未來兩三年內， 幾乎所有桌面系統上，都會安裝上這類的開放源碼軟體。

Firefox 之後，同樣出於 Mozilla 的郵件軟體 Thunderbird，極可 能是下一個搶灘 Windows 桌面的開放源碼軟體。它的目標就是取代 Outlook Express、Outlook 等軟體。搭配 Mozilla Sunbird 這一套 任務管理與行事曆軟體，彌補 Thunderbird 作為單純郵件軟體的不 足之處，相信這對雙人組未來可以有不錯的成績。

至於 Microsoft Office 這套可能是至今版圖最龐大的套件軟體呢？ Sun 的開放源碼 OpenOffice.org 2.0，似有一爭長短之能。Open- Office 遭人詬病的執行緩慢，到了 2.0 版將有長足進步。加上其絕 佳的 Office 檔案格式相容性，OpenOffice 可以成為 Microsoft Office 之外，相當好的辦公室套件軟體選擇。

對於軟體開發人員來說，開放源碼軟體工具似乎已是他們的首選了。 多數 Java 開發者會選用開放源碼平台與開發工具，C 與 C++ 呢？ GNU 工具可以滿足他們的需求。更別提其他 Perl、PHP、Python 等 開放源碼程式語言了。

◎ 大型商業公司浮出檯面支持開放源碼

去年開始，CA 等大型軟體公司以更為積極的態度跨入開放源碼領域， 開放源碼作為軟體開發方法，受到外界相當程度的重視。Apache、 Sendmail、PHP、Linux、MySQL 等軟體都是最佳例證。

預期 2005 年，開放源碼在企業領域的應用，將有超越以往的迅速成 長，而且其範圍不僅限於像 MySQL 這類在去年積極紮根於企業市場 的開放源碼軟體。這股企業開放源碼應用也不會侷限在 JBoss 這些 老牌開放源碼公司。像 SourceLabs、SpikeSource 等新進廠商，也 會是未來這股趨勢中的獲利者。

在 IBM、HP、CA 等大企業表態支援開放源碼之後，更多的企業用戶 對於採用開放源碼軟體的態度將更為正面。

未來一年內，對於希望起步採用開放源碼軟體的企業體，有許多困難 等待克服。最主要的將是人員訓練的問題。MySQL 執行長 Marten Mickos 表示，他們面對的許多企業用戶，對於人員再培訓，以及是 否需要養成內部開放源碼經驗等問題感到憂慮。

Mickos 認為，好消息是多半企業會發現自身已經有了開放源碼方面 的人才。有許多組織資訊人員在私底下早已使用過開放源碼軟體，甚 至應用到企業專案之中。雖然如此，正規訓練仍然是大型組織跨入開 放源碼應用這關的一大障礙。

雖然說還是有不少企業對開放源碼軟體採行敬而遠之的方針，甚至部 份公司從沒考慮過相關議題。往後一年內這類組織不會完全消失，但 無論如何，開放源碼軟體的影響力，最終還是會迫使各企業體正視這 個選擇。

◎ Linux 與開放源碼軟體於中國等遠東地區加速發展

Technology Business Research 分析師 Steve Trotta 認為，2005 年中國與其它遠東地區的 Linux 等開放源碼軟體發展將更為迅速。 受到 IBM 去年底將個人電腦業務賣給中國企業的影響，加上中國、 南韓、日本等國相當希望能擺脫對微軟的依賴性，這股趨勢將刺激 Linux 桌面的成長。

Trotta 也指出，在亞洲已成全球低價個人電腦製造地區此一背景下， 將對上述趨勢產生推波助瀾的效果。畢竟每一套 Windows 授權要價 125 美元，在一部僅售數百元的低價電腦中，佔去相當大的成本。像 Linux 這樣免費或可低價取得的作業系統，有助遠東地區電腦製造商 提高價格競爭力。

◎ 開放源碼獲利模式改變軟體公司企業模型

Forrester 應用軟體開發與基礎架構研究副總裁 John Rymer 指出， 軟體營利出現由軟體授權模式轉向支援與維護上。Sun 是其中一個例 子。該公司正打算把 Solaris 釋出，然後由支援和維護上獲利。Rymer 認為，這是受到開放源碼運動的影響。Rymer 也預測這股趨勢會在 2005 年持續下去，甚至更為蓬勃。

IDC 分析師 Dan Kusnetzky 說，現在 Linux 最常作為檔案伺服器、 資料庫伺服器、Java 伺服器或列印伺服器使用。在軟體成本上倍感 壓力的公司會轉而向開放源碼尋求其它可用選項。即使最後它們不選 用開放源碼產品，此舉也會促使其供應商提供開放源碼產品。

◎ 虛擬環境大行其道

Kusnetzky 表示，為降低成本，不少公司開始採納虛擬環境 （virtualized environments）。比起維護為數眾多的實體主機，在 企業級 Linux 大型主機上執行同數量的虛擬系統要來得便宜許多。 這些公司需要可自行安裝、管理、調整的軟體，而這類軟體多半採用 Linux 作為核心。

IBM 的隨需方案（on-demand initiatives）、HP 的調適性架構 （adaptive infrastructure）都屬於這類環境。Kusnetzky 估計虛 擬環境總市值約為 42 億美元，到了 2008 年更可成長至 142 億。

而 Linux 則從檔案、列印伺服器等基本架構，轉而成為金融服務、 運輸等強調模擬運算的場合中，高效能運算的平台選擇。Kusnetzky 表示，重要的是 Linux 正開始支援資料庫等構成商業運算環境的複 雜商用軟體。未來一年中這股趨勢成長將更為快速，對於開放源碼支 持者而言，這可說是最好的消息。

◎ 開放源碼相關法律服務盛行

可預見在 2005 年，有更多律師們會開始關心開放源碼社群，以及其 不依賴授權金的商業模式。當開放源碼只是黑客擺弄程式碼的代名詞， 法律議題可能派不上用場，遇上程式碼遭人濫用的情況，寫封文辭並 茂的電子郵件往往就足以解決問題。但是一旦大公司企業計畫中相關 作業系統或關鍵元件，有賴於開放源碼軟體授權上的解釋，法律顧問 就事關重大了。

在許多人眼中，開放源碼象徵的是一塊企業大餅，獲利空間大，相對 的，風險也高，在高風險環境下，將法律議題整合到企業計畫之內， 可以達到更大的成本效率。

為此，律師們得熟稔相關法律條文與規章，以及之前的種種判例。雖 然說開放源碼相關判例並不多見。預期在未來一年內，會有更多關於 開放源碼的法律事件，同時將伴隨著更多採納社群慣例的判例產生。
