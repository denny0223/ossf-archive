OSCON 2007 報導

建立日期 2007-07-24 16:00      作者是 謝良奇
2007年 O'Reilly 開放源碼大會 (O'Reilly Open Source Convention, OSCON) 日前在美國奧勒崗州波特蘭的奧勒崗會議中心舉辦，超過 3000 名產業名人、開發人員、黑客與企業家聚集到此討論及分享有關開放源碼計算的相關資訊。從商務到資料庫、Java、Linux、Perl、PHP、Ruby 等技術，OSCON 大會涵蓋了開放源碼中的各個領域。OSCON 大會前 2 天由鎖定技術主題的教學演說開跑，每天多達 80 個以上的主題。專題演說請來了包括 Simon Peyton-Jones、Jimmy Wales、Tim O'Reilly 與 Mark Shuttleworth 等人，不能免俗的也有 Tim O'Reilly 主持的 O'Reilly Radar。

◎ 微軟重申開放源碼立場

微軟在 OSCON 中透過向開放源碼組織 (Open Source Initiative, OSI) 提交共享源碼授權 (Shared Source License)爭取成為認證授權，重申與開放源碼合作的決心。

微軟平台策略總經理也是該公司開放源碼策略主導人的 Bill Hilf 在 OSCON 會中承認微軟與開放源碼社群間的關係的確是反反覆覆。不過微軟並沒有惡意。他表示微軟正向 OSI 提交爭取認證，他們得到的回應不差，但還有更多可以做的事，微軟與開放源碼未來合作的機會相當大。

微軟之前宣稱自由與開放源碼軟體侵害該公司超過 200 項專利，Hilf 被問到對此事的看法，他說，他們的確做得不對，如果照他的方式來，他一定不會這麼做。此事的後續效應一直還是他相當擔心的事。

接著他認為報導微軟專利立場的文章太激進，他當時立即在一次專訪中澄清那並非微軟的觀點。微軟與 Novell 間的協議雖然有許多人不認同，但也有不少人在其中看到好的部份。Hilf 認為，軟體專利是個棘手的議題，他也曾在 IBM 完成過不少專利。他認同智財權的概念，但關於專利與智財權仍有許多空間值得思考。

開放源碼框架 Spring 開發公司 Interface21 執行長 Rod Johnson 認為，廠商自訂授權造成授權間相容性的複雜問題，微軟尋求 OSI 認證表示微軟認同開放源碼並願意與其合作。

Sun 開放源碼長 Simon Phipps 表示，微軟這一步是眾所樂見的。他相信參與社群能修正微軟授權中的缺點。

Hilf 則表示他們正盡力讓微軟習慣與開放源碼互動。過去開放源碼彷彿是不能談論的話題，還有人問他可不可以瀏覽 Linux 的網站。如今微軟已成立 CodePlex 社群網站，與 JBoss 合作，最近還以非常自由的授權方式，釋出了 IronPython、IronRuby 與動態語言執行環境 (Dynamic Language Runtime)。

Hilf 說，微軟在開放源碼的互動上有一套“社群、合作與程式碼”的策略，程式碼是最重要的一環。

◎ Tim O'Reilly 遭批過分注重商業  Falkvinge 強調公民自由凌駕商業利益

在 OSCON 議程 Web 2.0 時代的授權(licensing in the Web 2.0 era)上，受邀參加的自由軟體基金會律師 Eben Moglen 當場給 O'Reilly 難看，指稱 O'Reilly 花太多時間在推銷 Web 2.0 上，應該把重點轉回放在自由軟體的核心議題。

他表示，他們應該談點跟每日企業新聞不同的話題，不應只是圍繞在誰出線了，誰淡出了這類的事。他說這個座談會不談重點，反而一直在強調無關緊要的東西。

Moglen 甚至描述 O'Reilly 目前對開放源碼軟體態度可以說是輕佻而無聊，並重批他只會跟在錢以及 Facebook 這些流行話題後面跑。他說，如何修改由一群支持末班企業模式並浪費時間在推銷商業產品的人所建立的公眾政策，才是我們應該要認真考慮的問題。

因為不斷向與會者宣傳 Web 2.0，的確讓 Tim O'Reilly 引來不少揶揄的笑聲，O'Reilly本人則認為雖然大夥都對這個字眼有微詞，但怎麼說這還是用來代表這波現象的一個不錯的代名詞。

O'Reilly 表示 Moglen 要將重點放在自由而非商業成功上的呼籲很重要，不過他也補充說商業也很重要。

Pirate Party 創辦人 Rickard Falkvinge 在“著作權制度與公民自由”(CopyrightRegime vs. Civil Liberties)演說中表示，為何我們不應允許電影製片商與唱片業者通過法律，讓他們可以檢查人們的電子通訊，為的是找出朋友間互相傳送的 mp3 檔案？以及應該讓法律與媒體大亨檢查人們發給醫生、律師或雇主的電子郵件嗎？

Falkvinge 表示，一邊是產業的收入來源，另一邊則是民主社會的基本價值。

◎ Firefox 延伸套件成 Web 平台

Mozilla 專案共同創辦人之一的 Mike Shaver 以 Firefox 工具列的生態圈 (The Firefox Toolbar Ecosystem) 為題發表演說，討論 Firefox 延伸套件的成果與開放源碼開發模式。與會者除了 Shaver 之外，還包括 AllPeers 的 Matthew Gertner 與 StumbleUpon 的 Garrett Camp，代表延伸套件的開發者社群。

主持人 O'Reilly 提出的第一個問題是，為什麼裝了延伸套件後 Firefox 效能就受到很大影響。Shaver 表示該瀏覽器的架構讓開發者可以輕易與 Firefox 中的其它服務互動，開發者為了取得完全的控制權以便塑造使用者的瀏覽器使用經驗，相對地在效能上就有所犧牲。

O'Reilly 提到 Netscape 共同創辦人 Mark Andreesen 將瀏覽器視為平台的想法，如今是否已經實現？與會者認為 Firefox 延伸套件的確建構在此一概念上。Shaver 甚至認為瀏覽器可視為存取 Web 這個產業至今最佳軟體平台的工具。

Camp 表示對自由軟體用戶的熱情印象深刻。當 StumbleUpon 修改 Internet Explorer 工具列的功能時，幾乎沒有收到回應，然而 Firefox 上做出修改時，卻收到潮水般的電子郵件回覆。

◎ Wikipedia 創辦人將發起開放源碼搜尋引擎計劃

Wikipedia 創辦人 Jimmy Wales 在 OSCON 上對外宣佈，他的下一個目標，也就是由社群開發的搜尋引擎，已經向前踏出一步。

在 OSCON 聚集的電腦科學家與開發人員面前，Wales 表示他所創立的 Wikia 已經收購了可做為新搜尋引擎基礎的搜尋技術 Grub。

Grub 採取類似於 SETI@home 專案尋找外星生命體的模式，借用自願者提供的電腦運算能力，產生全球資訊網索引。Grub 是 LookSmart 的技術，早期 LookSmart 經營網路目錄，目前是線上廣告公司。Wales 並未透露 Wikia 與 LookSmart 的交易細節。

Wales 的目的是希望藉由公開搜尋技術提高網路搜尋的精確率。Wales 表示會透過開放源碼授權釋出 Grub 原始碼，其它人也能加以改善。

Wales 認為僅能從少數幾個公司取得搜尋結果，而我們卻不知道這些結果如何產生，這並非眾所樂見。就好比你只有一個新聞來源。根據 Nielsen/Net- Ratings 的資料，美國 6 月所有的搜尋中，Google、Yahoo 與微軟就佔了約 92%。

儘管如此，Wales 以開放源碼開發搜尋引擎的想法，獲得的回應並不完全正面。例如搜尋引擎 Spock 執行長 Jaideep Singh 就不認為開放源碼一定適合搜尋引擎。開放源碼專案雖然可吸引成百上千的開發人員參與，但搜尋引擎背後除此之外還需要更多資源。從 4 月到 6 月，Google 在電腦設備與資料中心上，就花費了 5 億 7500 萬美元。

Wales 承認搜尋引擎的成本的確可觀，但他也認為開放源碼比傳統軟體開發模式，在經
濟上能發揮更大效率。

◎ Sun 靠 Indiana 專案加強 OpenSolaris 品質

Sun 作業系統長也是 Debian 作者 Ian Murdock 所描述的 Indiana 專案，似乎是一個類似紅帽 (Red Hat)的 Fedora 專案。這個新專案的核心圍繞著 OpenSolaris 的釋出頻率，希望每 6 個月就能釋出一版 Sun 所支援的新 OpenSolaris。

過去 Sun 約 3 年左右才會釋出完整的新版 Solaris 作業系統。使用者得透過支援服務取得新的功能，使用時也必須冒著一定風險。如今 Sun 為了使 Solaris 更像是正規與完整的作業系統，希望讓忠實支持者與開發人員更快拿到新的工具。Sun 希望在 2008 年春就在 Indiana 專案下釋出 OpenSolaris 的第一版本。

不少專家認為 Sun 想藉由 Indiana 專案讓 Solaris 向 Linux 靠攏。可以看的出來，Sun 正在調整 Solaris 的釋出間隔，而這件事早在 Sun 承諾讓 Solaris x86 復活並向開發者推廣這套作業系統時就應該完成。Murdock 則說，這不是學 Linux，而是 2 個國度的結合。

很少人會認為 Solaris 與 Linux 在開發社群享有同等級的關注，Sun 似乎正希望改變這個局面，Indiana 專案就是其中之一。加上 IBM、HP 不願外界開發者社群參與各自的 Linux 版本的開發，Sun 向開發者示好的動作應該能帶給 Sun 多一些競爭優勢。

◎ Oracle 的 Linux 分支版本將問世？

Ubuntu 贊助公司 Canonical 創辦人兼執行長 Mark Shuttleworth 表示，要不了多久，Oracle 很快地就會推出自己的 Linux 版本。

在 OSCON 接受專訪時，Shuttleworth 認為 Oracle 在釋出修補程式 (patches) 上投下太多人力，他們可能正在準備自己的散佈套件。

到目前為止，Oracle 涉獵 Linux 的程度只是針對紅帽企業版 Linux(Red Hat Enterprise Linux) 提供低成本的支援服務。Oracle 企業版 Linux (Oracle Enterprise Linux) 則是 Oracle 儘可能跟隨紅帽企業版 Linux 更新的開放源碼作業系統版本。

Shuttleworth 卻認為 Oracle 正在籌備自己的全功能版本 Linux 與紅帽、Novell 等公司競爭。

◎ 2007 年的開放源碼大獎得主出爐

Google 與 O'Reilly 的開放源碼大獎過去 2 年給過許多人在開放源碼創新、領導與奉獻上有卓越貢獻的人。過去得主包括 Cluetrain 宣言 (The Cluetrain Manifesto) 共同作者與 Linux Journal 資深編輯 Doc Searls、Ruby on Rails 創始人與 37Signals 合夥人 David Heinemeier Hansson、SQLite 作者 D. Richard Hipp 等。

今年的得主包括了最佳社群建構者 (Best Community Builder) Karl Fogel，身為 Subversion 專案創辦人與領導人，Subversion 和諧社群有賴於 Fogel 始終如一的領導與維持尊重文化所做的努力。

Pamela Jones 對於自由與開放源碼社群法律事件的研究與報導，使得其 Groklaw 網站成為開放源碼社群獲得法律事件分析的最佳去處，因而得到最佳 FUD 鬥士獎 (Best FUD Fighter)。

獲得最佳親和力架構師 (Best Accessibility Architect) 的 Aaron Leventhal 長期以來支持親和力議題。Aaron 在 Mozilla 專案開始之初就加入，並獨自將 Firefox 在網站親和力上的表現，從一開始極差的情況改善至今天成為該領域的首選解決方案。

David Recordon 最佳策略推手 (Best Strategist) 的獎項，來自推動 OpenID 所做的努力。他成功地將 OpenID 打造成可以取代非開放身份系統的選擇，

獲得卓越終身貢獻獎 (Outstanding Lifetime Contributions) 的 Paul Vixie 一直都是網域名稱系統 (Domain Name System) 的關鍵人物，他也是網路上最常見的網域名稱系統 BIND 的開發者與維護者，同時他更共同創辦了對抗垃圾信的非營利團體 MAPS，基於他對網際網路的卓越貢獻因而獲頒這個獎項。
