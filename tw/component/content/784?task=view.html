第三屆桌面架構會議登場 加速 Linux 桌面發展

建立日期 2006-11-28 08:00      最近更新在 2007-03-08 23:33 作者是 謝良奇
開放源碼開發實驗室 (Open Source Development Labs) 舉辦的第三屆桌面架構會議 (Desktop Architecture Meeting)，日前在美國奧勒岡州的波特蘭召開，吸引來自各機構超過 40 名開發人員參加，討論如何加速 Linux 桌面的發展。 來自 HP、X.org、紅帽 (Red Hat)、Nokia、Intel、OpenWengo、AMD、Xandros、KDE、GNOME 以及 Linuxprinting.org 等組織的開發人員，日前集聚波特蘭討論如何進一步加速 Linux 桌面的普及。

此次桌面架構會議的一大重點是解決 Linux 桌面上音效體驗的混亂現況。Linux 終端伺服器專案 (Linux Terminal Server) 的 Jim McQuillan 表示，這次會議最重要的議題之一，就是達成建構音效系統與單一完整 API 的共識。所幸 McQuillan 認為，這不代表 Linux桌面系統需要重頭來過，整個方向會從現有 API 加入缺少的部份進行。

由於這樣的 API 目前仍是構想，OpenWengo 專案的 Dave Neary 認為有必要舉行專門的會議討論音效問題，集聚應用軟體開發者、框架開發者、低階軟體開發者定義出單一的音效 API。

Neary 表示套件也是此次會議著墨的重點之一。他認為獨立軟體開發者在為應用軟體包裝套件時，面臨到為 Debian、Ubuntu、Fedora Core 各版本、RHEL 各版本、Mandriva、OpenSolaris 等不同散佈套件包裝不同套件的困難。套件問題連帶影響到硬體支援。Neary 認為散佈套件與核心版本多，增添了硬體廠商撰寫硬體驅動軟體時的困難。

Portland 專案也有所進展。開放源碼實驗室桌面 Linux 計劃經理 John Cherry 認為，Portland 專案有助應用軟體開發者選擇適當的開發工具套件，而無須擔憂用戶所使用的桌面 Linux 系統所可能帶來的差異。Portland 1.01 將會在一月時釋出，1.1 版則會在四月時釋出。Cherry 也表示，2007 底應該會有包含 Xsettings 的 Portland 1.5 釋出，讓跨越不同桌面的 X.org 組態控制更為簡單。

根據 Waldo Bastian 發表的郵件顯示，未來會有一個四人工作小組成立，目的在於了解獨立軟體商的需求，確認 GNOME 與 KDE 能夠用來實作 Portland 的現有功能，並提供初期實作。GNOME 專案的 Alex Graveley 與 KDE 專案的 George Staikos 將領導該小組並挑選其它 成員。

儘管桌面架構會議這種面對面討論效果不錯，對於開放源碼開發者而言，經常要參與大大小小的各種會議的確花去不少時間。Cherry 表示他們正計劃在 2007 年只舉辦一場會議，並且與其它會議相結合省去開發人員舟車勞頓之苦。目前 Cherry 只表示第四屆桌面架構會議會在 2007 年第四季與其它大型會議合併舉行。
