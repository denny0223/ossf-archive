Novell 公佈 SUSE Linux Enterprise Linux 開發計畫

建立日期 2008-03-25 08:00      作者是 謝良奇/編譯
於日前 Brainshare 2008 大會上，Novell 對外公佈 SUSE Linux Enterprise Linux (SLES) 11 的開發計畫。不過此一產品在 2009 甚至 2010 前不會出貨，2008 年底前也不會有確切的推出時間。儘管如此 Novell 強調了此一新版產品的關鍵功能，有意讓開放源碼社群開發者，了解該公司用戶提供的回應，以及社群端所能提供的協助。 Novell 對 SLES 11 擬出深具野心的計畫。SLES 11 將採用最新 Linux 核心，預計將是 2.6.27，並具備先進虛擬化技術、資料中心自動化、綠色 IT 以及整合式協同功能。Novell Linux 與開放平台解決方案產品行銷主任 Justin Steinman 表示，像關鍵任務資料中心技術、綠色 IT 等，將是各個釋出版本不變的核心目標，SLES 11 會特別注重相互運作性、Unix 移轉、虛擬化技術，這三項對中小企業具有關鍵重要性的領域。

Novell 和微軟過去的合作，根據 Steinman 的說法，將交由 SLES 11 持續拓展此一夥伴關係。由於中小企業組織內運用 Windows 普遍，SLES 11 的重點之一，即是擴展和 Windows 之間的相互運作性。 Steinman 認為 SUSE Linux Enterprise 可說是與 Windows 協同合作的最佳化 Linux，此項特點也是用戶做出 Linux 選擇時，該公司的核心差異點。

隨著 Novell 釋出 Mono 2.0 beta 版本，Novell 的 Windows 支援計畫之一，就是在 SLES 11 納入 Mono 2.0 開發框架，可以允許 .NET 應用軟體在 Linux 上執行。也將納入新版產品釋出的 Mono 移轉分析工具，可用來協助用戶了解其 .NET 應用軟體，是否適合移轉到 Linux 上。

此下一代 Linux 散佈套件在 Unix 移轉上，提出更好的支援。SLES 11 將納入一套 POSIX 叢集檔案系統，與分散式重複區塊設備支援，降低 Unix 移轉負擔。Steinman 表示，Unix 對中小企業太過昂貴與缺乏彈性，用戶得綁在昂貴的 SPARC 伺服器上。移轉至 Linux 可達同樣效能、可靠性與延展性，卻僅需 25% 成本。

至於虛擬化技術上，SLES 11 將運用最新的 Xen 虛擬化引擎，提供跨平台支援，包括統整 Linux、Windows 和 Netware 工作負載的能力。Steinman 表示 SLES 11 以簡化虛擬化技術為目標，並將有可嵌入於硬體中的獨立虛擬化平台。

SLES 11 另一項主要計畫，即為推出嵌入式、應用設備、即時版本，以及為 SAP 等特定軟體堆疊最佳化的版本。應用設備版本附帶工具集，可供用戶快速建構客製映像(images)。嵌入式版本如上所述，將由 OEM 廠商在硬體中，整合作業系統、虛擬化技術。即時版本提供的低延遲運算解決方案，可滿足金融機構等用戶對即時功能的需求。

儘管在北美地區，紅帽(Red Hat)仍為 Linux 伺服器市佔率之冠，Novell 宣稱 2007 年 SLES 平台已有 200% 成長，並增加 Wal-Mart、Office Depot、Siemens IT solutions、Casio 和 HSBC 等新用戶。
