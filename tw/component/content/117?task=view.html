Zimbra 目標對準微軟 Office、Exchange

建立日期 2007-01-11 04:51      最近更新在 2008-04-02 22:18 作者是 謝良奇/編譯
開放源碼傳訊與協同合作軟體供應商 Zimbra 正試圖用所謂 "mash-up" 的 Web 2.0 模式，為每日的電子郵件通訊嵌入網路服務 (Web Service)。將競爭目標放在微軟的 Office 與 Exchange，Zimbra 這間新興企業對於自己以 Ajax 為基礎的 API 模式信心滿滿。

Zimbra 執行長 Satish Dharmaraj 曾是 Sun JavaSoft 的一員，參與了 JavaServer Pages 的設計與開發工作。他認為 Sun 宣佈採用 GPL 將 Java 開放源碼令人驚訝，但時機已經太晚。

對於拿微軟 Outlook 與 Exchange 當對手，Dharmaraj 認為 Zimbra 採用開放標準是一大優勢。Zimbra Collaboration Suite 與協力資訊系統的整合機制 Zimlets，就是希望與 CRM、ERP 或網路應用軟體相結合。

Zimbra 目前已有 400 萬用戶，Dharmaraj 認為 Zimlets 讓用戶自行撰寫開放源碼的 API 功不可沒。他表示多達 70% 的用戶寫過內部的 Zimlets 在內部電子郵件中，存取如旅遊這類的資訊。許多人更是會撰寫 Zimlets 提供給 Zimbra 網站上的開放源碼社群使用。

相較於其它開放源碼競爭者如 Scalix 與 Open-Xchange，Dharmaraj 認為模仿私有軟體如微軟 Exchange 的做法並發展開放源碼版本，是無法與微軟競爭的，他們創辦 Zimbra 的目標，是走出一條獨特的路。

Dharmaraj 認為 GPL 完全是 Richard Stallman 一個人說了算的授權，因此 Zimbra 選擇採用 Mozilla 公用授權 (Mozilla Public License, MPL)，因為 MPL 給予開發者選擇是否要開放源碼的權利，創造了下游的附加價值。

除了 MPL 之外，Zimbra 的 Ajax web 與管理用戶端採用的是衍生自 MPL 的 Zimbra 公用授權 (Zimbra Public License, ZPL)。ZPL 要求衍生自 Zimbra 原始程式碼的產品，都需要標示其出處。

然而 ZPL 這類授權被 Bruce Perens、Larry Rosen 等開放源碼專家認為有偏離開放源碼精神的嫌疑。Bruce Perens 稱採用這類授權的軟體為標章軟體 (badgeware)。Zimbra則主張出處確認授權 (Attribution Assurance License) 也會要求標示軟體出處與作者。
