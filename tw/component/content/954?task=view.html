GNOME 將延伸至行動裝置 擴展開放源碼手機應用

建立日期 2007-04-21 08:00      最近更新在 2007-07-03 23:08 作者是 謝良奇

Nokia、Intel 等大廠支持的 GNOME 行動與嵌入式協會 (The GNOME Mobile & Embedded Initiative, GMAE) 的目的是提高開放源碼軟體在手機的開發與使用率，特別是開放源碼桌面技術 GNOME 專案的相關技術。

GNOME 基金會委員會董事之一也是 GNOME 行動與嵌入式協會創辦人 Jeff Waugh 表示，GNOME 一直是開放源碼與自由軟體創新的前驅者。透過 GNOME 行動與嵌入式協會，GNOME 會將軟體自由理念推展到新裝置、新市場與新的使用者身上。

在美國聖克拉拉舉行的嵌入式 Linux 大會 (Embedded Linux Con-ference)上公開宣佈成立的 GMAE，將開放給有意為新行動裝置開發 GNOME 應用軟體的開發者、製造商等。

自由軟體專案 GNOME 旨在開發 Linux 與基於 UNIX 作業系統上的完整桌面應用軟體，此外也包括建立新軟體的開發環境。同是 GMAE 會員之一的開放源碼開發商 Collabora 的經理 Philippe Kalaf 表示，嵌入式與行動市場以相當快的速度正在引入開放源碼，特別是 GNOME。

Canalys.com 分析師 Nick Spencer 認為，GMAE 希望將 GNOME 經驗拓展到行動與嵌入式裝置的想法相當不錯。行動裝置的 Linux 市場還有很長一段路要走，讓開發者有興趣為開放源碼作業系統開發應用軟體，並參與標準化過程是相當好的一件事。

根據 Canalys 的資料，2006 年底，相較於 Symbian 的 67% 市佔率與 Windows Mobile 的 14%，Linux 只佔智慧型手機約 6% 的市場，Blackberry 有 7%，其它作業系統包括 Palm 在內則佔 5%。

Spencer 認為，雖然 Linux 在智慧型手機上的應用率相對來說較低，隨著越來越多廠商從舊的私有手機作業系統出走，對於開放源碼作業系統的需求將會上升。亞洲地區特別是中國的需求成長十分快速。

GMAE 平台雖然只是 GNOME 平台的一部份，不過 GMAE 的開發者計劃為該平台擴充新功能，包括 Java Mobile & Embedded (JavaME) 平台、GeoClue 地理定位服務、ulseAudio 音效管理功能與手機電子郵件系統 Tinymail。

遵照 GNOME 的政策，GMAE 平台將以 LGPL 授權釋出，提供免授權費的技術使用，並鼓勵回饋平台程式碼修改給 GNOME 社群。
