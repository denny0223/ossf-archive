開放源碼2005年度回顧

建立日期 2005-12-23 08:00      最近更新在 2007-03-03 21:59 作者是 謝良奇

2005 這一年，開放源碼在個人應用軟體、客戶關係管理與資料庫等市場皆有斬獲，MySQL 等開放源碼公司成功地自 Oracle 等軟體大廠手中，取得若干版圖。

開放源碼開發組合 LAMP（Linux、Apache、MySQL 與 PHP、Perl 或 Python），逐漸成為企業應用軟體建置時，.Net 和 Java 之外的第三選擇。


麻薩諸塞州（Massachusetts）提案將以符合 OpenDocument 標準的應用軟體為採購標準，如此一來已宣稱不支援該標準格式的微軟產品，將被屏除於該州未來的採購程序之外。

儘管如此，開放源碼去年也並非完全順遂。首先，麻薩諸塞州改採標準格式 OpenDocument 的動作遭擱置。此外，SuSE 供應商 Novell 則在年底時宣佈裁撤 600 個工作職務。

專家認為雖然開放源碼要在市場上達到更廣泛的接受度，還是有不少障礙等待克服，但 2006 年對於開放源碼商業應用軟體數量與品質上的成長，將會是重要的一年，預期會有相當關鍵性的進展。

麻州 OpenDocument 提案受挫

去年 9 月，麻州公開其企業技術參考模式（Enterprise Technical Reference Model），其中對於辦公室套件軟體的文件格式做出規定。麻州決定只採用符合 OASIS 之 OpenDocument 標準的產品。由於微軟已經宣稱不會支援 OpenDocument 格式，這項行動表示未來該州的採購程序中，將不會有微軟產品的立足之地。

正當開放源碼社群為此感到高興時，麻州資訊技術部顧問長 Linda Hamel 與麻州資訊長 Peter Quinn 被要求參加參議院公聽會進行說明。該場公聽會傳出是因為國務卿 William Galvin 表示不支持該項決定而舉辦。

Hamel 在公聽會中指出反對採納 OpenDocument 檔案格式標準的團體可能受到來自微軟的影響。她說，這些團體大部份背後都有微軟捐款的支持。

反政府浪費（Citizens Against Government Waste）組織表示改用 OpenDocument，將因為州政府轉換現有檔案的需要，而增加不必要的成本。不過先前麻州網站上已經明白做出了解釋，只有 2007 年 1 月 1 日後建置的文件才必須採用 OpenDocument 格式。

此外，盲人電腦科學聯盟（National Federation of the Blind in Computer Science）則表達對於無障礙使用上的疑慮。該組織總裁 Curtis Chong 認為，微軟已經在 Office 產品中加入對於布拉耶（Braille）印表機與螢幕閱讀軟體的支援。Peter Quinn 澄清這些員工在找到其它替代軟體前，還是可以繼續使用微軟 Office。

舉行該場公聽會的參議員 Marc Pacheco 最後仍要求擱置改採 OpenDocument 的程序。

開放源碼應用軟體持續成長

開放源碼辦公室套件如 OpenOffice.org 在功能上穩定成長，這類軟體的用戶認為這些開放源碼軟體的表現已經持平甚至優於微軟 Office。

Linux 用戶與獨立顧問 David C. Niemi 認為 OpenOffice 已經有了與微軟 Office 競爭所需的 99% 功能。他表示 Linux 桌面作業系統也是一樣，如今所需的只是如何使其更快速、穩定。

11 月於開放源碼企業研討會（Open Source Business Conference）上展出的商業開放源碼資料庫廠商 MySQL 與 EnterpriseDB，在 2005 都有不錯的表現。開放源碼企業研討會主持人 Matt Asay 就表示，他相信這些公司證明了開放源碼能帶來獲利。

LAMP 穩佔應用軟體開發市場

俗稱 LAMP 的開放源碼軟體－Linux、Apache、MySQL 和腳本語言 PHP、Perl、Python 成了企業軟體開發的新興選擇，與 .Net 和 Java 鼎足而立。

ActiveGrid 在 6 月時宣佈與 MySQL、Covalent、Novell 與 Zend Technologies 合作，希望擴展 LAMP 在大型企業間的應用。

Sageza Group 資深分析師 Jim Balderston 表示，這並非一個業餘專案，當 Linux 的成功歷歷在目時，沒有理由認為同樣具備低成本、社群開發模式以及符合開放標準的 LAMP 不會享有相同的成果。

Balderston 認為 IBM 正緊密地與 LAMP 開發互動，例如與 ZEN Technologies 的合作。如此一來只會讓 LAMP 更為茁壯。

Oracle 收購 Innobase MySQL 用戶憂心

去年 10 月 Oracle 收購 Innobase 的消息，著實讓開放源碼資料庫 MySQL 的用戶開始擔心。Innobase 是一家芬蘭小公司，與 MySQL 有著緊密的合作關係，Innobase 開發的 InnoDB，是 MySQL 的關鍵元件之一。InnoDB 是一套交易引擎（transaction engine），負責 MySQL 5.0 若干先進資料庫功能的實作。

Oracle 此舉究竟是為了削弱潛在競爭對手的實力，或者有意將開放源碼產品納入企業的策略之一？MySQL 執行長 Marten Mickos 表示他相信整個開放源碼都在等待這個問題的答案。

前 Oracle 職員 Paola Lubet 認為，Oracle 為 MySQL 的用戶投下了相當大的不安全感。

分析師與市場專家表示，只有一件事是可以肯定的，那就是 Oracle 已經開始正視開放源碼資料庫成長所帶來的可能影響。

Oracle 在聲明中指出，該公司希望將業務支援擴展至開放源碼軟體。並且傳出 Oracle 總裁 Charles Phillips 致電 Marten Mickos，表達該公司希望繼續 MySQL 與 Innobase 的合約。不過 Oracle 發言人並不願對這個消息發表評論。
