KDE 4.2 正式釋出 加強 Plasma 桌面與跨平台相容性

建立日期 2009-01-27 08:00      最近更新在 2009-02-23 18:21 作者是 謝良奇/編譯

KDE 4.2 日前正式釋出，此一新版開放源碼桌面環境包含更為豐富完整的 Plasma shell，改進 3D 視覺效果支援。如果要問 Linux 桌面是否已經準備好推向主流用戶，最新 KDE 桌面背後的開發者可能會給予肯定的答案。

開放源碼 KDE 桌面環境背後的開發者已宣佈正式釋出 4.2 版。在 4.0 版本釋出後一年推出的此一最新版本，帶來更為優雅與完整的使用者體驗。KDE 4.2 是 KDE 4.x 桌面第二次主要釋出，其代號為 "The Answer"，透露出此一版本所設定的目標。此一釋出奠基於首次於 KDE 4.0 版問世的若干功能。

當然此一最新桌面環境會是 GNOME Linux 桌面的一大挑戰，也將是 Linux 體系挑戰微軟 Windows 的一大助力。

4.2 版穿插著許多重要的更新。Plasma 桌面 shell 加入額外的 panel 功能、新的警示 (notification) 系統，以及透過 Ruby 和 Python 撰寫 plasmoid 的支援。KDE 4.2 也改進了視覺效果與視窗管理功能。

這一次 KDE 的跨平台相容性在 4.2 版勝於以往，許多 KDE 的重要應用軟體，例如 Amarok、Kate 和 Konqueror，都可以搬到 Windows 和 Mac OS X 上使用。Windows 的移植也為成熟，甚至初步支援了 Plasma。

KDE 針對 4.2 版釋出的新聞稿表示，KDE 社群有自信，對於大多數使用者，這會是一套令人關注的產品。KDE 開發者 Wade Olson 表示，透過改良過的釋出指引與 liveCD， KDE4.2 測試與學習的門檻非常低。他說，他們對於 KDE 在過去僅僅 6 個月內的進展感到自豪。

KDE 4.2 的關鍵部份來自 Plasma 桌面介面的改進，KDE 開發者宣稱此一介面有助整理桌面。Plasma 採用 widget 的概念協助整理桌面，並存取應用軟體。Dolphin 檔案管理程式在使用性 (accessibility) 上也有所改善，當使用者拖拉檔案到資料夾時，資料夾會自動開啟。PowerDevil 工具改進了電源管理功能，可為行動運算需求最大化電池時間。

OpenSUSE 開發者 Will Stephenson 提到 KDE 4.2 的一項 ‘middle-click to ungroup’ 功能，有助於使用者管理相關連的視窗群組。他認為這項在 Plasma 中的工作列群組功能雖然簡單卻是必要的功能。

不過也不是每個人都樂見於在 2008 釋出的 KDE 4.0。例如根據報導 Linux 創造者 Linus Torvalds 在使用過之後，就從 KDE 跳槽變成了 GNOME 的用戶。Olson 對此表示，嘗試探索、學習與創新之時，飽受批評從來不是罕見的現象。他說 KDE 開發者希望人們能夠注意到 4.2 所帶來超越 4.1 的功能性、穩定性與效能，正如 4.1 超越 4.0 一樣。

紅帽 (Red Hat) Fedora 專案領導人 Paul Frields 對 KDE 4.2 釋出也表達了樂觀的看法。Frields 認為，許多 Plasma 在可用性的加強，以及對於用戶在 4.X 系列早期所不得見的功能性的回歸，都應該會讓 KDE 的用戶群樂見此一新版。Fedora 預計在 2 月 3 日的 Fedora 11 Alpha 中納入 KDE 4.2。

OpenSuse 社群經理 Joe Brockmeier 表示，OpenSUSE 用戶對 KDE 4.x 桌面釋出各有正反面的評價。

此外，近來 OpenChange 與 KDE 開放源碼專案的發展，將為微軟 Exchange 與開放源碼用戶端軟體，搭起傳訊與群組軟體相容性的橋樑。在 Linux 與開放源碼會議 Linux.conf.au 上，OpenChange 與 KDE 開發者 Brad Hards 表示，OpenChange 專案的最終目標，正是要實作 Outlook 使用的微軟 Exchange 協定 Exchange MAPI/RPC。

Hards 指出，他們正在處理微軟 Exchange 協定，達成大多數功能。其中，郵件算是簡單的，困難之處在於通訊錄 (address book)、行事曆 (calendar)、行程 (free-busy) 列表與備忘錄 (notes)。

OpenChange 擁有整合 Exchange 的用戶端與伺服器端程式庫，並相當依賴為 Samba 4 的程式碼。Hards 表示，用戶端的工作有較多進展，雖然已有伺服器的程式碼，但估計仍須 12 個月的開發時間，以便完成可供生產 (production) 環境使用的 OpenChange 伺服器。

儘管 OpenChange 已經擁有某些用戶端工具，但 Hards 也承認，這些工具對用戶而言不太容易上手，目前正在試圖整合到 GNOME 的 Evolution，以及 KDE 的個人資訊管理套件 Akonadi 中。

Akonadi 是一套框架，也是 KDE 4 個人資訊管理的中介程式 (broker)，但並非專屬於 KDE。Akonadi 伺服器是一套用戶端的應用程式，可作為 Kmail 之類用戶端應用軟體的資訊中介者。由於並非所有 KDE 應用軟體都已移植到 Akonadi，因此 Hards 預估 2009 年中，才會看到可供生產環境使用的 Exchange 與 KDE 4.3 整合。
