ARM 發起行動 Linux 計畫 開發 Linux 開放源碼平台

建立日期 2007-10-11 16:00      作者是 謝良奇/編譯
晶片製造商 ARM 在該公司的開發者大會上，宣佈與六家公司合作，將為下一代的行動應用，開發以 Linux 為基礎的開放源碼平台。與此一計畫相類似，皆以行動 Linux 平台為目標的組織，包括了 LiPS 論壇、LiMo 基金會與行動 Linux 計畫 (Mobile Linux Initiative，MLI)。ARM 的合作對象包含 Marvell、MontaVista、Movial、Mozilla、Samsung 與 德州儀器 (Texas Instruments，TI)。

這個名稱尚未決定的新組織，將開發以 Linux、Gnome Mobile、Mozilla Firefox 為基礎的開放源碼平台，並以 ARM 已開發的架構建立標準化平台，希望改善行動運算設備的彈性與功能。

藉由這個計畫，ARM 期望讓該公司的處理器在執行 Linux 的行動裝置上，支援當今的 Web 瀏覽體驗。為實現此一目標，GNOME、Firefox 與相關的瀏覽器延伸套件，將會被移植到基於 ARMv5/6/7 架構的特殊硬體上。

該平台將於 2008 年初釋出，使用該平台的設備預料要等到 2009 年才會問世。根據 ARM 表示，此一平台會採用開放源碼授權。

ARM 認為，多樣化的行動計算應用軟體與設備的上市時間短、競爭激烈以及用戶的高度需求，對於廠商而言是一大挑戰。要克服這些潛在障礙但無須縮短電池時間或限制用戶的網路瀏覽功能並非易事。

ARM 銷售與行銷執行副總裁 Mike Inglis 表示，今日的用戶技術知識豐富，他們希望不限時間與地點使用網路、存取多媒體內容與應用軟體，同時不希望在電池時間與功能上有所受限與不足。藉由與行動市場的關鍵廠商合作，我們將能以最為創新的裝置與應用軟體，滿足用戶的期望。

ARM 行動處理器團隊平台發展主任 Bob Morris 表示，ARM 新 Linux 計畫的第一階段會移植 GNOME Mobile 平台、Firefox 瀏覽器與 Flash、Java 等關鍵元件到合作夥伴的硬體上，先期硬體目標可能包括 Marvell 以 ARMv5 為基礎的晶片系統 (system-on-chip processors，SoCs) 與參考設計、Samsung 以 ARM11 為基礎的硬體，以及 TI 以 ARMv7 為基礎的 OMAP3430 晶片系統。之後也將移植到以剛發表的多核心 Cortex-A9 為基礎的硬體上。

第二階段將包含發展開放源碼應用軟體，來補足不足的功能缺口，如開放源碼 Linux 電信介面等。

Morris 認為他們的競爭對手 Intel 可說是 ARM 之所以投入移植 Web 技術到 ARM Linux 的原因之一。他說，在 Intel 的開發者論壇上，Intel 表示只有 x86 才能使用現代的網際網路，ARM 無法帶給使用者好的瀏覽體驗。Morris 認為，這並不是因為 ARM 處理器在功能上輸給對手。

Intel 在 7 月時創立了自己的行動 Linux 協會 Moblin，並提出以行動 Intel 架構處理器為基礎的 Linux 行動網際網路裝置 (Linux-based Mobile Internet Devices，MIDs)。

In-Stat 研究主任 Jim McGregor 表示，超行動裝置 (ultra mobile devices) 市場在新通訊標準、網際網路提供的內容豐富資料、新使用模式與新企業模式的推波助瀾之下，預料未來 10 年會有超過 20% 的復合年成長率，此一市場中的關鍵成功條件將有賴於在半導體、系統與軟體解決方案推展創新的產業標準。透過開放標準與不斷成長的產業支援，Linux 勢必有助推廣此一創新。
