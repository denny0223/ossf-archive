紅帽完成 JBoss 收購 服務導向架構產品整合蓄勢待發

建立日期 2006-06-06 08:00      最近更新在 2007-03-09 00:10 作者是 謝良奇
紅帽 (Red Hat) 宣佈收購 JBoss 的行動已經正式告一段落，這項消息來的比原先外界預料的晚。隨著 JBoss 成為紅帽的一個部門，一直以來被視為 Linux 散佈套件商的紅帽，也對外展現了成為服務導向架構商的企圖心。 紅帽是在四月時宣佈收購開放源碼 Java 中介軟體商 JBoss 的計劃，當時外界早已謠傳 JBoss 在尋求買主。紅帽已於 Red Hat Summit 年度使用者大會上，興奮地宣告這項收購已經完成的訊息。

紅帽計劃把 JBoss 產品整合到一系列有關服務導向架構的產品中，JBoss 已經擁有該領域的若干產品。

JBoss 創辦人兼執行長 Marc Fleury 已經被指派為資深副總裁與 JBoss 部門總經理的職務。

根據紅帽企業解決方案資深副總裁 Timothy Yeaton 的說法，紅帽最後所出的價錢是 4 億 2 千萬，40% 為現金，其餘 60% 則由紅帽股票支付。該價格比原先預期要高出 7 千萬美元。

依據紅帽官方的說法，這額外的 7 千萬美元的支付將視 JBoss 的財務情況而定。原本外界傳言有意收購 JBoss 的 Oracle，所出的價錢為 4 億 8 千萬美元。

JBoss 將立即成為紅帽部門之一，紅帽方面表示往後將由單一且可靠的全球產品支援組織提供用戶紅帽與 JBoss 產品。Yeaton 表示，JBoss 將會是自主部門，紅帽會保留整個核心 JBoss 團隊。

JBoss 讓紅帽收購後，最擔心的莫過於 JBoss 的 Windows 平台用戶。約有一半的 JBoss 用戶採用的是 Windows 平台，紅帽身為 Linux 廠商的事實讓這些用戶對於 JBoss 未來的 Windows 支援大感憂心。

IDC 系統軟體研究副總裁 Al Gillen 認為不需要太過擔心。他表示讓 JBoss 成為一個部門顯示 JBoss 產品將會延續下去。
