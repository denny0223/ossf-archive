IBM 全力支持開放源碼教育平台 向 Sakai Project 送贈源碼

建立日期 2007-03-24 08:00      最近更新在 2011-01-12 19:28 作者是 Linuxpilot
IBM 早前宣布將持有的教育軟體源碼，贈與開放源碼課程管理系統開發計畫 Sakai Project。Linuxpilot 抓緊機會，訪問了專程到香港的 IBM 國際教育產業市場發展部總裁 Michael D. King，進一步了解 IBM 對開放源碼和教育事業的藍圖。 Q： 可否簡單地介紹一下 Sakai Project？
A： Sakai Project 是由 University of Michigan、Indiana Univer- sity、MIT、Stanford 等大學於 2004 年共同推動的計畫，目的是打造一個免費的課程管理系統 (Course Management System)，作為全校教學使用。Michigan 大學於 2004 秋天首先開始全校性的試驗，其他的如 MIT、Stanford 等學校亦已於 2005 年秋天展開全面試驗。在本地亦有大學正在測試 Sakai，香港大學便是其中一家。Sakai 的技術以 Michigan 大學的 CHEF (CompreHensive collaborativE Framework) 為基礎，並以在美國大受歡迎的日本電視節目「料理之鐵人」中的（土反）井 (Sakai) 命名。

Q：市面上其實已經有同類的課程管理系統，Sakai Project 的意義又在哪裡？
A：北美和歐洲的大學在 1990 年代末至 2000 年初，用的都是像 Blackboard 或 WebCT 般的專屬軟體。這些軟體並不便宜，但卻比由學校自行開發的成本要低，是以這類專屬的課程管理系統才得以普及。現時在北美、歐洲及澳洲等地，教育機構引入課程管理系統的比率已差不多達百分之百，全科目都使用課程管理系統的學校比率亦超過百分之五十。隨著 Blackboard 於 2005 年收購 WebCT，課程管理系統的市場便出現了壟斷。Sakai Project 的意義在於學校可以共同架設一個開放源碼的教育平台，並開發代替現有專屬軟體的開放源碼產品。

Q：除了 Sakai Project 外，IBM 還參與了哪些開放源碼運動？
A：不論學界或企業，都必定希望有更多軟體可選擇，和更具彈性的開發路向，這正是我們參與眾多開放源碼開發計畫的理由。IBM 多年以來除了 Sakai Project 外，還一直支持著 Apache、Eclipse 等眾多開放源碼軟體的開發。去年我們更向 Sakai Project 送贈了以世界標準規格 SCORM(Sharable Contents Object Refference Model) 為基礎、改良共同學習環境的程式碼。Sakai 目前的最新版本為 2.2，可在 Linux、Solaris、Mac OS 10 和 Windows 台上運作。
