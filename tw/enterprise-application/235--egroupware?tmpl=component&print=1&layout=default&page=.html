中小企業資訊系統建置系列（四）群組軟體 eGroupware

建立日期 2006-01-19 08:00      最近更新在 2011-10-12 11:04 作者是 林崇仁
隨著各種資訊系統的成熟以及多元化，企業對資訊系統的依賴也日益增加。不過大部分的企業在建置企業內部整體的資訊架構時，很難在初期就想的很清楚，總是在一套接著一套購買以及開發後，突然間發現內部的系統已經盤根錯節，很難有單一的介面去管理。 而衍生出來的問題，如使用者單一的登入介面 (SSO：Single Sign On)、系統之間資料的交換或同步、使用者權限的控管問題..等。所以如果您有機會去接觸一些資訊系統較多的企業，通常的解決方案為，將所有的系統利用單 一的介面如員工入口網站，將所有的系統連結。而這樣的方式，並沒有實際的解決上述的問題，要解決這樣的問題，需要建立起一個協同作業的環境，而需要的資訊 軟體，一般我們稱之為群組軟體。今天筆者要分享的是開放原碼群組軟體的佼佼者－eGroupware。

eGroupware 軟體主要解決幾個主要常見的問題：

 

  1. 提供多元化的協同作業下的應用程式，包括：
    行事曆、檔案分享、討論區、新聞發佈、網站建立、電子郵件管理、工作流程、專案管理、線上聊 天、知識管理…等 20 幾套不同種類的軟體。而這些不同種類的應用程式之間，彼此的資料，又可以相互的應用。對於一般企業的需求已經相當的足夠。如果需要在 eGroupware下，建立其他的應用程式，只要依循其本身的設計架構，便可以很容易 Plug-In 到eGroupware 的使用環境。
  2. 解決權限的控管問題：
    當企業內部的資訊系統愈來愈多，首見問題便是，誰可以使用哪些的系統。eGroupwre 有非常好的權限控管機制，它可以選擇以群組或個人的方式，賦予使用者的使用權限。而值得一提的是，它的權限的控管部分，在某些的應用程式裡，甚至可以控管到資料的部份，相當的 難得。
  3. 使用者單一登入：
    新版的 eGroupware 除了原本的 LDAP 的認證及 e-mail 帳號的認證外，還提供與 Microsoft Active Directory 的認證，已經相當符合一般企業的需要。如果您新開發的程式，是依循 eGroupware 的架構，那認證及系統權限控管的問題，便可以交給 eGroupware 控管。
即將 release 的 Groupware 1.2 提供工作流程的應用，企業可以做一些表單流程設計或是簡單的線上簽核。另外，如果企業為 MSN 所苦惱，新版的 eGroupware 提供線上聊天的功能 (Chatty)， 將交談的對象局限在系統的使用者之下。其他諸如多國語言、使用者介面等，eGroupware 已經發展的相當的成熟完善。

而在實際導入系統時，所面臨的問題大多不會是在系統本身，而會是在使用者的使用習慣及接受度上。筆者的建議是，將幾個一般使用者非使用的功能，先 放在系統上。或是調整公司內部某些的行政程序，例如、將公司的公告放於其上，就是一種不錯的選擇。讓使用者有非用不可的理由，是群組軟體導入成功的關鍵。
